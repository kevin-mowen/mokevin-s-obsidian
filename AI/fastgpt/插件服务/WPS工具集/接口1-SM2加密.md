# 接口1：SM2 加密

## 基本信息

| 项目 | 说明 |
|------|------|
| 用途 | 将用户身份信息（notesId + 时间戳）进行国密 SM2 加密，生成密文用于后续 WPS 用户认证 |
| 地址 | `http://20.201.9.111:8180/api/sm2/encrypt` |
| 方法 | POST |
| Content-Type | application/json |
| 所属步骤 | 步骤 3.1（WPS 用户认证 - 加密阶段） |
| 调用方式 | 使用项目封装的 `POST` 函数（基于 fetch） |

## 请求参数

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| plaintext | string | 是 | 待加密的明文，为 JSON 字符串的 Base64 编码 |
| publicKey | string | 是 | SM2 加密公钥，固定值 |

### plaintext 构造过程

1. 构造 JSON 对象：`{ "timestamp": <当前毫秒时间戳>, "notes_id": <用户notesId> }`
2. 将 JSON 字符串进行 UTF-8 → Base64 编码
3. 编码后的字符串即为 plaintext 值

### publicKey 固定值

```
BL+htj+HB2hpBW8DBNKkMW7PenraiMTFM47OaFnWg6UbtcmUFnk7SWBB+JpUEhUSHZXDMOVlzBOG1MXi6Tc2iB0=
```

## 请求示例

```json
{
  "plaintext": "eyJ0aW1lc3RhbXAiOjE3MTI5MjgwMDAwMDAsIm5vdGVzX2lkIjoiemhhbmdzYW4ifQ==",
  "publicKey": "BL+htj+HB2hpBW8DBNKkMW7PenraiMTFM47OaFnWg6UbtcmUFnk7SWBB+JpUEhUSHZXDMOVlzBOG1MXi6Tc2iB0="
}
```

## 响应参数

| 字段 | 类型 | 说明 |
|------|------|------|
| ciphertext | string | SM2 加密后的密文字符串 |

## 响应示例

```json
{
  "ciphertext": "04a1b2c3d4e5f6..."
}
```

## 错误处理

- 当响应中 `ciphertext` 字段不存在时，抛出异常：`SM2 加密接口未返回 ciphertext: <完整响应体>`

## 注意事项

- 该服务为**内网服务**，地址 `20.201.9.111:8180`，部署环境需确保网络可达
- timestamp 为调用时的实时毫秒时间戳，不可复用
- notes_id 来源于 `systemVar.user.username`
- 该接口的输出（ciphertext）直接作为 [[接口2-获取AccessToken]] 的输入

## 上下游关系

- 上游输入：用户的 notesId（来自系统变量）
- 下游消费：[[接口2-获取AccessToken]] 使用返回的 ciphertext
