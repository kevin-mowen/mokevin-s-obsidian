# 接口4：获取下载链接

## 基本信息

| 项目 | 说明 |
|------|------|
| 用途 | 根据 drive_id 和 file_id 获取文件的最终下载 URL |
| 地址 | `https://weboffice.cib.com.cn/openapi/v7/drives/{driveId}/files/{fileId}/download?with_hash=true` |
| 方法 | GET |
| Content-Type | 空字符串 |
| 所属步骤 | 步骤 6（获取下载链接） |
| 调用方式 | 原生 `fetch` |

## 路径参数

| 参数 | 类型 | 来源 |
|------|------|------|
| driveId | string | [[接口3-获取分享文件详情]] 返回的 data.drive_id |
| fileId | string | [[接口3-获取分享文件详情]] 返回的 data.file_id |

## 查询参数

| 参数 | 值 | 说明 |
|------|-----|------|
| with_hash | true | 返回结果中包含文件哈希信息 |

## 请求头

| Header | 值 | 说明 |
|--------|-----|------|
| Content-Type | （空字符串） | 无请求体 |
| X-Kso-Date | RFC2822 格式时间 | **重新生成**，与接口 3 的时间不同 |
| X-Kso-Authorization | `KSO-1 {APP_ID}:{签名}` | **重新计算**的 KSO-1 签名 |
| Authorization | `Bearer {access_token}` | **复用**接口 2 返回的同一个令牌 |

## KSO-1 签名计算

针对本接口的签名参数：

| 参数 | 值 |
|------|-----|
| method | `GET` |
| uri | `/v7/drives/{driveId}/files/{fileId}/download?with_hash=true` |
| contentType | 空字符串 |
| ksoDate | 当前 RFC2822 时间（重新获取） |
| requestBody | 空字符串 |

签名步骤：
1. 因无请求体，SHA256Hex 为空字符串
2. 拼接签名原文：`"KSO-1" + "GET" + "/v7/drives/{driveId}/files/{fileId}/download?with_hash=true" + "" + ksoDate + ""`
3. 使用 APP_KEY（`SKmizelipfjracld`）对签名原文做 HMAC-SHA256，输出 hex
4. 最终值：`KSO-1 VIGEKDPAZAWRRFXK:{hex签名}`

> **注意**：URI 中包含查询参数 `?with_hash=true`，签名计算时需包含完整 URI（含查询参数）。

## 成功响应

| 路径 | 类型 | 说明 |
|------|------|------|
| data | object | 下载信息对象 |
| data.url | string | 文件的实际下载地址 |

### 成功响应示例

```json
{
  "data": {
    "url": "https://weboffice.cib.com.cn/download/xxxxx?token=yyyyy&expires=zzzzz"
  }
}
```

## 失败响应

HTTP 状态码 400 或响应体中含 `msg` 字段：

| 路径 | 类型 | 说明 |
|------|------|------|
| msg | string | 错误描述信息 |

### 失败响应示例

```json
{
  "msg": "文件不存在或无权限访问"
}
```

## 错误处理

代码中对三种情况分别处理：

| 条件 | 处理方式 |
|------|---------|
| `data.url` 存在 | 返回 `{ result: 下载URL }` |
| HTTP 400 或存在 `msg` 字段 | 返回 `{ result: "获取下载链接失败，报错信息为：{msg}" }` |
| 其他情况 | 返回 `{ result: "获取下载链接失败，未返回 data.url，响应：{完整响应}" }` |

> **注意**：本接口的错误不会抛异常，而是将错误信息包装在 result 字段中返回给用户。这与接口 1-3 的行为（抛异常）不同。

## 注意事项

- KSO 签名的时间和签名值必须**重新生成**，不能复用接口 3 的签名
- access_token 则**复用**接口 2 获取的同一个令牌
- 返回的下载 URL 可能有时效性，具体过期时间由 WPS 侧控制
- URI 中的查询参数 `with_hash=true` 参与签名计算

## 上下游关系

- 上游输入：[[接口3-获取分享文件详情]] 返回的 drive_id 和 file_id + [[接口2-获取AccessToken]] 返回的 access_token
- 下游消费：无，返回的下载 URL 即为最终输出
