# 接口3：获取分享文件详情

## 基本信息

| 项目 | 说明 |
|------|------|
| 用途 | 根据分享链接中的 shareFileId 获取文件的 drive_id 和 file_id，用于后续下载 |
| 地址 | `https://weboffice.cib.com.cn/openapi/v7/links/{shareFileId}/meta` |
| 方法 | GET |
| Content-Type | 空字符串 |
| 所属步骤 | 步骤 4（获取分享文件详情） |
| 调用方式 | 原生 `fetch` |

## 路径参数

| 参数 | 类型 | 说明 |
|------|------|------|
| shareFileId | string | 从分享链接中正则提取，匹配规则：`weboffice\.cib\.com\.cn/weboffice/l/([^\s]+)` |

## 请求头

| Header | 值 | 说明 |
|--------|-----|------|
| Content-Type | （空字符串） | 无请求体 |
| X-Kso-Date | RFC2822 格式时间 | 如 `Sat, 12 Apr 2026 12:00:00 GMT` |
| X-Kso-Authorization | `KSO-1 {APP_ID}:{签名}` | KSO-1 签名鉴权 |
| Authorization | `Bearer {access_token}` | [[接口2-获取AccessToken]] 返回的令牌 |

## KSO-1 签名计算

针对本接口的签名参数：

| 参数 | 值 |
|------|-----|
| method | `GET` |
| uri | `/v7/links/{shareFileId}/meta` |
| contentType | 空字符串 |
| ksoDate | 当前 RFC2822 时间 |
| requestBody | 空字符串 |

签名步骤：
1. 因无请求体，SHA256Hex 为空字符串
2. 拼接签名原文：`"KSO-1" + "GET" + "/v7/links/{shareFileId}/meta" + "" + ksoDate + ""`
3. 使用 APP_KEY（`SKmizelipfjracld`）对签名原文做 HMAC-SHA256，输出 hex
4. 最终值：`KSO-1 VIGEKDPAZAWRRFXK:{hex签名}`

## 响应参数

| 路径 | 类型 | 说明 |
|------|------|------|
| data | object | 文件详情对象 |
| data.drive_id | string | 文件所在的驱动器 ID |
| data.file_id | string | 文件 ID |

## 响应示例

```json
{
  "data": {
    "drive_id": "drv_abc123",
    "file_id": "file_xyz789"
  }
}
```

## 错误处理

- 当 `data.drive_id` 或 `data.file_id` 不存在时，抛出异常：`获取分享文件详情失败，未返回 drive_id 或 file_id: <完整响应体>`

## 注意事项

- X-Kso-Date 和 X-Kso-Authorization 必须配对使用，签名计算中的时间必须与请求头中的 X-Kso-Date 一致
- 该接口同时需要 KSO 签名鉴权（应用级）和 Bearer Token 鉴权（用户级）
- Content-Type 设置为空字符串，不是省略——请求头中显式传空值

## 上下游关系

- 上游输入：正则提取的 shareFileId + [[接口2-获取AccessToken]] 返回的 access_token
- 下游消费：[[接口4-获取下载链接]] 使用返回的 drive_id 和 file_id
