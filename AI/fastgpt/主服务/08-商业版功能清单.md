# FastGPT 商业版功能清单

> 版本：4.14.3.9 | 分析日期：2025-12-06

---

## 1. 概述

FastGPT 采用开源 + 商业版的模式。开源版包含核心功能，商业版提供企业级高级功能。

### 架构说明

商业版功能通过 `/api/proApi/*` 路由提供，这些接口分为两类：

| 类型 | 说明 | 数量 |
|------|------|------|
| **开源实现** | 在 `projects/app/src/pages/api/proApi/` 目录下有完整实现 | ~46 个 |
| **商业版专属** | 需要配置 `PRO_URL` 指向商业版后端服务，否则返回 404 | ~60+ 个 |

## 3. 商业版专属功能详解

### 3.1 应用评估系统 (Evaluation)

**功能描述**：批量测试 AI 应用的回答质量，自动评分。

**前端页面**：
- `/dashboard/evaluation` - 评估列表
- `/dashboard/evaluation/create` - 创建评估任务

**API 接口**（全部商业版专属）：

| 接口 | 方法 | 说明 |
|------|------|------|
| `/proApi/core/app/evaluation/create` | POST | 创建评估任务 |
| `/proApi/core/app/evaluation/list` | POST | 获取评估列表 |
| `/proApi/core/app/evaluation/delete` | DELETE | 删除评估任务 |
| `/proApi/core/app/evaluation/listItems` | POST | 获取评估项列表 |
| `/proApi/core/app/evaluation/deleteItem` | DELETE | 删除评估项 |
| `/proApi/core/app/evaluation/retryItem` | POST | 重试评估项 |
| `/proApi/core/app/evaluation/updateItem` | POST | 更新评估项 |
| `/proApi/core/app/evaluation/exportItems` | GET | 导出评估结果 |

**数据结构**（开源版已定义）：
- `packages/service/core/app/evaluation/evalSchema.ts` - 评估任务模型
- `packages/service/core/app/evaluation/evalItemSchema.ts` - 评估项模型

**评分维度**：
- `accuracy` - 准确性
- `relevance` - 相关性
- `semanticAccuracy` - 语义准确性
- `score` - 综合评分

## 5. 配置说明

### 5.1 启用商业版功能

在 `.env.local` 中配置：

```bash
# 商业版服务地址
PRO_URL=https://your-pro-service-url

# 商业版授权 Token（如需要）
PRO_TOKEN=your-pro-token
```

### 5.2 功能检测

前端通过以下方式检测商业版功能：

```typescript
// 检查 PRO_URL 是否配置
const isPro = !!process.env.PRO_URL;

// 系统配置中的功能开关
const { feConfigs } = useSystemStore();
// feConfigs.isPlus - 是否为商业版
```

## 6. 开发建议

### 6.1 自行实现商业版功能

如需在开源版中实现商业版功能，可参考：

1. **数据模型**：大部分数据模型已在 `packages/service/` 中定义
2. **类型定义**：类型定义在 `packages/global/` 中
3. **API 路由**：在 `projects/app/src/pages/api/` 下创建对应路由
4. **前端调用**：修改 `projects/app/src/web/` 下的 API 调用，将 `/proApi/` 改为 `/api/`

### 6.2 示例：实现评估功能

```typescript
// 1. 数据模型已存在
// packages/service/core/app/evaluation/evalSchema.ts
// packages/service/core/app/evaluation/evalItemSchema.ts

// 2. 创建 API 路由
// projects/app/src/pages/api/core/app/evaluation/create.ts
// projects/app/src/pages/api/core/app/evaluation/list.ts
// ...

// 3. 修改前端 API 调用
// projects/app/src/web/core/app/api/evaluation.ts
// 将 '/proApi/core/app/evaluation/...' 改为 '/api/core/app/evaluation/...'
```
