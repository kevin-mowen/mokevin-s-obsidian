## 1. 功能概述

应用管理系统负责 FastGPT 中 AI 应用的完整生命周期管理，包括：

- **应用类型管理**：简单应用、Agent、工作流、插件等 7 种类型
- **应用 CRUD**：创建、读取、更新、删除
- **版本管理**：自动保存、发布版本、版本回滚
- **权限控制**：协作者管理、权限继承
- **应用分享**：外链分享、iframe 嵌入

### 4.3 应用状态管理

**文件位置**: `projects/app/src/web/core/app/store/useAppStore.ts`

```typescript
import { create } from 'zustand';
import { getAppDetailById, updateApp } from '@/web/core/app/api';

type AppState = {
  appDetail: AppDetailType | null;
  isLoading: boolean;

  // Actions
  loadAppDetail: (appId: string) => Promise<void>;
  updateAppDetail: (updates: Partial<AppDetailType>) => void;
  saveApp: () => Promise<void>;
};

export const useAppStore = create<AppState>((set, get) => ({
  appDetail: null,
  isLoading: false,

  loadAppDetail: async (appId) => {
    set({ isLoading: true });
    try {
      const detail = await getAppDetailById(appId);
      set({ appDetail: detail });
    } finally {
      set({ isLoading: false });
    }
  },

  updateAppDetail: (updates) => {
    set((state) => ({
      appDetail: state.appDetail
        ? { ...state.appDetail, ...updates }
        : null
    }));
  },

  saveApp: async () => {
    const { appDetail } = get();
    if (!appDetail) return;

    await updateApp({
      appId: appDetail._id,
      modules: appDetail.modules,
      edges: appDetail.edges,
      chatConfig: appDetail.chatConfig
    });
  }
}));
```

### 5.1 创建应用 API

**文件位置**: `projects/app/src/pages/api/core/app/create.ts`

```typescript
import { NextAPI } from '@/service/middleware/entry';
import { authUserPer } from '@fastgpt/service/support/permission/user/auth';
import { authApp } from '@fastgpt/service/support/permission/app/auth';
import { MongoApp } from '@fastgpt/service/core/app/schema';
import { checkTeamAppLimit } from '@fastgpt/service/support/permission/teamLimit';

export type CreateAppBody = {
  parentId?: ParentIdType;
  name?: string;
  avatar?: string;
  intro?: string;
  type?: AppTypeEnum;
  modules: AppSchema['modules'];
  edges?: AppSchema['edges'];
  chatConfig?: AppSchema['chatConfig'];
  templateId?: string;           // 来源模板 ID
  utmParams?: ShortUrlParams;    // 营销追踪参数
};

async function handler(req: ApiRequestProps<CreateAppBody>) {
  const { parentId, name, avatar, intro, type, modules, edges, chatConfig, templateId, utmParams } = req.body;

  if (!name || !type || !Array.isArray(modules)) {
    return Promise.reject(CommonErrEnum.inheritPermissionError);
  }

  // 1. 权限验证（有父目录则验证父目录写权限，否则验证团队创建应用权限）
  const { teamId, tmbId, userId, isRoot } = parentId
    ? await authApp({ req, appId: parentId, per: WritePermissionVal, authToken: true })
    : await authUserPer({ req, authToken: true, per: TeamAppCreatePermissionVal });

  // 2. 团队应用数量上限校验
  await checkTeamAppLimit(teamId);

  // 3. 创建应用（内部处理事务）
  const appId = await onCreateApp({
    parentId,
    name,
    avatar,
    intro,
    type,
    modules: await removeUnauthModels({ modules, allowedModels: myModels }),  // 过滤无权限的模型
    edges,
    chatConfig,
    teamId,
    tmbId,
    templateId
  });

  // 4. 追踪统计
  pushTrack.createApp({ type, uid: userId, teamId, tmbId, appId, ...utmParams });

  return appId;
}

export default NextAPI(handler);

// onCreateApp 内部实现（事务中执行）：
// 1. 验证父文件夹类型匹配（工具只能放工具文件夹，应用只能放应用文件夹）
// 2. 创建应用记录 MongoApp.create()
// 3. 非文件夹类型：创建初始版本 MongoAppVersion.create()
// 4. 创建所有者权限记录 MongoResourcePermission.insertOne()
// 5. 记录审计日志 addAuditLog()
```

**关键点说明**：

1. **权限验证分支**：有 `parentId` 时验证文件夹写权限，否则验证团队创建应用权限
2. **模型过滤**：创建时会过滤掉用户无权使用的 AI 模型
3. **初始版本**：非文件夹类型会同时创建一个初始发布版本
4. **事务执行**：创建过程在 MongoDB 事务中执行，保证原子性

### 5.2 获取应用列表 API

**文件位置**: `projects/app/src/pages/api/core/app/list.ts`

```typescript
import { NextAPI } from '@/service/middleware/entry';
import { authUserPer } from '@fastgpt/service/support/permission/user/auth';
import { MongoApp } from '@fastgpt/service/core/app/schema';
import { MongoResourcePermission } from '@fastgpt/service/support/permission/schema';

export type ListAppBody = {
  includeReadable?: boolean;      // 是否包含只读应用（默认只返回可写/可管理的）
  parentId?: ParentIdType;        // 父文件夹 ID
  type?: AppTypeEnum | AppTypeEnum[];  // 应用类型过滤
  getRecentlyChat?: boolean;      // 获取最近聊天的应用
  searchKey?: string;             // 搜索关键词
};

/*
  获取 APP 列表权限流程：
  1. 校验 folder 权限和获取 team 权限（owner 单独处理）
  2. 获取 team 下所有 app 权限，获取我的所有组，计算出我所有的 app 权限
  3. 过滤我有权限的 app，以及当前 parentId 的 app
  4. 根据过滤条件获取 app 列表
  5. 遍历搜索出来的 app，并赋予权限（继承的 app 使用 parent 的权限）
  6. 再根据 read 权限进行一次过滤
*/

async function handler(req: ApiRequestProps<ListAppBody>): Promise<AppListItemType[]> {
  const { includeReadable = false, parentId, type, getRecentlyChat, searchKey } = req.body;

  // 1. 权限验证（如果有 parentId，同时验证文件夹权限）
  const [{ tmbId, teamId, permission: teamPer }] = await Promise.all([
    authUserPer({
      req,
      authToken: true,
      authApiKey: true,
      per: ReadPermissionVal
    }),
    ...(parentId
      ? [authApp({ req, authToken: true, authApiKey: true, appId: parentId, per: ReadPermissionVal })]
      : [])
  ]);

  // 2. 获取团队所有应用权限、我的组、我的组织
  const [roleList, myGroupMap, myOrgSet] = await Promise.all([
    MongoResourcePermission.find({
      resourceType: PerResourceTypeEnum.app,
      teamId,
      resourceId: { $exists: true }
    }).lean(),
    getGroupsByTmbId({ tmbId, teamId }).then((items) => {
      const map = new Map<string, 1>();
      items.forEach((item) => map.set(String(item._id), 1));
      return map;
    }),
    getOrgIdSetWithParentByTmbId({ teamId, tmbId })
  ]);

  // 3. 过滤出我的权限列表（个人权限 + 组权限 + 组织权限）
  const myPerList = roleList.filter(
    (item) =>
      String(item.tmbId) === String(tmbId) ||
      myGroupMap.has(String(item.groupId)) ||
      myOrgSet.has(String(item.orgId))
  );

  // 4. 构建查询条件（根据不同场景）
  const findAppsQuery = (() => {
    // 最近聊天场景
    if (getRecentlyChat) {
      return {
        visibilityType: 0,  // 只查看可见的
        teamId,
        type: { $in: [AppTypeEnum.workflow, AppTypeEnum.simple, AppTypeEnum.workflowTool] }
      };
    }

    // 权限过滤：非 owner 只能看到有权限的应用
    const idList = { _id: { $in: myPerList.map((item) => item.resourceId) } };
    const appPerQuery = teamPer.isOwner
      ? { parentId: parentId ? parseParentIdInMongo(parentId) : null }
      : parentId
        ? { $or: [idList, parseParentIdInMongo(parentId)] }
        : { $or: [idList, { parentId: null }] };

    // 搜索条件
    const searchMatch = searchKey
      ? { $or: [
          { name: { $regex: new RegExp(`${replaceRegChars(searchKey)}`, 'i') } },
          { intro: { $regex: new RegExp(`${replaceRegChars(searchKey)}`, 'i') } }
        ]}
      : {};

    // 类型过滤
    const _type = type
      ? (Array.isArray(type) ? { $in: type } : type)
      : { $ne: AppTypeEnum.hidden };  // 默认排除 hidden 类型

    return {
      visibilityType: 0,
      ...appPerQuery,
      teamId,
      type: _type,
      ...searchMatch,
      ...parseParentIdInMongo(parentId)
    };
  })();

  // 5. 查询应用列表
  const myApps = await MongoApp.find(
    findAppsQuery,
    '_id parentId avatar type name intro tmbId updateTime pluginData inheritPermission modules'
  )
    .sort({ updateTime: -1 })
    .limit(getRecentlyChat ? 15 : searchKey ? 50 : undefined)
    .lean();

  // 6. 计算每个应用的权限（处理继承权限）
  const formatApps = myApps.map((app) => {
    const getPer = (appId: string) => {
      const tmbRole = myPerList.find(
        (item) => String(item.resourceId) === appId && !!item.tmbId
      )?.permission;
      const groupAndOrgRole = sumPer(
        ...myPerList
          .filter((item) => String(item.resourceId) === appId && (!!item.groupId || !!item.orgId))
          .map((item) => item.permission)
      );
      return new AppPermission({
        role: tmbRole ?? groupAndOrgRole,
        isOwner: String(app.tmbId) === String(tmbId) || teamPer.isOwner
      });
    };

    // 继承权限处理：非文件夹 + 有父级 + 开启继承 → 合并父级权限
    const Per = (!AppFolderTypeList.includes(app.type) && app.parentId && app.inheritPermission)
      ? getPer(String(app.parentId)).addRole(getPer(String(app._id)).role)
      : getPer(String(app._id));

    return { ...app, permission: Per, private: /* 计算是否私有 */ };
  })
  // 7. 按权限过滤
  .filter((app) =>
    includeReadable
      ? app.permission.hasReadPer
      : app.permission.hasWritePer || app.permission.hasManagePer
  );

  // 8. 添加创建者信息
  return addSourceMember({ list: formatApps });
}

export default NextAPI(handler);
```

**核心逻辑说明**：

1. **权限来源**：个人权限 + 所属组权限 + 所属组织权限
2. **继承权限**：非文件夹应用如果开启 `inheritPermission`，会合并父文件夹的权限
3. **默认过滤**：默认只返回有写权限或管理权限的应用，`includeReadable=true` 时包含只读
4. **visibilityType**：用于过滤隐藏的应用

### 5.3 更新应用 API

**文件位置**: `projects/app/src/pages/api/core/app/update.ts`

```typescript
import { NextAPI } from '@/service/middleware/entry';
import { authApp } from '@fastgpt/service/support/permission/app/auth';
import { authUserPer } from '@fastgpt/service/support/permission/user/auth';
import { MongoApp } from '@fastgpt/service/core/app/schema';
import { beforeUpdateAppFormat } from '@fastgpt/service/core/app/controller';

export type AppUpdateQuery = { appId: string };
export type AppUpdateBody = AppUpdateParams;

// 更新应用接口，包括：
// 1. 更新应用信息（名称、头像、介绍等）
// 2. 更新应用编排信息（nodes、edges、chatConfig）
// 3. 移动应用到其他文件夹
//
// 权限要求：
// - 更新信息和编排：需要应用的写权限
// - 移动应用：需要源目录和目标目录的管理权限，根目录操作需要团队创建应用权限

async function handler(req: ApiRequestProps<AppUpdateBody, AppUpdateQuery>) {
  const { parentId, name, avatar, intro, nodes, edges, chatConfig, teamTags } = req.body;
  const { appId } = req.query;

  if (!appId) {
    Promise.reject(CommonErrEnum.missingParams);
  }

  const isMove = parentId !== undefined;

  // 1. 获取应用信息和基础权限（先用读权限，后续根据操作类型验证）
  const { app, permission, teamId, tmbId } = await authApp({
    req,
    authToken: true,
    appId,
    per: ReadPermissionVal
  });

  // 2. 移动操作的权限验证
  if (isMove) {
    // 移动到目标文件夹：验证目标文件夹管理权限
    if (parentId) {
      await authApp({ req, authToken: true, appId: parentId, per: ManagePermissionVal });
    }
    // 从源文件夹移出：验证源文件夹管理权限
    if (app.parentId) {
      await authApp({ req, authToken: true, appId: app.parentId, per: ManagePermissionVal });
    }
    // 涉及根目录：验证团队创建应用权限
    if (parentId === null || !app.parentId) {
      await authUserPer({ req, authToken: true, per: TeamAppCreatePermissionVal });
    }
  } else {
    // 非移动操作：验证写权限
    if (!permission.hasWritePer) {
      return Promise.reject(AppErrEnum.unAuthApp);
    }
  }

  // 3. 格式化节点数据（加密密钥等）
  beforeUpdateAppFormat({ nodes });

  // 4. 执行更新（移动操作需要事务处理权限同步）
  if (isMove) {
    await mongoSessionRun(async (session) => {
      // 获取目标文件夹的协作者
      const parentClbs = await getResourceOwnedClbs({ teamId, resourceId: parentId, ... });
      // 同步自身权限
      await syncCollaborators({ resourceId: app._id, collaborators: parentClbs, ... });
      // 同步子应用权限
      await syncChildrenPermission({ resource: app, collaborators: parentClbs, ... });
      // 更新应用
      await MongoApp.findByIdAndUpdate(appId, { ...updateData, inheritPermission: true }, { session });
    });
  } else {
    await MongoApp.findByIdAndUpdate(appId, updateData);
  }

  // 5. 记录审计日志
  addAuditLog({ tmbId, teamId, event: isMove ? AuditEventEnum.MOVE_APP : AuditEventEnum.UPDATE_APP_INFO, ... });
}

export default NextAPI(handler);
```

**关键点说明**：

1. **权限分层验证**：移动操作需要源目录、目标目录的管理权限，普通更新只需写权限
2. **权限同步**：移动应用时会同步目标文件夹的协作者权限到应用及其子应用
3. **节点格式化**：`beforeUpdateAppFormat` 处理密钥加密、数据集格式化等
4. **审计日志**：区分移动和更新两种操作类型

### 5.4 删除应用 API

**文件位置**: `projects/app/src/pages/api/core/app/del.ts`

```typescript
import type { NextApiRequest, NextApiResponse } from 'next';
import { authApp } from '@fastgpt/service/support/permission/app/auth';
import { NextAPI } from '@/service/middleware/entry';
import { OwnerPermissionVal } from '@fastgpt/global/support/permission/constant';
import { onDelOneApp } from '@fastgpt/service/core/app/controller';
import { addAuditLog } from '@fastgpt/service/support/user/audit/util';
import { pushTrack } from '@fastgpt/service/common/middle/tracks/utils';

async function handler(req: NextApiRequest, res: NextApiResponse<string[]>) {
  const { appId } = req.query as { appId: string };

  if (!appId) {
    return Promise.reject('参数错误');
  }

  // 1. 权限验证 - 需要所有者权限（文件夹所有者可删除文件夹内所有应用）
  const { teamId, tmbId, userId, app } = await authApp({
    req,
    authToken: true,
    appId,
    per: OwnerPermissionVal
  });

  // 2. 删除应用（内部会递归删除子应用、聊天记录、权限等）
  const deletedAppIds = await onDelOneApp({ teamId, appId });

  // 3. 记录审计日志（异步执行，不阻塞响应）
  (async () => {
    addAuditLog({
      tmbId,
      teamId,
      event: AuditEventEnum.DELETE_APP,
      params: {
        appName: app.name,
        appType: getI18nAppType(app.type)
      }
    });
  })();

  // 4. 统计追踪
  pushTrack.countAppNodes({ teamId, tmbId, uid: userId, appId });

  return deletedAppIds;
}

export default NextAPI(handler);
```

**关键点说明**：

1. **所有者权限**：删除应用需要 `OwnerPermissionVal`，只有应用所有者或团队 owner 可以删除
2. **递归删除**：`onDelOneApp` 内部会处理文件夹的递归删除逻辑
3. **返回值**：返回所有被删除的应用 ID 数组（包括子应用）
4. **审计日志**：异步记录删除操作，不影响接口响应速度

```typescript
/**
 * 递归查找应用及其所有子应用（深度优先）
 */
export async function findAppAndAllChildren({
  teamId,
  appId,
  fields
}: {
  teamId: string;
  appId: string;
  fields?: string;
}): Promise<AppSchema[]> {
  const find = async (id: string) => {
    const children = await MongoApp.find({ teamId, parentId: id }, fields).lean();
    let apps = children;
    for (const child of children) {
      const grandChildren = await find(child._id);
      apps = apps.concat(grandChildren);
    }
    return apps;
  };

  const [app, childApps] = await Promise.all([
    MongoApp.findById(appId, fields),
    find(appId)
  ]);

  if (!app) return Promise.reject('App not found');
  return [app, ...childApps];
}

 * 删除应用及其所有子应用

export const onDelOneApp = async ({
  teamId,
  appId,
  session
}: {
  teamId: string;
  appId: string;
  session?: ClientSession;
}): Promise<string[]> => {
  // 1. 递归查找所有子应用
  const apps = await findAppAndAllChildren({ teamId, appId, fields: '_id avatar' });

  // 2. 获取非文件夹应用 ID（用于返回）
  const deletedAppIds = apps
    .filter((app) => !AppFolderTypeList.includes(app.type))
    .map((app) => String(app._id));

  // 3. 移除评估任务
  const evalJobs = await MongoEvaluation.find({ appId: { $in: apps.map((a) => a._id) } }, '_id').lean();
  await Promise.all(evalJobs.map((job) => removeEvaluationJob(job._id)));

  // 4. 删除每个应用的关联数据
  for await (const app of apps) {
    const appId = String(app._id);

    // 删除聊天文件和记录（不在事务中）
    await deleteChatFiles({ appId });
    await MongoChatItemResponse.deleteMany({ appId });
    await MongoChatItem.deleteMany({ appId });
    await MongoChat.deleteMany({ appId });
    await getS3ChatSource().deleteChatFilesByPrefix({ appId });
  }

  // 5. 事务中删除应用相关记录
  const del = async (app: AppSchema, session: ClientSession) => {
    const appId = String(app._id);
    await MongoOutLink.deleteMany({ appId }).session(session);
    await MongoOpenApi.deleteMany({ appId }).session(session);
    await MongoAppVersion.deleteMany({ appId }).session(session);
    await MongoChatInputGuide.deleteMany({ appId }).session(session);
    await MongoChatFavouriteApp.deleteMany({ teamId, appId }).session(session);
    await MongoChatSetting.updateMany({ teamId }, { $pull: { quickAppIds: { id: appId } } }).session(session);
    await MongoResourcePermission.deleteMany({ resourceType: PerResourceTypeEnum.app, teamId, resourceId: appId }).session(session);
    await MongoAppLogKeys.deleteMany({ appId }).session(session);
    await MongoApp.deleteOne({ _id: appId }, { session });
    await removeImageByPath(app.avatar, session);
  };

  for await (const app of apps) {
    if (session) {
      await del(app, session);
    } else {
      await mongoSessionRun((session) => del(app, session));
    }
  }

  return deletedAppIds;
};
```

```typescript
import { MongoAppVersion } from '@fastgpt/service/core/app/version/schema';
import { MongoApp } from '@fastgpt/service/core/app/schema';
import { mongoSessionRun } from '@fastgpt/service/common/mongo/sessionRun';

export type PostPublishAppProps = {
  nodes: StoreNodeItemType[];
  edges: StoreEdgeItemType[];
  chatConfig: AppChatConfigType;
  isPublish?: boolean;       // 是否发布（否则只是保存）
  versionName?: string;      // 版本名称
  autoSave?: boolean;        // 是否自动保存
};

async function handler(req: ApiRequestProps<PostPublishAppProps>) {
  const { appId } = req.query;
  const { nodes, edges, chatConfig, isPublish, versionName, autoSave } = req.body;

  // 1. 权限验证
  const { app, tmbId, teamId } = await authApp({ appId, req, per: WritePermissionVal, authToken: true });

  // 2. 格式化节点数据
  beforeUpdateAppFormat({ nodes });

  // 3. 自动保存模式：更新或创建自动保存版本
  if (autoSave) {
    await mongoSessionRun(async (session) => {
      await MongoAppVersion.updateOne(
        { appId, isAutoSave: true },
        { tmbId, appId, nodes, edges, chatConfig, versionName: '自动保存', time: new Date() },
        { session, upsert: true }
      );
      await MongoApp.updateOne({ _id: appId }, { modules: nodes, edges, chatConfig, updateTime: new Date() }, { session });
    });
    return;
  }

  // 4. 创建新版本并更新应用
  await mongoSessionRun(async (session) => {
    // 创建版本记录
    const [{ _id }] = await MongoAppVersion.create([{
      appId, nodes, edges, chatConfig, isPublish, versionName, tmbId
    }], { session });

    // 更新应用数据
    await MongoApp.updateOne(
      { _id: appId },
      {
        modules: nodes,
        edges,
        chatConfig,
        updateTime: new Date(),
        version: 'v2',
        'pluginData.nodeVersion': _id,
        // 发布时更新定时触发器配置
        ...(isPublish && chatConfig?.scheduledTriggerConfig?.cronString
          ? { scheduledTriggerConfig: chatConfig.scheduledTriggerConfig, scheduledTriggerNextTime: getNextTime(...) }
          : { $unset: { scheduledTriggerConfig: '', scheduledTriggerNextTime: '' } })
      },
      { session }
    );
  });

  // 5. 记录审计日志
  addAuditLog({ tmbId, teamId, event: AuditEventEnum.UPDATE_PUBLISH_APP, ... });
}
```

**关键点说明**：

1. **自动保存**：`autoSave=true` 时使用 upsert 更新自动保存版本，不创建新版本记录
2. **版本与应用同步**：发布时同时更新 `MongoApp.modules` 和 `MongoAppVersion`
3. **nodeVersion 关联**：`pluginData.nodeVersion` 指向最新版本的 `_id`
4. **定时触发器**：只有发布时才会更新定时触发器配置

## 7. 权限管理

### 7.1 应用权限验证

**文件位置**: `packages/service/support/permission/app/auth.ts`

```typescript
import { AppPermission } from '@fastgpt/global/support/permission/app/controller';
import { getTmbPermission } from '../controller';
import { getTmbInfoByTmbId } from '../../user/team/controller';

/**
 * 通过团队成员 ID 验证应用权限
 */
export const authAppByTmbId = async ({
  tmbId,
  appId, 
  per,
  isRoot,
  authAppPublicPer
}: {
  tmbId: string;
  appId: string;
  per: PermissionValueType;
  isRoot?: boolean;
  authAppPublicPer?: boolean;
}): Promise<{ app: AppDetailType }> => {
  // 1. 获取团队成员信息
  const { teamId, permission: tmbPer } = await getTmbInfoByTmbId({ tmbId });

  // 2. 获取应用信息
  const app = await MongoApp.findOne({ _id: appId }).lean();
  if (!app) {
    return Promise.reject(AppErrEnum.unExist);
  }

  // 3. Root 用户拥有所有权限
  if (isRoot) {
    return { app: { ...app, permission: new AppPermission({ isOwner: true }) } };
  }

  // 4. 验证团队归属
  if (String(app.teamId) !== teamId) {
    return Promise.reject(AppErrEnum.unAuthApp);
  }

  // 5. hidden 类型应用的特殊处理
  if (app.type === AppTypeEnum.hidden) {
    if (per === AppReadChatLogPerVal && !tmbPer.hasManagePer) {
      return Promise.reject(AppErrEnum.unAuthApp);
    }
    if (per !== ReadPermissionVal) {
      return Promise.reject(AppErrEnum.unAuthApp);
    }
    return { app: { ...app, permission: new AppPermission({ isOwner: false, role: ReadRoleVal }) } };
  }

  // 6. 判断是否为所有者
  const isOwner = tmbPer.isOwner || String(app.tmbId) === String(tmbId);

  // 7. 获取权限（同时获取父文件夹权限和自身权限）
  const isGetParentClb = app.inheritPermission && !AppFolderTypeList.includes(app.type) && !!app.parentId;

  const [folderPer = NullRoleVal, myPer = NullRoleVal] = await Promise.all([
    isGetParentClb
      ? getTmbPermission({ teamId, tmbId, resourceId: app.parentId!, resourceType: PerResourceTypeEnum.app })
      : NullRoleVal,
    getTmbPermission({ teamId, tmbId, resourceId: appId, resourceType: PerResourceTypeEnum.app })
  ]);

  // 8. 合并权限（父文件夹权限 + 自身权限）
  const Per = new AppPermission({ role: sumPer(folderPer, myPer), isOwner });

  // 9. 收藏和快捷应用自动获得读权限
  if (app.favourite || app.quick) {
    Per.addRole(ReadRoleVal);
  }

  // 10. 权限检查（close_auth 模式下跳过）
  if (!global.systemEnv.close_auth && !Per.checkPer(per)) {
    return Promise.reject(AppErrEnum.unAuthApp);
  }

  return { app: { ...app, permission: Per } };
};

/**
 * 统一入口：解析请求头 + 验证应用权限
 */
export const authApp = async ({ appId, per, ...props }: AuthModeType & { appId: ParentIdType; per: PermissionValueType }) => {
  const result = await parseHeaderCert(props);  // 解析 token/apiKey
  const { tmbId } = result;

  if (!appId) {
    return Promise.reject(AppErrEnum.unExist);
  }

  const { app } = await authAppByTmbId({ tmbId, appId, per, isRoot: result.isRoot, authAppPublicPer: props.authAppPublicPer });

  return { ...result, permission: app.permission, app };
};
```

**关键点说明**：

1. **两层验证函数**：`authApp` 是统一入口（解析请求头），`authAppByTmbId` 是核心逻辑
2. **继承权限**：非文件夹且有父目录时，同时获取父文件夹权限并合并
3. **特殊类型处理**：`hidden` 类型应用只允许读权限或管理员查看日志
4. **收藏/快捷应用**：自动获得读权限
5. **close_auth 模式**：全局关闭权限验证时跳过检查

```typescript
export const getSystemToolsWithInstalled = async ({
  teamId,
  isRoot
}: {
  teamId: string;
  isRoot: boolean;
}) => {
  const [tools, { installedSet, uninstalledSet }] = await Promise.all([
    getSystemTools(),
    MongoTeamInstalledPlugin.find({ teamId, pluginType: 'tool' }, 'pluginId installed')
      .lean()
      .then((res) => {
        const installedSet = new Set<string>();
        const uninstalledSet = new Set<string>();
        res.forEach((item) => {
          if (item.installed) {
            installedSet.add(item.pluginId);
          } else {
            uninstalledSet.add(item.pluginId);
          }
        });
        return { installedSet, uninstalledSet };
      })
  ]);

  // 计算每个工具的安装状态
  return tools.map((tool) => {
    const installed = (() => {
      if (installedSet.has(tool.id)) return true;
      // Root 用户默认安装未明确卸载的工具
      if (isRoot && !uninstalledSet.has(tool.id)) return true;
      // 默认安装的工具
      if (tool.defaultInstalled && !uninstalledSet.has(tool.id)) return true;
      return false;
    })();
    return { ...tool, installed };
  });
};
```

### 10.7 工具 ID 解析

工具 ID 采用复合格式，通过 `splitCombineToolId` 函数解析：

**文件位置**: `packages/global/core/app/tool/utils.ts`

```typescript
// 工具 ID 格式：{source}-{pluginId}
// 例如：system-search_baidu, personal-648a1234..., mcp-648a1234.../toolName

export const splitCombineToolId = (id: string) => {
  const [source, ...rest] = id.split('-');
  return {
    source: source as AppToolSourceEnum,
    pluginId: rest.join('-')
  };
};

export const combineToolId = (source: AppToolSourceEnum, pluginId: string) => {
  return `${source}-${pluginId}`;
};
```

### 10.8 工具预览节点生成

当用户在工作流中添加工具节点时，需要生成预览节点数据：

**API**: `GET /api/core/app/tool/getPreviewNode`

**核心函数**: `getChildAppPreviewNode()`

```typescript
export async function getChildAppPreviewNode({
  appId,
  versionId,
  lang = 'en'
}: {
  appId: string;
  versionId?: string;
  lang?: localeType;
}): Promise<FlowNodeTemplateType> {
  const { source, pluginId } = splitCombineToolId(appId);

  // 根据来源获取工具数据
  const app = await (async () => {
    switch (source) {
      case AppToolSourceEnum.personal:
        // 从 apps 集合获取团队工具
        return getPersonalTool(pluginId, versionId);
      case AppToolSourceEnum.mcp:
        // 获取 MCP 工具
        return getMCPTool(pluginId);
      case AppToolSourceEnum.http:
        // 获取 HTTP 工具
        return getHTTPTool(pluginId);
      case AppToolSourceEnum.systemTool:
      default:
        // 从系统工具缓存获取
        return getSystemToolByIdAndVersionId(pluginId, versionId);
    }
  })();

  // 转换为 FlowNodeTemplateType
  return formatToFlowNode(app);
}
```

```typescript
export async function getChildAppRuntimeById({
  id,
  versionId,
  lang = 'en'

  id: string;
  versionId?: string;
  lang?: localeType;
}): Promise<AppToolRuntimeType> {

    // 团队工具：获取应用数据和版本

      tmbId: String(item.tmbId),
      nodes: version.nodes,
      edges: version.edges,
      // ...
    };
  } else {
    // 系统工具：从缓存和外部服务获取
    return getSystemToolByIdAndVersionId(pluginId, versionId);
  }
}
```

### 11.8 关键设计要点

| 设计点 | 说明 |
|--------|------|
| **shareId 唯一性** | 使用 24 位 nanoid 生成，确保不可猜测 |

| **限流机制** | QPM（每分钟请求数）限制，防止滥用 |
| **积分控制** | maxUsagePoints 限制单个渠道的最大消耗 |
| **身份验证** | hookUrl 支持外部系统验证用户身份 |
| **商业功能** | 第三方 IM 集成为商业版功能，消息处理在 Pro 服务 |
| **每应用限制** | 分享链接每个应用最多创建 10 个 |

### 12.9 关键设计决策

| 设计点 | 决策 | 原因 |
|--------|------|------|
| 类型分离 | `AppSchema` / `AppListItemType` / `AppDetailType` | 列表页性能优化，避免传输 MB 级工作流数据 |
| 权限继承 | `inheritPermission` + `parentId` | 文件夹层级管理，简化批量权限配置 |
| 版本管理 | 独立 `AppVersion` 表 | 支持版本回滚、自动保存、发布历史 |
| 工作流存储 | `modules[]` + `edges[]` | 兼容 React Flow，支持复杂拓扑结构 |
| 工具 ID 格式 | `source-pluginId` | 统一处理系统工具、MCP、HTTP、个人应用 |
| 删除策略 | 递归删除 + 事务 | 保证数据一致性，避免孤儿记录 |

## 13. 常见问题解答 (FAQ)

### 13.1 AppTypeEnum.agent 是什么应用类型？

`agent = 'agent'` 是一个**预留但目前未使用**的应用类型。

在 `AppTypeList` 中虽然包含了三种类型：

```typescript
export const AppTypeList = [AppTypeEnum.simple, AppTypeEnum.agent, AppTypeEnum.workflow];
```

但在实际的 UI 创建界面中：

- **对话 Agent** → 对应 `AppTypeEnum.simple`
- **工作流** → 对应 `AppTypeEnum.workflow`

`agent` 类型在 `appTypeTagMap` 中设为 `undefined`，说明它没有对应的标签/图标配置，是为未来功能预留的类型。

### 13.2 pluginData 字段的作用是什么？

`pluginData` 是**工具/插件类型应用**的专用配置数据：

| 字段 | 用途 |
|------|------|
| `nodeVersion` | 插件节点的版本 ID，关联到 `AppVersion` 表的 `_id` |
| `pluginUniId` | 插件的唯一标识符（插件名称） |
| `apiSchemaStr` | **HTTP 插件专用** - OpenAPI/Swagger 规范的 JSON 字符串 |
| `customHeaders` | **HTTP 插件专用** - 自定义 HTTP 请求头配置 |

主要用于 `workflowTool`、`httpToolSet`、`mcpToolSet` 等工具类型应用。

### 13.3 nodeVersion 是随机生成的还是有关联？

`nodeVersion` **不是随机创建的**，它是 `AppVersion` 集合中某条版本记录的 `_id`。

**关联关系**：

```text
MongoAppVersion (版本记录表)
  └── _id: ObjectId  ←──────────────────┐
                                        │
MongoApp (应用表)                        │
  └── pluginData.nodeVersion ───────────┘
```

**生成流程** (在 `publish.ts` 中)：

```typescript
// 1. 创建版本记录
const [{ _id }] = await MongoAppVersion.create([{ appId, nodes, edges, ... }]);

// 2. 更新应用的 nodeVersion
await MongoApp.updateOne({ _id: appId }, { 'pluginData.nodeVersion': _id });
```

### 13.4 inheritPermission 在没有 parentId 时如何理解？

`inheritPermission` 的生效条件（来自 `auth.ts`）：

```typescript
const isGetParentClb =
  app.inheritPermission && !AppFolderTypeList.includes(app.type) && !!app.parentId;
```

**只有同时满足以下三个条件时，才会获取父级权限**：

1. `inheritPermission === true`
2. 应用不是文件夹类型
3. `parentId` 存在

| 场景 | `parentId` | `inheritPermission` | 权限来源 |
|------|------------|---------------------|----------|
| 根目录应用 | `null` | `true` | 仅自身权限（字段无效） |
| 根目录应用 | `null` | `false` | 仅自身权限（字段无效） |
| 文件夹内应用 | 有值 | `true` | 自身权限 + 父级权限 |
| 文件夹内应用 | 有值 | `false` | 仅自身权限（独立配置） |

**对于根目录的应用**：`inheritPermission` 字段存在但无实际作用，默认值 `true` 是为了保持一致性，如果将来该应用被移动到某个文件夹内，会自动生效。

### 13.5 为什么有 AppListItemType 和 AppDetailType，不统一用 AppSchema？

这是典型的 **DTO (Data Transfer Object)** 模式设计：

| 类型 | 角色 | 说明 |
|------|------|------|
| `AppSchema` | Entity | 数据库实体，完整字段 |
| `AppListItemType` | DTO | 列表展示，精简 + 计算字段 |
| `AppDetailType` | DTO | 详情展示，完整 + 计算字段 |

**设计原因**：

1. **数据量差异**：
   - `AppSchema` 包含 `modules`（节点数组）和 `edges`（边数组），可能有 MB 级数据
   - `AppListItemType` 只需要展示字段，KB 级数据

2. **字段来源不同**：
   - `AppSchema` 纯数据库字段
   - `AppListItemType` 包含计算字段如 `permission`、`sourceMember`

3. **性能优化**：列表页加载 100 个应用时，用精简类型可秒开

### 13.6 schema.ts 文件的作用是什么？

`packages/service/core/app/schema.ts` 是 **Mongoose Schema 定义文件**，职责包括：

- **定义集合名称**：`export const AppCollectionName = 'apps'`
- **定义字段类型和约束**：`name: { type: String, required: true }`
- **定义字段关联 (ref)**：`parentId: { type: Schema.Types.ObjectId, ref: 'apps' }`
- **定义索引**：`AppSchema.index({ teamId: 1, updateTime: -1 })`
- **导出 Model**：`export const MongoApp = getMongoModel<AppType>(AppCollectionName, AppSchema)`

**与 type.d.ts 的关系**：

- `type.d.ts`：TypeScript 类型定义（编译时类型检查）
- `schema.ts`：Mongoose Schema（运行时数据库约束）
- 两者需要**手动保持同步**
