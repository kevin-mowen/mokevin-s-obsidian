# 错误码手册（/api/v1/kb/*）

本文只覆盖 12 个对外 OpenAPI 接口（`/api/v1/kb/*`）。响应统一走 `PWResponse` 结构：

```json
{
  "success": false,
  "code": 40001,
  "message": "access_credential 不能为空"
}
```

- `code == 0` 且 `success == true` 才表示成功
- `code > 0` 的含义如下表

## 错误码总表

| HTTP | code | 类别 | 代表性 message | 调用方建议动作 |
|:----:|:----:|------|----------------|---------------|
| 401 | 401 | 鉴权失败 | `Unauthorized` | **不要重试**。检查 `Authorization` 头、确认是**系统级** Token（KB 级 Token 对外接口不可用）。联系管理员核对 token 是否已作废 |
| 403 | 403 | 权限不足 | `权限不足，需要完全控制或文档管理权限` | **不要重试**。当前 token 的 permission 等级不够，要么让管理员升权，要么换一个权限更高的 token |
| 200 | 40001 | 参数错误 / 解密失败 | `access_credential 不能为空`、`system_id 不能为空`、`query 不能为空`、`doc_id 不能为空`、`请求参数无效`、以及 `access_credential` AES-GCM 解密失败 | **不要重试**。先排查必填字段；如果确认都传了，八成是 secret 配错 / Base64 编码不对 / nonce 长度不对。见 [credential-encrypt.md](./credential-encrypt.md#常见坑) |
| 200 | 40001 | 业务规则 | `doc_name 不能超过 N 个字符`、`文件夹嵌套层级不能超过 N 级`、`不支持的文件类型`、`doc_ids 最多支持 20 个` | **不要重试**。调用方侧做输入校验，把错误提示给最终用户 |
| 200 | 40003 | 用户 / 资源不存在或无权访问 | `用户不存在`、`无权访问该知识库`、`无权访问该文档`、`无权访问目标文件夹`、`非管理员用户不能创建文件夹` | **不要重试**。常见子情况：<br>• `用户不存在` → notes_id 没通过 CAS 同步<br>• `无权访问 XX` → 该用户在 PandaWiki 没有对应 KB/Node 的访问权限<br>• 管理员限定动作失败 → 换一个有 admin 权限的用户 |
| 200 | 40004 | 资源未找到 | `Not Found` 或具体资源文案 | **不要重试**。检查传入的 ID 是否正确 |
| 200 | 50001 | 服务端异常 | `获取知识库权限失败`、`查询用户关联信息失败`、`文件处理超时`、`文件解析结果为空`、`查询文件夹失败` | **可重试**。建议**指数退避**重试 2~3 次；连续失败报告运维排查 PandaWiki 日志 |

## 重要行为约定

1. **HTTP 状态码总体返回 200**（除 401 / 403 外），业务语义通过 `code` 字段区分。调用方**不要只看 HTTP 状态**，必须解析 body 里的 `success` 和 `code`。
2. **流式接口**（`/chat` 在 `stream=true` 时）错误通过 SSE `event: error` 推送，payload 仍然是 `{code, message}`，之后不再有事件。
3. `40001` 里**解密失败**和**参数缺失**共用同一个 code，通过 `message` 区分：
   - `access_credential 不能为空` → 没传
   - 其他解密相关（如 `invalid base64`、`decryption failed`）→ secret / 编码 / nonce 出问题

## 重试策略建议

| 错误类别 | 重试？ | 退避 | 最大次数 |
|----------|-------|------|---------|
| 401 / 403 | ❌ 不重试 | — | — |
| 40001 / 40003 / 40004 | ❌ 不重试 | — | — |
| 50001 | ✅ 重试 | 指数退避：1s / 2s / 4s | 3 |
| 网络层错误（超时、连接拒绝）| ✅ 重试 | 指数退避：1s / 2s / 4s | 3 |
| `/ping` 返回失败 | — | — | 报警到运维 |

## 排障流程图

```
接口失败？
  ├─ HTTP 401 → 检查 Authorization 头与 token 有效性
  ├─ HTTP 403 → 联系管理员升权或换 token
  ├─ code 40001
  │    ├─ message 含 "不能为空" → 补齐必填字段
  │    ├─ message 含解密关键字  → 核对 secret、检查 Base64/编码
  │    └─ 其他业务规则           → 修正输入
  ├─ code 40003
  │    ├─ "用户不存在"     → 检查 notes_id、确认 CAS 已同步
  │    └─ "无权访问 XX"    → 管理员给该用户对应 KB/Node 权限
  ├─ code 50001 → 先重试 3 次，仍失败报告运维
  └─ 网络层超时/拒绝 → 先 curl /api/v1/kb/ping，区分是网络还是服务挂了
```
