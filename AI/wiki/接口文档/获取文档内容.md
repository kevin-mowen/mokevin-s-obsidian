# 获取文档内容

## 功能描述

根据文档 ID 获取用户有权限访问的已发布文档内容。支持按字符数分片返回，防止超大文档撑爆 LLM 上下文。

## API 接口

| 方法 | 路径 |
|------|------|
| POST | `/api/v1/kb/doc/detail` |

## 请求头

| Header | 必填 | 说明 |
|--------|------|------|
| Authorization | 是 | `Bearer <system_api_token>`，仅支持系统级 API Token，权限需为完全控制或文档管理。Token 需找管理员获取 |
| Content-Type | 是 | `application/json` |

## 请求参数

| 参数                | 类型     | 必填  | 说明                                                               | 示例                |
| ----------------- | ------ | --- | ---------------------------------------------------------------- | ----------------- |
| system_id         | string | 是   | 调用方系统编号                                                          | `"fastgpt"`       |
| agent_id          | string | 否   | 智能体 ID                                                           | `"bot_12345"`     |
| access_credential | string | 是   | 使用 notes_id + Secret 经 AES-256-GCM 加密后的 Base64 密文，Secret 需找管理员获取 | `"nGhJ8kL2mN..."` |
| doc_id            | string | 是   | 文档 ID                                                            | `"019b6e86-..."`  |
| format            | string | 否   | 返回格式：raw / md / html（默认 html）                                    | `"html"`          |
| max_length        | int    | 否   | content 单次返回的最大字符数（rune），默认 30000，上限 50000                       | `30000`           |
| offset            | int    | 否   | content 的起始字符偏移量（rune），用于分片续读，默认 0                               | `0`               |
| mode              | string | 否   | `full`（默认，按 max_length/offset 截断）或 `summary`（仅返回摘要/前 2000 字预览）   | `"full"`          |

## 调用示例

```bash
# 一次性获取（默认上限 30000 字符）
curl -s -X POST http://localhost:18000/api/v1/kb/doc/detail \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <system_api_token>" \
  -d '{
    "system_id": "fastgpt",
    "access_credential": "<AES-256-GCM 加密后的用户工号（base64）>",
    "doc_id": "019d959c-5b21-78fc-92f8-3bcf574586fb",
    "format": "html"
  }' | jq

# 大文档续读：先取第一片
curl -s -X POST http://localhost:18000/api/v1/kb/doc/detail \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <system_api_token>" \
  -d '{
    "system_id": "fastgpt",
    "access_credential": "<...>",
    "doc_id": "019b6e86-...",
    "max_length": 30000,
    "offset": 0
  }' | jq
# 若响应 truncated=true，用返回的 next_offset 继续读

# 只要摘要
curl -s -X POST http://localhost:18000/api/v1/kb/doc/detail \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <system_api_token>" \
  -d '{
    "system_id": "fastgpt",
    "access_credential": "<...>",
    "doc_id": "019b6e86-...",
    "mode": "summary"
  }' | jq
```

## 成功响应

```json
{
  "success": true,
  "code": 0,
  "message": "success",
  "data": {
    "doc_id": "019b6e86-...",
    "doc_name": "数据库配置指南",
    "content": "<h1>数据库配置指南</h1><p>...</p>",
    "content_type": "md",
    "summary": "本文档介绍了数据库连接配置的方法...",
    "kb_id": "3adce2bd-...",
    "kb_name": "技术文档库",
    "content_length": 45230,
    "returned_length": 30000,
    "truncated": true,
    "next_offset": 30000,
    "mode": "full"
  }
}
```

## 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| data.doc_id | string | 文档 ID |
| data.doc_name | string | 文档标题 |
| data.content | string | 文档内容（格式由 format 参数决定，可能被 max_length/offset 截断） |
| data.content_type | string | 内容原始类型：md / html / excel |
| data.summary | string | 文档摘要 |
| data.kb_id | string | 所属知识库 ID |
| data.kb_name | string | 所属知识库名称 |
| data.content_length | int | content 字段原始总 rune 长度 |
| data.returned_length | int | 本次返回 content 的 rune 长度 |
| data.truncated | bool | content 是否被截断 |
| data.next_offset | int | 下次续读的 offset，-1 表示已读完 |
| data.mode | string | 实际生效的 mode：`full` 或 `summary` |

## 截断与续读规则

- **max_length**：单次返回最大字符数（按 rune 计），默认 30000，上限 50000
- **offset**：续读起点，传入 `next_offset` 即可续读下一片；`next_offset=-1` 表示已读完
- **mode=summary**：优先返回数据库已有的 summary 字段；若为空则返回 content 的前 2000 rune 作为预览。此模式下 `next_offset` 固定为 -1（不支持续读，需改 `mode=full` 才能分片读全文）
- **token 预算参考**：默认 30000 rune 约 ≈ 20k tokens，符合 64k LLM 上下文安全使用

## 权限校验

1. **系统 Token 认证**：仅接受系统级 API Token，权限需为 full_control 或 doc_manage
2. **用户身份解密**：使用 Secret 解密 access_credential
3. **知识库范围**：校验文档所属 KB 在 Token 授权范围内
4. **知识库访问权限**：校验用户对该知识库的访问权限
5. **文档权限**：文档自身 + 父文件夹 visitable 权限双层校验

## 错误码

| 错误码 | 说明 |
|--------|------|
| 401 | 非系统级 API Token 或 Token 无效 |
| 403 | Token 权限不足（需要完全控制或文档管理） |
| 40001 | 必填参数为空或 access_credential 解密失败 |
| 40003 | 用户不存在、无权访问知识库或文档、文档不存在或未发布 |
