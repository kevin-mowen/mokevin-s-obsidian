# 编辑文档内容

## 功能描述

编辑用户有权限访问的文档内容，保存为草稿。如需发布，请使用 `/api/v1/kb/doc/publish` 接口。

## API 接口

| 方法 | 路径 |
|------|------|
| POST | `/api/v1/kb/doc/edit` |

## 请求头

| Header        | 必填  | 说明                                                         |
| ------------- | --- | ---------------------------------------------------------- |
| Authorization | 是   | `Bearer <system_api_token>`，仅支持系统级 API Token，权限需为完全控制或文档管理。Token 需找管理员获取 |
| Content-Type  | 是   | `application/json`                                         |

## 请求参数

| 参数                | 类型     | 必填  | 说明                                   | 示例                  |
| ----------------- | ------ | --- | ------------------------------------ | ------------------- |
| system_id         | string | 是   | 调用方系统编号                              | `"fastgpt"`         |
| agent_id          | string | 否   | 智能体 ID                               | `"bot_12345"`       |
| access_credential | string | 是   | 使用 notes_id + Secret 经 AES-256-GCM 加密后的 Base64 密文，Secret 需找管理员获取 | `"nGhJ8kL2mN..."`   |
| doc_id            | string | 是   | 文档 ID                                | `"019b6e86-..."`    |
| content           | string | 否   | 替换后的文档内容（content、name、summary 至少传一个） | `"# 新标题\n\n新内容..."` |
| content_type      | string | 否   | 内容格式：md / html，传 content 时建议一并指定     | `"md"`              |
| name              | string | 否   | 文档标题                                 | `"数据库配置指南"`         |
| summary           | string | 否   | 文档摘要                                 | `"介绍数据库配置方法"`       |

## 调用示例

```bash
curl -s -X POST http://localhost:18000/api/v1/kb/doc/edit \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <system_api_token>" \
  -d '{
    "system_id": "fastgpt",
    "access_credential": "<AES-256-GCM加密后的Base64密文>",
    "doc_id": "019b6e86-xxxx-xxxx-xxxx-xxxxxxxxxxxx",
    "content": "# 更新后的内容\n\n这是修改后的文档内容...",
    "content_type": "md"
  }' | jq
```

## 成功响应

```json
{
  "success": true,
  "code": 0,
  "message": "success",
  "data": {
    "doc_id": "019b6e86-...",
    "status": "draft",
    "message": "文档已保存为草稿"
  }
}
```

## 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| data.doc_id | string | 文档 ID |
| data.status | string | 固定为 draft |
| data.message | string | 状态说明 |

## 权限校验

1. **系统 Token 认证**：仅接受系统级 API Token，权限需为 full_control 或 doc_manage
2. **用户身份解密**：使用 Secret 解密 access_credential
3. **知识库范围**：校验文档所属 KB 在 Token 授权范围内
4. **知识库访问权限**：校验用户对该知识库的访问权限
5. **文档权限**：文档自身 + 父文件夹 visitable 权限双层校验

## 错误码

| 错误码 | 说明 |
|--------|------|
| 401 | 非系统级 API Token 或 Token 无效 |
| 403 | Token 权限不足（需要完全控制或文档管理） |
| 40001 | 必填参数为空、access_credential 解密失败或未传任何可修改字段 |
| 40003 | 用户不存在、无权访问知识库或文档、文档不存在 |
| 50001 | 更新文档失败、发布失败或提交贡献失败 |

## 注意事项

- content_type 为 md 时，系统会自动将 Markdown 转为 HTML 后保存（前端编辑器使用 HTML 格式）
- Markdown 内容中，标题（# / ## 等）前面需要空行（`\n\n`），单个 `\n` 在 Markdown 中是软换行，不会产生段落分隔
- 不传 content_type 时保持原文档格式不变
