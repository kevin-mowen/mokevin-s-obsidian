# 删除文件夹（仅管理员）

## 接口信息

| 项目               | 值                                          |
| ---------------- | ------------------------------------------ |
| **路径**           | `POST /api/v1/kb/folder/delete`            |
| **认证**           | `Authorization: Bearer <system_api_token>` |
| **Content-Type** | `application/json`                         |
| **权限要求**         | 系统级 API Token + **知识库 full_control 管理员**   |

## 功能说明

删除指定文件夹节点。非空文件夹必须显式传 `recursive=true` 才会递归删除所有子节点（含子文件夹和文档），避免误删整棵子树。**仅知识库 full_control 管理员可调用。**

## 请求参数

| 参数                | 类型     | 必填 | 说明                                       |
| ----------------- | ------ | -- | ---------------------------------------- |
| system_id         | string | 是  | 调用方系统标识                                  |
| agent_id          | string | 否  | 智能体 ID                                   |
| access_credential | string | 是  | AES-256-GCM 加密后的用户工号（base64）             |
| kb_id             | string | 是  | 知识库 ID                                   |
| folder_id         | string | 是  | 要删除的文件夹节点 ID（type 必须是 folder）           |
| recursive         | bool   | 否  | 非空文件夹必须显式传 `true` 才允许递归删除，默认 `false`     |

### recursive 行为

- 文件夹为空时，`recursive` 传不传都可以删除
- 文件夹非空（存在直接子节点）且 `recursive=false` → 返回 40001，不执行删除
- 文件夹非空且 `recursive=true` → 递归删除自身及所有子节点

## curl 示例

### 删除空文件夹

```bash
curl -X POST 'http://dev.localhost:18000/api/v1/kb/folder/delete' \
  -H 'Authorization: Bearer <system_api_token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "system_id": "obsidian",
    "access_credential": "AES-256-GCM 加密后的用户工号（base64）",
    "kb_id": "be8666e3-7823-4e20-9f1c-f77ac604b47a",
    "folder_id": "019db082-2629-7115-b066-2e92f6036485"
  }'
```

### 递归删除非空文件夹

```bash
curl -X POST 'http://dev.localhost:18000/api/v1/kb/folder/delete' \
  -H 'Authorization: Bearer <system_api_token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "system_id": "obsidian",
    "access_credential": "AES-256-GCM 加密后的用户工号（base64）",
    "kb_id": "be8666e3-7823-4e20-9f1c-f77ac604b47a",
    "folder_id": "019db082-2629-7115-b066-2e92f6036485",
    "recursive": true
  }'
```

## 响应示例

### 删除成功

```json
{
  "success": true,
  "code": 0,
  "message": "success",
  "data": {
    "folder_id": "019db082-2629-7115-b066-2e92f6036485",
    "folder_name": "2 AI ECN",
    "deleted_count": 8
  }
}
```

### 非空文件夹未传 recursive

```json
{
  "success": false,
  "code": 40001,
  "message": "文件夹非空（直接子节点 7 个），如需删除请设置 recursive=true"
}
```

### 文件夹不存在 / 跨 KB

```json
{ "success": false, "code": 40003, "message": "文件夹不存在" }
```

```json
{ "success": false, "code": 40003, "message": "文件夹不属于该知识库" }
```

### 传入文档 id

```json
{ "success": false, "code": 40001, "message": "该节点是文档，请使用 /doc/delete" }
```

## 响应字段

| 字段            | 类型     | 说明                                             |
| ------------- | ------ | ---------------------------------------------- |
| folder_id     | string | 被删除的文件夹 id（即请求传入值）                             |
| folder_name   | string | 被删除的文件夹名称（删除前快照）                               |
| deleted_count | int    | 本次删除的节点总数（含文件夹自身和所有子节点）<br>= `直接子节点数量 + 1`   |

> `deleted_count` 基于**直接子节点**计数 + 1。如果子文件夹里还有更深层节点，底层 NodeAction 会一并删除，但该字段不反映深层数量。需要准确总数时请调用前先查目录树。

## 权限检查流程

1. **系统级 Token 校验** — 必须使用系统级 API Token（KB 级别 Token 不可用）
2. **基础参数校验** — system_id / access_credential / kb_id / folder_id 都不能为空
3. **access_credential 解密** — 用 Token 的 secret 做 AES-256-GCM 解密得到 notes_id
4. **用户身份解析** — 通过 notes_id 查 authID，不存在返回 40003
5. **知识库授权** — kb_id 必须在 Token 授权的知识库范围内
6. **管理员严格校验** — notes_id 对应的 admin user 在 `kb_users` 中必须是 `full_control`
7. **节点校验** — 节点必须存在 / 属于本 KB / 类型为 folder
8. **非空确认** — 非空且未传 `recursive=true` 直接拒绝，不执行删除

## 错误码

| code  | 说明                                                    |
| ----- | ----------------------------------------------------- |
| 40001 | 参数错误（缺字段 / 解密失败 / 传入了文档节点 / 非空未显式 recursive）        |
| 40003 | 权限不足（用户不存在 / 无 KB 权限 / 非 full_control / 文件夹不存在 / 跨 KB）|
| 50001 | 查询子节点失败                                               |

其它删除阶段异常统一走 `NewResponseWithError`。

## 注意事项

- **`recursive` 是整棵子树开关**：一旦传 `true`，folder_id 下所有层级子节点（文件夹 + 文档）全部被删除，无法恢复；客户端应在执行前做好二次确认或备份。
- **和 `/doc/delete` 的类型隔离**：文件夹节点走这里，文档节点走 `/doc/delete`。传错类型会被拦，不会误删。
- **权限高于创建**：创建文件夹允许 `doc_manage`，删除只允许 `full_control`。
- **`deleted_count` 仅精确到直接子节点 + 1**：需要精确统计深层影响，调用前请先用 `/api/v1/kb/folders` 查目录树估算。
- **KB 内节点名可重名**：folder_name 仅作为删除前快照返回，无法作为唯一标识使用。
