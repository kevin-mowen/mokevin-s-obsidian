# 获取文件夹文档列表

## 功能描述

获取指定知识库中某个文件夹下的文档列表，支持分页。不传 folder_id 则返回根目录下的文档。

## API 接口

| 方法 | 路径 |
|------|------|
| POST | `/api/v1/kb/folder/docs` |

## 请求头

| Header | 必填 | 说明 |
|--------|------|------|
| Authorization | 是 | `Bearer <system_api_token>`，仅支持系统级 API Token（在系统配置 → 系统 API Token 中创建），KB 级别 Token 不可用。Token 需找管理员获取 |
| Content-Type | 是 | `application/json` |

## 请求参数

| 参数 | 类型 | 必填 | 说明 | 示例 |
|------|------|------|------|------|
| system_id | string | 是 | 调用方系统编号 | `"fastgpt"` |
| agent_id | string | 否 | 智能体 ID | `"bot_12345"` |
| access_credential | string | 是 | 使用 notes_id + Secret 经 AES-256-GCM 加密后的 Base64 密文，Secret 需找管理员获取 | `"nGhJ8kL2mN..."` |
| kb_id | string | 是 | 知识库 ID | `"be8666e3-..."` |
| folder_id | string | 否 | 文件夹 ID（从 `/api/v1/kb/folders` 获取），不传返回根目录文档 | `"aaa-bbb-ccc"` |
| page | int | 否 | 页码，默认 1 | `1` |
| page_size | int | 否 | 每页数量，默认 20，最大 50（原 100 已下调） | `20` |

## 调用示例

```bash
curl -s -X POST http://localhost:18000/api/v1/kb/folder/docs \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <system_api_token>" \
  -d '{
    "system_id": "fastgpt",
    "access_credential": "<AES-256-GCM加密后的Base64密文>",
    "kb_id": "be8666e3-7**********",
    "folder_id": "019b48ef-******",
    "page": 1,
    "page_size": 20
  }' | jq
```

## 成功响应

```json
{
  "success": true,
  "code": 0,
  "message": "success",
  "data": {
    "total": 156,
    "page": 1,
    "page_size": 20,
    "has_more": true,
    "items": [
      {
        "doc_id": "019b6e86-...",
        "doc_name": "数据库配置指南",
        "summary": "本文档介绍了...",
        "summary_truncated": false
      }
    ]
  }
}
```

## 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| data.total | number | 该文件夹下文档总数 |
| data.page | number | 当前页码 |
| data.page_size | number | 每页数量 |
| data.has_more | boolean | 是否还有下一页 |
| data.items[].doc_id | string | 文档 ID，可用于搜索接口的 node_ids 参数 |
| data.items[].doc_name | string | 文档名称 |
| data.items[].summary | string | 文档摘要（超 200 rune 会被截断） |
| data.items[].summary_truncated | bool | summary 是否被截断 |

## 限制规则

- `page_size` 上限 50（传入更大值会静默钳制到 50）
- 每条 `summary` 超 200 rune 截断
- 获取完整内容：通过 `doc_id` 调 `/api/v1/kb/doc/detail` 接口

## 权限过滤

返回的文档已经过 visitable 权限过滤，closed/partial（用户无权）的文档不会出现在列表中。

## 错误码

| 错误码 | 说明 |
|--------|------|
| 401 | 非系统级 API Token 或 Token 无效 |
| 40001 | system_id、access_credential 或 kb_id 为空，或解密失败 |
| 40003 | 用户不存在或无权访问该知识库 |
