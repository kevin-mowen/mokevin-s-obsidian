# 批量发布文档

## 功能描述

批量发布指定知识库下的文档，自动发布文档所在的父文件夹。管理员可直接发布，非管理员发布时提交到贡献列表待审核。

## API 接口

| 方法 | 路径 |
|------|------|
| POST | `/api/v1/kb/doc/publish` |

## 请求头

| Header        | 必填  | 说明                                                         |
| ------------- | --- | ---------------------------------------------------------- |
| Authorization | 是   | `Bearer <system_api_token>`，仅支持系统级 API Token，权限需为完全控制或文档管理。Token 需找管理员获取 |
| Content-Type  | 是   | `application/json`                                         |

## 请求参数

| 参数                | 类型       | 必填  | 说明                                   | 示例                  |
| ----------------- | -------- | --- | ------------------------------------ | ------------------- |
| system_id         | string   | 是   | 调用方系统编号                              | `"fastgpt"`         |
| agent_id          | string   | 否   | 智能体 ID                               | `"bot_12345"`       |
| access_credential | string   | 是   | 使用 notes_id + Secret 经 AES-256-GCM 加密后的 Base64 密文，Secret 需找管理员获取 | `"nGhJ8kL2mN.."`   |
| kb_id             | string   | 是   | 知识库 ID                               | `"be8666e3-..."`    |
| doc_ids           | []string | 是   | 要发布的文档 ID 列表，最多 20 个，所有文档必须属于同一知识库   | `["019b6e86-...", "019b6e87-..."]` |
| message           | string   | 否   | 发布说明，默认 "OpenAPI 批量发布"               | `"v1.2 文档更新"`      |

## 调用示例

```bash
curl -s -X POST http://localhost:18000/api/v1/kb/doc/publish \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <system_api_token>" \
  -d '{
    "system_id": "fastgpt",
    "access_credential": "<AES-256-GCM加密后的Base64密文>",
    "kb_id": "be8666e3-7*********",
    "doc_ids": [
      "019d941b-7d10-*******",
      "019d959c-f044-*******"
    ],
    "message": "v1.2 文档更新"
  }' | jq
```

## 成功响应

```json
{
  "success": true,
  "code": 0,
  "message": "success",
  "data": {
    "status": "published",
    "message": "成功发布 2 篇文档",
    "published": ["019b6e86-...", "019b6e87-..."],
    "failed": []
  }
}
```

## 部分失败响应

```json
{
  "success": true,
  "code": 0,
  "message": "success",
  "data": {
    "status": "published",
    "message": "成功发布 1 篇文档",
    "published": ["019b6e86-..."],
    "failed": [
      {"doc_id": "019b6e87-...", "reason": "文档不存在"}
    ]
  }
}
```

## 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| data.status | string | 状态：published / pending_review |
| data.message | string | 状态说明 |
| data.published | []string | 成功发布（或提交审核）的文档 ID |
| data.failed | []object | 失败的文档列表 |
| data.failed[].doc_id | string | 失败的文档 ID |
| data.failed[].reason | string | 失败原因 |

## 发布行为

| 用户类型 | 行为 |
|---------|------|
| 管理员（user_auths 有关联） | 直接发布（含 RAG 学习），自动发布父文件夹，status=published |
| 非管理员（无关联） | 逐个提交贡献待审核，status=pending_review。**文档不会直接发布，需要知识库管理员在管理后台"贡献列表"中审核通过后才会正式发布** |

## 权限校验

1. **系统 Token 认证**：仅接受系统级 API Token，权限需为 full_control 或 doc_manage
2. **用户身份解密**：使用 Secret 解密 access_credential
3. **知识库范围**：校验 kb_id 在 Token 授权范围内
4. **知识库访问权限**：校验用户对该知识库的访问权限
5. **文档归属校验**：校验所有 doc_ids 属于指定 kb_id
6. **文档权限**：文档自身 + 父文件夹 visitable 权限双层校验

## 错误码

| 错误码 | 说明 |
|--------|------|
| 401 | 非系统级 API Token 或 Token 无效 |
| 403 | Token 权限不足（需要完全控制或文档管理） |
| 40001 | 必填参数为空、access_credential 解密失败或 doc_ids 超过 20 个 |
| 40003 | 用户不存在、无权访问知识库、没有可发布的文档 |
| 50001 | 查询文档信息失败、发布失败 |

## 注意事项

- doc_ids 最多支持 20 个，超过请分批调用
- 所有 doc_ids 必须属于同一个知识库（kb_id）
- 发布时自动将文档的父文件夹一并发布，无需单独处理
- 不存在、不属于该知识库、或无权限的文档会记入 failed 列表，不影响其他文档发布
- 编辑文档（`/doc/edit`）和上传文档（`/upload`）不再支持发布，需使用本接口
