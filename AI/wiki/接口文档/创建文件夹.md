# 创建文件夹（仅管理员）

## 接口信息

| 项目               | 值                                          |
| ---------------- | ------------------------------------------ |
| **路径**           | `POST /api/v1/kb/folder/create`            |
| **认证**           | `Authorization: Bearer <system_api_token>` |
| **Content-Type** | `application/json`                         |
| **权限要求**         | 系统级 API Token + **知识库管理员**（非管理员不可调用）       |

## 功能说明

支持路径式创建文件夹（类似 `mkdir -p`）。传入以 `/` 分隔的路径，系统逐级处理：已存在的文件夹直接复用，不存在的自动创建。**仅知识库管理员可调用此接口，非管理员用户调用会返回 40003 权限错误。**

## 请求参数

| 参数                | 类型     | 必填  | 说明                                 |
| ----------------- | ------ | --- | ---------------------------------- |
| system_id         | string | 是   | 调用方系统标识                            |
| agent_id          | string | 否   | 智能体 ID                             |
| access_credential | string | 是   | AES-256-GCM 加密后的用户工号（base64）       |
| kb_id             | string | 是   | 知识库 ID                             |
| path              | string | 是   | 文件夹路径，用 `/` 分隔层级，如 `"技术文档/后端/API"` |

### path 规则

- 用 `/` 分隔多级目录，如 `"一级/二级/三级"`
- 每级名称 trim 后不能为空
- 单级名称最长 100 个字符
- 不允许包含 `\` 字符
- 最多支持 10 级嵌套

## curl 示例

```bash
curl -X POST 'http://dev.localhost:18000/api/v1/kb/folder/create' \
  -H 'Authorization: Bearer <system_api_token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "system_id": "obsidian",
    "access_credential": "AES-256-GCM 加密后的用户工号（base64）",
    "kb_id": "be8666e3-7823-4e20-9f1c-f77ac604b47a",
    "path": "技术文档1/后端/API设计"
  }'
```

## 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| folder_id | string | 最终叶子文件夹的 ID |
| folder_name | string | 最终叶子文件夹的名称 |
| created | array | 本次新创建的文件夹列表 |
| created[].folder_id | string | 新建文件夹 ID |
| created[].folder_name | string | 新建文件夹名称 |
| reused | array | 已存在直接复用的文件夹列表 |
| reused[].folder_id | string | 复用文件夹 ID |
| reused[].folder_name | string | 复用文件夹名称 |

## 权限检查流程

1. **系统级 Token 校验** — 必须使用系统级 API Token（KB 级别 Token 不可用）
2. **Token 权限** — 必须为 `full_control` 或 `doc_manage`
3. **用户身份解析** — 解密 access_credential 获取用户工号，查找对应 authID
4. **知识库授权** — kb_id 必须在 Token 授权的知识库范围内
5. **用户 KB 权限** — 用户必须有该知识库的访问权限
6. **管理员校验** — 必须是管理员用户，非管理员返回 40003

## 错误码

| code | 说明 |
|------|------|
| 40001 | 参数错误（缺少必填字段、path 格式错误、名称超长、嵌套超限等） |
| 40003 | 权限不足（用户不存在、无 KB 权限、非管理员） |
| 50001 | 服务端错误（查询或创建失败） |

## 注意事项

- 文件夹创建后为**未发布状态**，但通过"查询文件夹"接口可以查到
- 同级目录下不允许同名文件夹，传入已存在的路径会直接复用
- 返回的 `folder_id` 可直接用于上传文档时指定 `parent_id`（如果上传接口支持）
- 路径中的 `/` 仅作为层级分隔符，文件夹名称本身不能包含 `/` 或 `\`
