# 创建文档

## 功能描述

创建一个**空文档节点**（仅创建节点，不写入内容），对应管理后台"创建文档"弹窗的接口化版本。支持富文本（HTML）与 Markdown 两种类型；Excel 类型不在本接口范围内。如需发布，请使用 `/api/v1/kb/doc/publish` 接口。

## 功能特性

- 仅创建空文档，不携带正文内容；创建后处于**未发布**状态
- 支持富文本 / Markdown 两种 content_type（Excel 不支持）
- 可指定 `parent_id` 落到指定文件夹；不传则创建在知识库根目录
- 可选设置 `emoji` 作为文档图标

## 实现文件

### 后端

- `backend/handler/openapi/kb.go` — 路由注册 `POST /api/v1/kb/doc/create` 与 `CreateDoc` handler
- `backend/usecase/node.go` — `NodeUsecase.Create` 节点创建逻辑（本接口复用）
- `backend/domain/node.go` — `CreateNodeReq` / `NodeType` / `ContentType` 枚举

### 前端

- 管理后台"创建文档"弹窗：`web/admin/src/pages/document/component/AddDocBtn.tsx`

## API 接口

| 方法 | 路径 | Content-Type |
|------|------|--------------|
| POST | `/api/v1/kb/doc/create` | `application/json` |

## 请求头

| Header | 必填 | 说明 |
|--------|------|------|
| Authorization | 是 | `Bearer <system_api_token>`，仅支持系统级 API Token，KB 级别 Token 不可用 |

## 数据结构

### 请求字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| system_id | string | 是 | 调用方系统标识 |
| agent_id | string | 否 | 智能体 ID |
| access_credential | string | 是 | notes_id + Secret 经 AES-256-GCM 加密后的 Base64 密文 |
| kb_id | string | 是 | 目标知识库 ID |
| parent_id | string | 否 | 父文件夹 ID，不传放根目录 |
| doc_name | string | 是 | 文档名称，trim 后非空，最长 100 字符，不允许包含 `/` 或 `\` |
| content_type | string | 是 | 文档类型：`html`（富文本）或 `md`（Markdown） |
| emoji | string | 否 | 文档图标 emoji |

### 响应字段

| 字段 | 类型 | 说明 |
|------|------|------|
| data.doc_id | string | 新创建的文档 ID |
| data.doc_name | string | 文档名称 |

## curl 示例

```bash
curl -X POST 'http://dev.localhost:18000/api/v1/kb/doc/create' \
  -H 'Authorization: Bearer <system_api_token>' \
  -H 'Content-Type: application/json' \
  -d '{
    "system_id": "obsidian",
    "access_credential": "AES-256-GCM 加密后的用户工号（base64）",
    "kb_id": "be8666e3-7823-4e20-9f1c-f77ac604b47a",
    "parent_id": "019b48ef-4ef0-...",
    "doc_name": "数据库配置指南",
    "content_type": "md",
    "emoji": "📘"
  }'
```

{

## 权限控制

| 操作 | 所需权限 |
|------|---------|
| 调用本接口 | 系统级 API Token + Token 权限为 `full_control` 或 `doc_manage` |
| kb_id | 必须在 Token 授权的知识库范围内 |
| 用户 | 必须能解析到 authID 且拥有该知识库访问权限 |
| parent_id | 若传入，用户必须拥有该文件夹的 visitable 权限 |

### 校验流程

1. 系统级 Token 校验（KB 级 Token 拒绝）
2. Token 权限为 `full_control` 或 `doc_manage`
3. 请求参数校验（包括 content_type 白名单）
4. 解密 `access_credential` 得到 notes_id
5. 校验 `kb_id` 在 Token 授权范围内
6. 解析 authID → 校验用户对该 KB 的访问权限
7. 若传 `parent_id` → 校验 visitable 权限
8. 调用 `NodeUsecase.Create` 创建未发布文档节点

## 依赖关系

- **上游依赖**：系统 API Token 模块、access_credential 加解密模块、用户认证模块、知识库权限模块、文件夹 visitable 权限（parent_id 校验）
- **下游影响**：
  - `/api/v1/kb/doc/edit`：后续写入正文
  - `/api/v1/kb/doc/publish`：后续发布
  - 知识库目录树、文档列表

## 功能边界

- **本模块负责**：创建空文档节点、名称/类型/parent 校验、权限校验、写入调用日志
- **本模块不负责**：
  - 文档正文内容写入（由 `/api/v1/kb/doc/edit` 处理）
  - 文档发布（由 `/api/v1/kb/doc/publish` 处理）
  - 文件上传解析（由 `/api/v1/kb/upload` 处理，带内容的文档通过该接口）
  - Excel 类型文档创建（当前接口不支持，后续如需支持需补充处理分支）
  - 同级同名去重（允许同级同名文档存在，不做覆盖/复用）

## 注意事项

- 创建后文档状态为**未发布**，在前台站点不可见，需再调 `/api/v1/kb/doc/publish` 发布
- `content_type=html` 时后端内部 ContentType 置空，走默认富文本路径（与前端"富文本"等价）
- `content_type=md` 时以 Markdown 形式保存；渲染阶段编辑器会按 md 解析
- 不支持 `content_type=excel`，传入会返回 40001
- 同级目录下允许同名文档（本接口不做查重），如需覆盖请先通过其他接口删除旧节点
- `doc_name` 长度上限 100 字符（按 Unicode 字符计）

## 错误码

| code | 说明 |
|------|------|
| 401 | 非系统级 API Token 或 Token 无效 |
| 403 | Token 权限不足（需要 full_control 或 doc_manage） |
| 40001 | 参数错误（缺字段、doc_name 非法、content_type 非白名单、access_credential 解密失败等） |
| 40003 | 权限不足（用户不存在、无 KB 权限、无 parent_id 文件夹权限） |
| 50001 | 服务端错误（查询或创建失败） |
