# 删除节点 RAG 文档残留修复

## 问题描述

删除已发布的文档节点时，raglite 中对应的 document 和 chunks 没有被正确删除，导致向量检索可能返回已删除文章的内容。

### 问题现象

- 用户删除已发布的文档后，搜索仍能检索到该文档的内容
- raglite `documents` 表中存在孤儿数据（已删除节点对应的文档）
- 向量检索结果包含过时或已删除的内容

### 问题根因

在消息队列处理流程中，**`DocID` 字段在消息转换时丢失**：

```
usecase/node.go                    repo/mq/rag.go                      handler/mq/rag.go
─────────────────────────────────────────────────────────────────────────────────────────
NodeReleaseVectorRequest {         NodeContentVectorRequest {          DeleteRecords(docIDs)
  KBID: "kb-123",                    KBID: "kb-123",                     ↑
  NodeReleaseID: "rel-456",   →      ID: "rel-456",  ←─ 错误的 ID！     │
  DocID: "doc-789",  ←─ 丢失！       Action: "delete",                  │
  Action: "delete",                }                                    │
}                                                                       │
                                   DeleteRecords 需要的是 "doc-789"  ────┘
```

**具体问题**：

1. `usecase/node.go` 正确设置了 `DocID`（raglite 的文档 ID）
2. `repo/mq/rag.go` 转换消息时，只映射了 `NodeReleaseID` → `ID`，**丢失了 `DocID`**
3. `handler/mq/rag.go` 使用 `request.ID`（实际是 `NodeReleaseID`）调用删除
4. `DeleteRecords` 需要 raglite 的 `doc_id`，而不是 panda-wiki 的 `node_release_id`

## 解决方案

### 修改文件

1. `backend/domain/mq.go`
2. `backend/repo/mq/rag.go`
3. `backend/handler/mq/rag.go`

### 修改内容

#### 1. domain/mq.go - 添加 DocID 字段

```go
// 修改前
type NodeContentVectorRequest struct {
    KBID   string `json:"kb_id"`
    ID     string `json:"id"`
    Action string `json:"action"` // upsert, delete
}

// 修改后
type NodeContentVectorRequest struct {
    KBID   string `json:"kb_id"`
    ID     string `json:"id"`
    DocID  string `json:"doc_id"` // raglite document ID, used for delete action
    Action string `json:"action"` // upsert, delete
}
```

#### 2. repo/mq/rag.go - 转换时保留 DocID

```diff
 func (r *RAGRepository) AsyncUpdateNodeReleaseVector(ctx context.Context, request []*domain.NodeReleaseVectorRequest) error {
     for _, req := range request {
         contentReq := &domain.NodeContentVectorRequest{
             KBID:   req.KBID,
             ID:     req.NodeReleaseID,
+            DocID:  req.DocID,         // 保留 DocID，用于删除操作
             Action: req.Action,
         }
         // ...
     }
 }
```

#### 3. handler/mq/rag.go - 使用正确的 DocID 删除

```diff
 case "delete":
     h.logger.Info("delete node content vector request", log.Any("request", request))
+
+    // 使用 DocID 进行删除，DocID 是 raglite 中的文档 ID
+    docIDToDelete := request.DocID
+    if docIDToDelete == "" {
+        h.logger.Warn("DocID is empty for delete action, skipping", log.String("request_id", request.ID))
+        return nil
+    }
+
     kb, err := h.kbRepo.GetKnowledgeBaseByID(ctx, request.KBID)
     if err != nil {
         h.logger.Error("get kb failed", log.Error(err))
         return err
     }
-    if err := h.rag.DeleteRecords(ctx, kb.DatasetID, []string{request.ID}); err != nil {
-        h.logger.Error("delete node content vector failed", log.Error(err))
+    if err := h.rag.DeleteRecords(ctx, kb.DatasetID, []string{docIDToDelete}); err != nil {
+        h.logger.Error("delete node content vector failed", log.Error(err), log.String("doc_id", docIDToDelete))
         return err
     }
-    h.logger.Info("delete node content vector success", log.Any("deleted_id", request.ID))
+    h.logger.Info("delete node content vector success", log.String("doc_id", docIDToDelete))
 }
```

## 清理历史数据

对于已存在的孤儿 document，可以通过以下步骤清理：

```sql
-- 1. 在 panda-wiki 中查询所有已发布文档的 doc_id
SELECT doc_id FROM node_releases WHERE doc_id IS NOT NULL AND doc_id != '';

-- 2. 在 raglite 中查询所有 documents
SELECT id FROM documents;

-- 3. 对比找出孤儿 documents（在 raglite 中存在但不在 panda-wiki node_releases.doc_id 中）
-- 4. 手动删除孤儿 documents
```
