# 修复重新学习显示旧失败状态问题

**日期**: 2025-01-24
**修改文件**:
- `backend/usecase/node.go`
- `backend/handler/mq/rag.go`

## 问题描述

用户在执行"重新学习"操作后：
1. 后台实际学习成功
2. 但前端弹窗仍显示旧的失败错误信息
3. 关闭弹窗后查看，文档状态实际是成功的

错误示例：
```
快速上手 - 新手必读 ！！！: 向量数据库更新失败: failed to update chunk ee6fb6a5-...
keywords: SetPayload() failed: raglite: rpc error: code = NotFound desc = Not found
```

## 问题分析

### 根本原因

**时序问题**：`NodeRestudy` 函数发送 MQ 消息后，没有立即更新节点状态为"学习中"。

### 问题时序图

### 关键问题点

1. **前端轮询逻辑**：检测到 `ENHANCE_FAILED` 是终态，立即停止轮询并显示失败
2. **后端未清状态**：`NodeRestudy` 只发送 MQ 消息，未将旧的失败状态清除
3. **竞态条件**：轮询在 MQ 处理完成前就读到了旧状态

## 修复方案

### 修复 1: NodeRestudy 立即更新状态

**文件**: `backend/usecase/node.go`

**修改内容**: 在发送 MQ 消息前，立即将节点状态更新为 `BASIC_RUNNING`

```go
func (u *NodeUsecase) NodeRestudy(ctx context.Context, req *v1.NodeRestudyReq) error {
    nodeReleases, err := u.nodeRepo.GetLatestNodeReleaseByNodeIDs(ctx, req.KbId, req.NodeIds)
    if err != nil {
        return fmt.Errorf("get latest node release failed: %w", err)
    }

    for _, nodeRelease := range nodeReleases {
        // ✅ 新增: 立即更新节点状态为"学习中"，防止前端轮询时读到旧的失败状态
        if err := u.nodeRepo.Update(ctx, nodeRelease.NodeID, map[string]any{
            "rag_info": domain.RagInfo{
                Status:  consts.NodeRagStatusBasicRunning,
                Message: "",  // 清空旧的错误信息
            },
        }); err != nil {
            u.logger.Error("update node rag status to running failed",
                log.String("node_id", nodeRelease.NodeID),
                log.Error(err))
            // 即使更新失败也继续发送 MQ 消息
        }

        // 发送 MQ 消息触发向量化
        if err := u.ragRepo.AsyncUpdateNodeReleaseVector(ctx, []*domain.NodeReleaseVectorRequest{
            // ...
        }); err != nil {
            // ...
        }
    }
    return nil
}
```

### 修复 2: 删除旧文档防止残留

**文件**: `backend/handler/mq/rag.go`

**修改内容**: 在创建新文档前，先删除旧文档

```go
// Delete old document if exists (for restudy scenario)
// This prevents multiple document versions from accumulating in raglite
if nodeRelease.DocID != "" {
    h.logger.Info("deleting old document before upsert",
        log.String("old_doc_id", nodeRelease.DocID),
        log.String("node_release_id", request.ID))
    if err := h.rag.DeleteRecords(ctx, kb.DatasetID, []string{nodeRelease.DocID}); err != nil {
        h.logger.Warn("failed to delete old document, continuing with upsert",
            log.String("old_doc_id", nodeRelease.DocID),
            log.Error(err))
        // Don't return error, continue with upsert
    }
}
```

## 验证方式

### 验证修复

1. 找一个显示"增强学习失败"的文档
2. 点击"重新学习"
3. 应该看到状态变为"学习中"（而不是立即显示旧错误）
4. 等待学习完成后显示"成功"

### 回归测试

1. **首次学习**: 新文档首次学习应正常工作
2. **重新学习成功文档**: 对已成功的文档重新学习应正常
3. **重新学习失败文档**: 对失败文档重新学习应显示正确状态
4. **批量学习**: 同时学习多个文档应正常

## 教训总结

1. **异步操作需要状态管理**: 在触发异步操作前，应先更新状态为"进行中"
2. **前端轮询的终态判断**: 需要考虑轮询可能读到旧状态的情况
3. **清理旧数据**: 重新操作时应清理旧数据，避免残留影响
