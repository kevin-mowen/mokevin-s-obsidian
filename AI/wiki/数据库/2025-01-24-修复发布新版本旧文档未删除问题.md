# 修复发布新版本旧文档未删除问题

**日期**: 2025-01-24
**修改文件**:
- `backend/repo/pg/node.go`
- `backend/handler/mq/rag.go`
- `web/admin/src/pages/stat/Statistic/index.tsx`

## 问题描述

### 问题 1：快速发布后搜索到旧版本信息

用户发布文档新版本后，搜索时仍能找到旧版本的内容。

### 问题 2：发布后快速删除仍生成 RAG 向量

用户发布文档后快速删除，但 RAG 向量仍会被创建。

## 问题分析

### 问题 1 根因

发布新版本时的代码流程：

```
发布新版本 → 创建新 node_release (DocID="")
                ↓
MQ handler 检查: nodeRelease.DocID != "" ?
                ↓
            否（新 release 的 DocID 为空），跳过删除
                ↓
旧 releases 的 doc_ids 仍然存在于 raglite ❌
```

**之前的修复**只检查**当前 release** 的 DocID（用于重新学习场景），但新 release 的 DocID 是空的。应该查找**同一 node 的所有历史 releases** 的 doc_ids。

### 问题 2 根因

```
T0: 发布 → MQ 消息 {action: "upsert", releaseID: "xxx"}
T1: 删除 → 硬删除 nodes + node_releases → MQ 消息 {action: "delete"}
T2: MQ 处理 upsert → GetNodeReleaseWithDirPathByID 返回 error（已删除）
    → 被当作临时错误，触发重试
    → 重试多次后才放弃
```

## 修复方案

### 修复 1：获取同一节点的历史 doc_ids

**文件**: `backend/repo/pg/node.go`

新增方法获取同一节点的历史 doc_ids：

```go
// GetOldDocIDsByNodeID 获取同一节点的历史 doc_ids（排除当前 release）
// 用于发布新版本时删除旧文档
func (r *NodeRepository) GetOldDocIDsByNodeID(ctx context.Context, nodeID, excludeReleaseID string) ([]string, error) {
    var docIDs []string
    if err := r.db.WithContext(ctx).
        Model(&domain.NodeRelease{}).
        Where("node_id = ?", nodeID).
        Where("id != ?", excludeReleaseID).
        Where("doc_id != ''").
        Pluck("doc_id", &docIDs).Error; err != nil {
        return nil, err
    }
    return docIDs, nil
}
```

### 修复 2：MQ Handler 删除所有历史文档

**文件**: `backend/handler/mq/rag.go`

修改 upsert 逻辑，删除同一节点的所有历史文档：

```go
// Delete old documents for this node (for publish new version / restudy scenario)
// This prevents multiple document versions from accumulating in raglite
oldDocIDs, err := h.nodeRepo.GetOldDocIDsByNodeID(ctx, nodeRelease.NodeID, request.ID)
if err != nil {
    h.logger.Warn("failed to get old doc_ids, continuing with upsert", log.Error(err))
} else if len(oldDocIDs) > 0 {
    h.logger.Info("deleting old documents before upsert",
        log.Any("old_doc_ids", oldDocIDs),
        log.String("node_id", nodeRelease.NodeID),
        log.String("node_release_id", request.ID))
    if err := h.rag.DeleteRecords(ctx, kb.DatasetID, oldDocIDs); err != nil {
        h.logger.Warn("failed to delete old documents, continuing with upsert",
            log.Any("old_doc_ids", oldDocIDs),
            log.Error(err))
    }
}
```

### 修复 3：区分"已删除"和"临时错误"

**文件**: `backend/handler/mq/rag.go`

当节点已被删除时，不再重试，直接跳过：

```go
nodeRelease, err := h.nodeRepo.GetNodeReleaseWithDirPathByID(ctx, request.ID)
if err != nil {
    // 如果记录不存在（已被删除），直接跳过不重试
    if errors.Is(err, gorm.ErrRecordNotFound) {
        h.logger.Info("node release not found (deleted), skip upsert",
            log.String("node_release_id", request.ID))
        return nil
    }
    // 其他错误按原逻辑重试
    ...
}
```

### 修复 4：delete 操作添加最大重试次数限制

**文件**: `backend/handler/mq/rag.go`

delete 操作之前没有最大重试次数限制，如果 RAG 服务持续不可用会无限重试：

```go
case "delete":
    // ... 现有代码 ...

    kb, err := h.kbRepo.GetKnowledgeBaseByID(ctx, request.KBID)
    if err != nil {
        h.logger.Error("get kb failed", log.Error(err))
        // 添加最大重试次数检查
        if numDelivered >= MaxRAGRetry {
            h.logger.Error("get kb failed after max retries, giving up delete",
                log.String("doc_id", docIDToDelete),
                log.Any("retries", numDelivered))
            return nil
        }
        return err
    }
    if err := h.rag.DeleteRecords(ctx, kb.DatasetID, []string{docIDToDelete}); err != nil {
        h.logger.Error("delete node content vector failed", ...)
        // 添加最大重试次数检查
        if numDelivered >= MaxRAGRetry {
            h.logger.Error("delete failed after max retries, document may be orphaned",
                log.String("doc_id", docIDToDelete),
                log.Any("retries", numDelivered))
            return nil
        }
        return err
    }
```

## API 说明

### 向量统计 API（仅超级管理员）

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/v1/vector/stats` | GET | 获取指定知识库的向量统计 |
| `/api/v1/vector/stats/all` | GET | 获取所有知识库的向量统计 |
| `/api/v1/vector/consistency` | GET | 检查数据一致性 |
| `/api/v1/vector/cleanup/orphaned` | POST | 清理孤立文档 |
| `/api/v1/vector/cleanup/multi-version` | POST | 清理多版本 |

### 清理功能说明

- **孤立文档**: RAG 中存在但数据库中已删除的文档
- **多版本**: 同一节点在 RAG 中存在多个版本的向量（只应保留最新版）
