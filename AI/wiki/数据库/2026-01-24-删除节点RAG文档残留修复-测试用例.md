## 日志验证

修复后，删除操作的日志应该显示：

```
INFO  delete node content vector request  request={"kb_id":"xxx","id":"yyy","doc_id":"zzz","action":"delete"}
INFO  delete node content vector success  doc_id=zzz
```

**关键检查点**：
- `doc_id` 字段不为空
- `doc_id` 值与 `node_releases.doc_id` 一致

## 数据验证 SQL

### 验证数据一致性

```sql
-- panda-wiki 数据库：查询所有已发布文档的 doc_id
SELECT
    nr.id as release_id,
    nr.node_id,
    nr.doc_id,
    n.name as node_name
FROM node_releases nr
JOIN nodes n ON nr.node_id = n.id
WHERE nr.doc_id IS NOT NULL AND nr.doc_id != '';

-- raglite 数据库：查询所有 documents
SELECT id, name, dataset_id FROM documents;
```

### 检查孤儿数据

```sql
-- 找出 raglite 中存在但 panda-wiki 中不存在的 document（孤儿数据）
-- 在 raglite 中执行
SELECT id FROM documents
WHERE id NOT IN (
    SELECT doc_id FROM panda_wiki.node_releases
    WHERE doc_id IS NOT NULL AND doc_id != ''
);
```

## 备注

1. **已知问题**：当前实现中，删除文档节点时可能只删除最新版本的 RAG document。如果该文档有多次发布历史，旧版本的 documents 可能仍会残留。此问题需要后续优化。

2. **测试数据清理**：测试完成后，确保清理所有测试创建的文档和 document。

3. **关联修复**：此修复与「删除知识库 RAG 数据残留修复」属于同一类问题，建议一起回归测试。
