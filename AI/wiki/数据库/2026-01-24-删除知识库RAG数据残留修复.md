# 删除知识库 RAG 数据残留修复

## 问题描述

删除知识库时，RAG 服务（raglite）中的 dataset 没有被正确删除，导致数据不一致。

### 问题现象

- panda-wiki 数据库 `knowledge_bases` 表有 4 条记录
- raglite 数据库 `datasets` 表有 6 条记录
- 多出的 2 条是已删除知识库对应的孤儿 dataset

### 问题根因

在 `usecase/knowledge_base.go` 的 `DeleteKnowledgeBase` 函数中，调用 RAG 服务删除 dataset 时，**传递了错误的参数**：

```go
// 修复前的代码
func (u *KnowledgeBaseUsecase) DeleteKnowledgeBase(ctx context.Context, kbID string) error {
    if err := u.repo.DeleteKnowledgeBase(ctx, kbID); err != nil {
        return err
    }
    // ❌ Bug: 这里传的是 kbID，但 RAG 服务需要的是 datasetID
    if err := u.rag.DeleteKnowledgeBase(ctx, kbID); err != nil {
        return err
    }
    // ...
}
```

**说明**：
- `kbID` 是 panda-wiki 的知识库 ID（如 `abc-123`）
- `datasetID` 是 raglite 的数据集 ID（如 `xyz-789`）
- 两者是不同的值，通过 `knowledge_bases.dataset_id` 字段关联

## 解决方案

在删除前先获取知识库信息，拿到正确的 `datasetID`，再调用 RAG 服务删除。

### 修改文件

`backend/usecase/knowledge_base.go`

### 修改内容

```go
// 修复后的代码
func (u *KnowledgeBaseUsecase) DeleteKnowledgeBase(ctx context.Context, kbID string) error {
    // 先获取知识库信息，拿到 datasetID
    kb, err := u.repo.GetKnowledgeBaseByID(ctx, kbID)
    if err != nil {
        return err
    }

    if err := u.repo.DeleteKnowledgeBase(ctx, kbID); err != nil {
        return err
    }
    // delete vector store (使用正确的 datasetID)
    if err := u.rag.DeleteKnowledgeBase(ctx, kb.DatasetID); err != nil {
        return err
    }
    if err := u.kbCache.DeleteKB(ctx, kbID); err != nil {
        return err
    }
    return nil
}
```

## 清理历史数据

对于已存在的孤儿 dataset，需要手动清理：

```sql
-- 1. 在 raglite 中查询所有 dataset id
SELECT id FROM datasets;

-- 2. 在 panda-wiki 中查询所有 dataset_id
SELECT dataset_id FROM knowledge_bases;

-- 3. 找出不在 panda-wiki 中的 dataset（孤儿数据）
-- 4. 手动删除孤儿 dataset
```
