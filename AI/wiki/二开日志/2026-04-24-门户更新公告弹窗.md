# 门户更新公告弹窗

## 功能描述
门户首页（`/`）登录后自动弹出最新版本更新公告，可在 admin 后台 CRUD 公告。

## 功能特性
- Admin 创建 / 编辑 / 删除 / 发布开关公告
- 公告含版本号、标题、发布日期、多条更新条目（6 种类型）
- 门户首次访问或发布新版本时自动弹窗（已登录用户）
- 弹窗支持滚动，可展示全部条目
- "不再提示该版本"写入 localStorage，永久忽略该 version
- "稍后查看"仅关闭本次，下次访问仍弹
- "查看全部"跳转到 admin 配置的 KB 文档（`/node/{id}?kb={kb_id}`），若未配置则按钮隐藏
- Admin 在公告页顶部选"同步目标 KB + 文档"；公告发布/更新/删除时自动把所有已发布公告按版本号倒序拼成 Markdown 整体覆盖写入该文档
- 匿名用户不弹（门户 layout 已登录保护）

## 数据结构

### 主表: announcements
| 字段 | 类型 | 说明 |
|-----|------|------|
| id | TEXT | 主键（UUID v7） |
| version | TEXT | 版本号，如 v1.8.0，UNIQUE |
| title | TEXT | 公告标题 |
| publish_date | DATE | 发布日期 |
| items | JSONB | 条目数组 `[{type, title, summary}]` |
| is_published | BOOLEAN | 是否已发布，默认 false |
| created_at | TIMESTAMPTZ | 创建时间 |
| updated_at | TIMESTAMPTZ | 更新时间 |

索引: (is_published, publish_date DESC)

### 条目类型枚举
type: feature（新功能）| improve（优化）| fix（修复）| breaking（重要变更）| security（安全通告）| maint（停机维护）

### 条目结构 items[i]
| 字段 | 类型 | 说明 |
|-----|------|------|
| type | string | 上述枚举之一 |
| title | string | 条目标题 |
| summary | string | 条目描述（可选） |

## API 接口

### Admin（/api/v1/announcement）
| 方法 | 路径 | 说明 |
|-----|------|------|
| GET | /api/v1/announcement/list | 分页列表（含草稿） |
| GET | /api/v1/announcement/:id | 详情 |
| POST | /api/v1/announcement | 新建（发布状态下触发 Node 同步） |
| PUT | /api/v1/announcement/:id | 更新（触发 Node 同步） |
| DELETE | /api/v1/announcement/:id | 删除（触发 Node 同步，对应版本内容从文档消失） |
| POST | /api/v1/announcement/:id/publish | 发布/取消发布（触发 Node 同步） |
| GET | /api/v1/announcement/config | 读取同步目标（kb_id + node_id） |
| PUT | /api/v1/announcement/config | 保存同步目标（保存后立即同步一次） |

### 门户（/share/v1/announcement）
| 方法 | 路径 | 说明 |
|-----|------|------|
| GET | /share/v1/announcement/latest | 最新已发布（弹窗用，nil 表示不弹） |
| GET | /share/v1/announcement/list | 全部已发布（备用） |
| GET | /share/v1/announcement/config | 查看全部按钮跳转目标 |

## 权限控制
| 操作 | 所需权限 |
|-----|---------|
| Admin CRUD / 发布开关 | 登录态（V1 Authorize） |
| 门户拉最新公告 | 登录态（PortalAuthorize），匿名返回 401 |

## 依赖关系
- **上游依赖**: 用户认证（session.user_id）、门户登录 layout (`(portal)/layout.tsx`)
- **下游影响**: 无（独立模块，不被其他模块依赖）

## 功能边界
- **本模块负责**: 公告数据 CRUD、弹窗展示、dismiss 状态（localStorage）、/announcements 日志页
- **本模块不负责**:
  - KB 级别的公告隔离（当前为系统级全局，所有 KB 共用）
  - 用户维度的 dismiss 跨设备同步（仅存本地 localStorage）
  - 公告内容的富文本渲染（仅支持纯文本 title + summary）
  - KB 文档形态的版本更新页（如需参考"版本更新记录"KB doc 的 TOC/浏览量，需走 node 模块）

## 注意事项
- **版本号唯一**：`version` UNIQUE，重复会报错
- **date 格式**：传 YYYY-MM-DD，后端用 `time.Parse("2006-01-02")` 解析
- **弹窗判定**：`localStorage.pw_announcement_dismissed_versions` 是字符串数组，`latest.version` 不在其中才弹
- **新版本强制再弹**：admin 发布更高版本号后，所有浏览器会再弹（因 version 不匹配 dismissed）
- **匿名绕过跳登录**：组件用原生 fetch 而非 httpClient，401 静默，避免把匿名用户从公开门户踢到登录页
- **门户挂载点**：只在 `src/app/(portal)/page.tsx`（`/`）挂，不要挂到 `/home`（那是 KB 文档首页，不是门户）

## 踩坑记录
### 问题 1: 弹窗最初挂到了 /home 不是门户
- **原因**: 以为 `(pages)/(doc)/home/page.tsx` 是门户，实际它是 KB 文档首页
- **解决**: 真正门户在 `src/app/(portal)/page.tsx`，对应 URL `/`

### 问题 2: 列表只显示前 3 条没法滚动
- **原因**: 设计稿里 `highlights = items.slice(0, 3)` 只取前 3
- **解决**: 去掉 slice，容器加 `maxHeight: 360px, overflowY: auto`

### 问题 3: 401 被 httpClient 自动重定向
- **原因**: `web/app/src/request/httpClient.ts` 对 401 默认 redirect 到 `/auth/login`
- **解决**: 弹窗/日志页不走 httpClient，直接 `fetch(...)` + `credentials: 'include'`，401 静默返回
