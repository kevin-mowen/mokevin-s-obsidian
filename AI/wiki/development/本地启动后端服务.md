# 本地启动后端服务指南

本文档说明如何在本地（非 Docker 容器内）直接启动 PandaWiki 后端 API / Consumer 服务。

> 推荐做法：直接用 `scripts/dev-services-cl.sh` 菜单选 `6`（启动 API）/ `8`（启动 Consumer），脚本已内置端口探测和 kill_port 逻辑。本文只在"需要手工控制/调试"时参考。

## 前置条件

### 1. 启动依赖的基础服务

后端服务依赖以下基础服务，通过 Docker 运行（由 `docker/dev/docker-compose.*.yml` 逐个拆分管理）：

| 服务 | 容器名 | 宿主机端口 | 说明 |
|---|---|---|---|
| PostgreSQL | `panda-wiki-postgres` | `13100` | 主数据库 |
| Redis | `panda-wiki-redis` | `13500` | 缓存 |
| NATS | `panda-wiki-nats` | `13200` | 消息队列 |
| MinIO | `panda-wiki-minio` | `13600` / `13601`（控制台） | S3 对象存储 |
| Qdrant | `panda-wiki-qdrant` | `13701` | 向量数据库 |
| Raglite | `panda-wiki-raglite` | `13400` | RAG 服务 |
| Crawler | `panda-wiki-crawler` | `13800` | 爬虫服务 |

一键启动所有基础服务：

```bash
bash ./scripts/dev-services-cl.sh
# 菜单选 1 = 启动所有基础服务
# 或选 30 = 查看状态
```

或单独起某个：

```bash
docker-compose -f docker/dev/docker-compose.postgres.yml up -d
docker-compose -f docker/dev/docker-compose.raglite.yml up -d
```

### 2. 安装 Go 环境

确保已安装 Go 1.24.3 或更高版本（与 `backend/go.mod` 一致）：

```bash
go version
```

## 配置说明

本地启动使用的配置文件为 `backend/config/config.local.yml`，该文件已配置为通过 `dev.localhost` 域名访问基础服务（域名映射由 `sudo bash ./scripts/dev-setup-hosts.sh` 写入 `/etc/hosts`）。

### 关键配置项（当前版本）

```yaml
# HTTP 监听端口
http:
  port: 18000

# PostgreSQL
pg:
  dsn: 'host=dev.localhost user=panda-wiki password=panda_wiki_local_2025 dbname=panda-wiki port=13100 sslmode=disable TimeZone=Asia/Shanghai'

# NATS
mq:
  type: nats
  nats:
    server: 'nats://dev.localhost:13200'
    user: 'panda-wiki'
    password: 'panda_wiki_local_2025'

# Redis
redis:
  addr: 'dev.localhost:13500'
  password: 'panda_wiki_local_2025'

# MinIO（S3 兼容）
s3:
  endpoint: 'dev.localhost:13600'
  access_key: 's3panda-wiki'
  secret_key: 'panda_wiki_local_2025'

# Raglite（v2.x，不带 /api/v1 后缀）
rag:
  provider: ct
  ct_rag:
    base_url: 'http://dev.localhost:13400'
    api_key: 'sk-1234567890'

# Crawler
crawler:
  host: 'dev.localhost'
  port: 13800
```

> **Raglite 端口注意**：默认 SDK（`chaitin/raglite-go-sdk v0.2.1`）不再使用 `/api/v1` 前缀，新 base_url 是裸端口。参考 `backend/config/config.go:155`。

### 前后端互引地址

```yaml
wiki_base_url: 'http://dev.localhost:3010'   # App 前台（next dev -p 3010）
admin_base_url: 'http://dev.localhost:3000'  # Admin 管理台（vite dev）
nginx_base_url: 'http://dev.localhost:13600' # 静态文件，本地开发指向 MinIO

api:
  host: 'dev.localhost'
  port: 18000
```

## 本地环境数据存储

本地运行时，以下数据存储在 `docker/data` 目录：

- **Machine ID 文件**：`docker/data/.machine_id` — 用于遥测唯一标识
- **SSL 证书**：`docker/data/nginx/ssl/`（dev 模式一般用不到，local 模式用）

Docker 与本地环境共享该目录，便于统一管理。

## 启动步骤

**重要提示**：`make dev` 会构建 Docker 镜像并启动容器，不适合本地直接运行！本地跑后端请走下面方式。

### 方式一：通过脚本菜单（推荐）

```bash
bash ./scripts/dev-services-cl.sh
# 选 6 = 启动 API（在新 Terminal 窗口跑 `go run ./cmd/api`）
# 选 8 = 启动 Consumer
```

### 方式二：直接运行

```bash
cd backend
go run ./cmd/api
```

**注意**：不要使用 `go run cmd/api/main.go`，这样会缺少 Wire 生成的 `wire_gen.go`。

### 方式三：编译后运行

```bash
cd backend
go build -o panda-wiki-api ./cmd/api
./panda-wiki-api
```

### 方式四：Air 热重载（重度开发推荐）

```bash
go install github.com/air-verse/air@latest  # 注意：cosmtrek/air 已迁移
cd backend
air
```

Air 配置示例 (`.air.toml`):

```toml
root = "."
tmp_dir = "tmp"

[build]
  bin = "./tmp/main"
  cmd = "go build -o ./tmp/main ./cmd/api"
  delay = 1000
  exclude_dir = ["tmp", "vendor", "pro"]
  exclude_regex = ["_test.go"]
  include_ext = ["go", "yml"]
  log = "build-errors.log"
  stop_on_error = true
```

## 环境变量覆盖

如需用环境变量覆盖 `config.local.yml`（优先级高于文件）：

```bash
# 完整 PostgreSQL DSN（优先级最高）
export PG_DSN="host=dev.localhost user=panda-wiki password=panda_wiki_local_2025 dbname=panda-wiki port=13100 sslmode=disable TimeZone=Asia/Shanghai"

# NATS / Redis / MinIO / Raglite
export MQ_NATS_SERVER="nats://dev.localhost:13200"
export REDIS_ADDR="dev.localhost:13500"
export S3_ENDPOINT="dev.localhost:13600"
export RAGLITE_URL="http://dev.localhost:13400"

# 密码
export POSTGRES_PASSWORD="panda_wiki_local_2025"
export REDIS_PASSWORD="panda_wiki_local_2025"
export NATS_PASSWORD="panda_wiki_local_2025"
export S3_SECRET_KEY="panda_wiki_local_2025"
export JWT_SECRET="panda_wiki_jwt_secret_dev_2025"
export ADMIN_PASSWORD="Admin@2025"
```

> 环境变量 `PG_DSN` 优先级高于 `POSTGRES_PASSWORD`。同时设置时以 `PG_DSN` 为准。

## 常见问题

### 1. 无法连接到 PostgreSQL

**症状**：启动时报 `connection refused` 或 `no such host: dev.localhost`

**排查**：
- 容器是否在跑：`docker ps | grep panda-wiki-postgres`
- 端口映射是否是 `13100`：`docker port panda-wiki-postgres`
- 是否跑过 `sudo bash ./scripts/dev-setup-hosts.sh` 把 `dev.localhost` 写入 hosts：`cat /etc/hosts | grep dev.localhost`

### 2. 无法连接到 Redis / NATS / MinIO

**排查**：
```bash
# 统一查状态
bash ./scripts/dev-services-cl.sh   # 选 30

# 端口占用
lsof -i :13500   # Redis
lsof -i :13200   # NATS
lsof -i :13600   # MinIO
```

### 3. Raglite 连接失败

- v2.x 的 base_url 是 `http://dev.localhost:13400`（**不带** `/api/v1`）
- 容器日志：`docker logs panda-wiki-raglite --tail 100 -f`
- `raglite_v2.ai_models` 表需预置 `embedding` / `analysis` / `chat` 三组模型配置

### 4. Wire 依赖注入错误

如果编译时报 "multiple bindings" 之类：

```bash
# 项目根目录跑
bash scripts/wire-auto-register.sh
cd backend && make generate
```

详见 `docs/注意事项/Wire-依赖注入多重绑定错误.md`。

## 与 Docker 全容器环境对比

| 项目 | dev 模式（本文） | local 模式（完整容器化） |
|---|---|---|
| 启动脚本 | `scripts/dev-services-cl.sh` | `scripts/local-services-cl.sh` |
| 后端服务 | 宿主机 `go run` | 容器内运行 |
| 配置文件来源 | `config.local.yml`（dev.localhost） | 容器内挂载 |
| PostgreSQL 地址 | `dev.localhost:13100` | `panda-wiki-postgres:5432`（容器间） |
| 反向代理 | 无（直连端口） | Caddy 8080/2443 |
| 热重载 | Go 直接重启 / Air | 需重建镜像 |
