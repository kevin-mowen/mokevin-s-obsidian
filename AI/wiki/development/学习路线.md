## 阶段 2 — 进阶机制（当前阶段）

按依赖顺序学，每条都是一个独立子系统：

- [ ] 🅒 **Redis 缓存策略**（轻热身）
  - 代码：`store/cache/redis.go` / `repo/cache/*.go` / 各 PG Repo 内嵌 cache
  - 关键：Cache-Aside、TTL、分布式锁、Session Store

- [ ] 🅐 **NATS 消息队列 + Consumer 进程**（重点）
  - 代码：`cmd/consumer/` / `mq/nats/` / `handler/mq/` / `repo/mq/`
  - 关键：双进程架构、Producer/Consumer、JetStream 持久化、消息确认重试

- [ ] 🅓 **MinIO / S3 文件存储**
  - 代码：`store/s3/` / `usecase/file.go`
  - 关键：预签名 URL、Bucket 策略、流式上传

- [ ] 🅔 **多租户 / 节点级权限**（修 Bug 准备）
  - 代码：`repo/pg/node_auth_user.go` / `node_group.go` / `middleware/share_auth.go`
  - 关键：树形权限继承、用户组、share token + JWT 双轨
  - 配套：修 `chat-permission-bypass` Bug

- [ ] 🅑 **RAG / Eino 大模型编排**
  - 代码：`store/rag/ct.go` / `usecase/llm.go` / `usecase/chat.go` / `usecase/vector_stats.go`
  - 关键：raglite SDK、文档分块、向量召回、Eino 链式编排、SSE 流式

- [ ] 🅕 **爬虫与多源导入**
  - 代码：`pkg/anydoc/` / `pkg/doc2md/` / `usecase/crawler.go` / `usecase/sync.go`
  - 关键：策略模式、断点续传、定时同步、markitdown-api 微服务
