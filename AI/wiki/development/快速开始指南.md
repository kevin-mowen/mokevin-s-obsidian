# PandaWiki 快速开始指南

本指南将帮助您在 10 分钟内快速搭建 PandaWiki 开发环境。

> **开发模式说明**：本文档描述的是 **dev 模式**（基础设施容器化 + 业务代码本地热重载），由 `scripts/dev-services-cl.sh` 管理。如需完整容器化 + Caddy 反向代理的模式，请看 `scripts/local-services-cl.sh`。

## 🎯 系统要求

### 必需软件
- **Docker Desktop**: 4.0+ (macOS) / Docker Engine 20.10+ (Linux)
- **Git**: 2.30+
- **Make**: 3.8+
- **sudo 权限**: 用于修改 hosts 文件

### 硬件要求
- **内存**: 最少 8GB，推荐 16GB
- **存储**: 最少 10GB 可用空间
- **CPU**: 支持 x64 架构

## 🚀 快速启动（3步）

### 第 1 步：克隆项目

```bash
# 克隆项目
git clone git@github.com:chaitin/PandaWiki.git
cd PandaWiki

# 检查项目结构
ls -la
```

### 第 2 步：配置域名

```bash
# 配置本地域名映射（需要 sudo 权限）
sudo bash ./scripts/dev-setup-hosts.sh
```

**脚本功能**：
- 向 `/etc/hosts` 添加 `dev.localhost` 及知识库子域名映射
- 自动检测已有配置，避免重复添加

### 第 3 步：启动开发环境

使用交互式脚本（基础服务走 Docker，业务服务 `go run` / `pnpm dev` 跑在宿主机）：

```bash
bash ./scripts/dev-services-cl.sh
```

在菜单中选择：
- **`20`** — 启动完整开发环境（基础服务 + Admin + API + Consumer + App）
- **`21`** — 仅启动 4 个本地业务服务（基础服务已在跑时）
- **`1`** — 仅启动 Docker 基础服务（PostgreSQL / Redis / NATS / MinIO / Qdrant / Raglite / Crawler）
- **`23`** — 停止所有服务
- **`30`** — 查看所有服务状态

**首次启动**需要拉取基础服务镜像和安装 Go/pnpm 依赖，约需 3–5 分钟。

## ✅ 验证安装

启动完成后，访问以下地址验证服务：

| 服务 | 地址 | 说明 |
|------|------|------|
| Admin 管理后台 | http://localhost:3000 | Vite 开发服务器 |
| App 前台 | http://localhost:3010 | Next.js 开发服务器 |
| Backend API | http://dev.localhost:18000 | Go API 服务 |
| MinIO 控制台 | http://dev.localhost:13601 | 对象存储管理 |

### 服务状态检查

```bash
# 通过脚本统一查看
bash ./scripts/dev-services-cl.sh   # 选 30

# 或直接查容器
docker ps --filter "name=panda-wiki"
```

正常情况应看到以下容器为 `Up` 状态：
`panda-wiki-postgres`、`panda-wiki-redis`、`panda-wiki-nats`、`panda-wiki-minio`、`panda-wiki-qdrant`、`panda-wiki-raglite`、`panda-wiki-crawler`。

## 🎯 创建第一个知识库

### 1. 访问管理后台

打开浏览器访问：http://localhost:2443

### 2. 注册管理员账号

- 首次访问需要注册账号
- 填写邮箱、用户名和密码
- 系统会自动分配管理员权限

### 3. 创建知识库

1. 登录后进入"知识库管理"
2. 点击"创建知识库"
3. 填写配置信息：

```
知识库名称: 我的知识库
域名: kb1.localhost          # 重要：使用配置的域名
端口: 8080                   # 重要：必须使用 8080
描述: 我的第一个知识库
```

4. 点击"保存"创建知识库

### 4. 访问知识库

创建完成后，访问：http://kb1.localhost:8080

## 🛠️ 开发工作流

### 前端开发

```bash
cd web/

# 安装依赖
pnpm install

# 启动开发服务器
pnpm dev
```

**开发服务器地址**：
- Admin 应用: http://localhost:3000
- App 应用: http://localhost:3010

### 后端开发

```bash
cd backend/

# 生成代码（Swagger、依赖注入）
make generate

# 运行代码检查
make lint

# 本地开发（需要先启动依赖服务）
make dev
```

### API 开发

修改 API 后，重新生成前端客户端：

```bash
# Admin 应用
cd web/admin/
pnpm api

# App 应用
cd web/app/
pnpm api
```

## 🔧 常用开发命令

### 环境管理

所有 dev 环境的起停、状态查询统一通过交互脚本完成：

```bash
bash ./scripts/dev-services-cl.sh
#   20 启动完整 / 23 全部停止 / 30 查看状态
```

### 日志查看

```bash
# 基础服务（Docker 容器）
docker logs -f panda-wiki-postgres
docker logs -f panda-wiki-raglite

# 或通过对应的 compose 文件
docker-compose -f docker/dev/docker-compose.raglite.yml logs -f

# 本地业务服务（Admin / App）
# 由 dev-services-cl.sh 在独立 Terminal 窗口启动，日志直接看对应窗口
```

### 数据库操作

```bash
# 连接数据库（容器名 panda-wiki-postgres，DB 名 panda-wiki）
docker exec -it panda-wiki-postgres psql -U panda-wiki -d panda-wiki

# 本地连接
psql -h dev.localhost -p 13100 -U panda-wiki -d panda-wiki

# 创建新的迁移
cd backend/
make migrate_sql SEQ_NAME=add_new_table
```

## 🚨 常见问题

### 问题 1: 端口被占用

**症状**: 启动时提示端口 3000 / 3010 / 18000 / 13xxx 被占用

**解决方案**:
```bash
# 查看端口占用（示例端口）
lsof -i :3000
lsof -i :18000
lsof -i :13100

# 脚本内置了 kill_port 逻辑，重新选择对应菜单项即可自动重启

# 或手动停止占用进程
kill -9 <PID>
```

### 问题 2: 域名无法访问

**症状**: 访问 kb1.localhost:8080 失败

**解决方案**:
```bash
# 重新配置域名
sudo bash dev-setup-hosts.sh

# 检查 hosts 文件
cat /etc/hosts | grep localhost

# 清除浏览器缓存
# Chrome: Ctrl+Shift+R (Windows/Linux) / Cmd+Shift+R (macOS)
```

### 问题 3: 服务启动失败

**症状**: 某个容器无法启动

**解决方案**:
```bash
# 查看详细日志
docker-compose -f docker/docker-compose.dev.yml logs <service_name>

# 重新构建镜像
docker-compose -f docker/docker-compose.dev.yml build <service_name>

# 重启特定服务
docker-compose -f docker/docker-compose.dev.yml restart <service_name>
```

### 问题 4: 前端构建失败

**症状**: pnpm install 或 pnpm build 失败

**解决方案**:
```bash
# 清除缓存
rm -rf node_modules
rm -rf .pnpm-store

# 重新安装
pnpm install

# 检查 Node.js 版本
node --version  # 应该是 18+ 或 20+
pnpm --version  # 应该是 8+
```
