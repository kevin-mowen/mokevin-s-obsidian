# 业务域总览（6 大域）

> Backend 39 个 domain 文件按业务职责切成 6 个子域，构成数据层骨架。

## 6 大业务域

| # | 业务域 | 主要 domain 文件 | 主要 PG 表 | 一句话懂 |
|---|---|---|---|---|
| **1** | [[1-身份与权限]] | `auth.go` `auth_group.go` `api_token.go` | `users` `auth_groups` `kb_users` `api_tokens` `system_api_tokens` | 一个人怎么被授权访问一个 kb 下的一个 node |
| **2** | [[2-知识库与文档]] ⭐核心 | `knowledge_base.go` `node.go` `app.go` | `knowledge_bases` `nodes` `node_releases` `apps` | node 的树形结构 + version + release 三个概念 |
| **3** | [[3-AI对话]] | `chat.go` `conversation.go` `llm.go` `model.go` | `conversations` `conversation_messages` `models` `prompts` | Chat 一次请求背后的 5 步 |
| **4** | [[4-内容接入]] | `crawler.go` `contribute.go` `file.go` `creation.go` | `contributes` `openapi_call_logs` | 外部内容怎么变成 node（4 条路径） |
| **5** | [[5-用户反馈]] | `comment.go` `stat.go` | `user_comment` `document_feedback` `stat_pages` `stat_page_hours` | feedback 与 stat 的分工 |
| **6** | [[6-系统设施]] | `license.go` `setting.go` `icon.go` `block_word.go`、公告 | `license` `settings` `block_words` `announcements` | 全局配置 vs kb 级配置 |

## 域间关系图

```
   [[1-身份与权限]]  ──── 鉴权 ────►  [[2-知识库与文档]]  ⭐
                                          ↓ 异步学习
                                       [[3-AI对话]]
                                          ↑ 检索
   [[4-内容接入]]  ──── 入库为 node ──►  [[2-知识库与文档]]
                                          ↓
                                       [[5-用户反馈]]

   [[6-系统设施]]  ──── 全局参数 ────►  上述所有域
```

## 表-子系统映射

```
前端 ─┬──► Admin 管 [1][2][3][4][5][6] 全部
      └──► App  只管 [2][3][5] 读侧

Backend API  ─┬──► handler/v1/      → 域 [1]-[6] 写侧
              ├──► handler/share/   → 域 [2][3][5] 读+用户动作
              └──► handler/openapi/ → 域 [2] 对外只读/写

Consumer ──► handler/mq/  → 域 [2] node 学习、[4] 异步导入、[1] CUC 同步
```

## 学习顺序建议

1. 先读 ⭐ [[2-知识库与文档]]（核心，其他都绕着它转）
2. 然后 [[1-身份与权限]]（决定谁能看什么）
3. 然后 [[3-AI对话]]（这是项目灵魂）
4. 最后 [[4-内容接入]] / [[5-用户反馈]] / [[6-系统设施]]
