# 业务域 4　内容接入

> 外部内容怎么变成 [[2-知识库与文档|node]]。共 4 条路径。

## 4 条路径

| 路径 | 触发方式 | 实现 | 中间服务 |
|---|---|---|---|
| **手动创建** | Admin 富文本编辑器 | usecase/node.go 直接写 | 无 |
| **爬虫导入** | URL / Sitemap / RSS | usecase/crawler.go | [[Crawler]] |
| **文件导入** | 上传 .md/.docx/.pdf/.zip 等 | usecase/file.go + usecase/crawler.go | doc2md / markitdown（宿主机 :8490） |
| **贡献提交** | 用户在前台 App 提建议 | usecase/contribute.go | 走审批后入 node |

## 域内 domain 文件

| 文件 | 职责 |
|---|---|
| `crawler.go` | URL/sitemap/RSS 抓取任务 |
| `contribute.go` | 用户贡献的修改建议 |
| `file.go` | 文件上传/下载 |
| `creation.go` | AI 创作辅助 |

## 主要 PG 表

```
contributes                贡献提交记录
openapi_call_logs          OpenAPI 调用审计（属于"对外接入"）
（文件元数据散在 nodes 表的 file_url 字段）
```

## 文件上传链路

```
浏览器选文件
   ↓
API: POST /api/v1/file/upload (handler/v1/file.go)
   ↓
写 MinIO + 调 doc2md 转 Markdown（如非 .md）
   ↓
发 NATS：import.file
   ↓
Consumer 收到：解析 + 入库为 node
   ↓
node.release → raglite 学习
```
