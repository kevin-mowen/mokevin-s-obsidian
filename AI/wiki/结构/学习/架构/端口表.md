# 端口表（dev 模式权威）

> 文档冲突时一律以本表为准。每行都标注权威来源。

## 业务进程（宿主机）

| 服务 | 地址 | 来源 |
|---|---|---|
| Admin | `http://localhost:3000` | `web/admin/vite.config.ts:57` |
| App | `http://localhost:3010` | `web/app/package.json` → `next dev -p 3010` |
| Backend API | `http://dev.localhost:18000` | `config.local.yml:8 http.port` |
| Consumer | 无端口 | `go run ./cmd/consumer` 后台进程 |

## 宿主机外挂服务（容器外）

| 服务 | 地址 | 用途 |
|---|---|---|
| Ollama | `http://localhost:11434` | 模型推理（embedding / chat） |
| one-api | `http://localhost:3002` 或 `:4000` | OpenAI 协议聚合网关，给 raglite 的 chat 用 |
| doc2md / markitdown | `http://localhost:8490` | Word/PDF → MD 转换 |
| FastGPT 推送 | `http://localhost:4000` | 可选，消息推送 |
| CUC | `http://appstore.oa.cib:80` | 公司内网，组织架构同步 |

## 易错点

- ❌ Raglite v2.x base_url 不带 `/api/v1`：`http://dev.localhost:13400` 而不是 `:13400/api/v1`
- ❌ 用 `localhost` 访问基础服务 → 应是 `dev.localhost`（依赖 hosts 配置）
- ❌ 看到 `:8080` 当成 dev 模式 → 那是 [[dev 模式与 local 模式|local 模式]]

## 配置 hosts

```bash
sudo bash ./scripts/dev-setup-hosts.sh
# 写入 /etc/hosts: 127.0.0.1 dev.localhost
```
