## 第一层：核心概念速查表

| 概念         | 定义                          | 记忆技巧                  | 易混点                   |
| ---------- | --------------------------- | --------------------- | --------------------- |
| OSI 七层     | 物理、数据链路、网络、传输、会话、表示、应用      | 物数网传会表应               | 不要和 TCP/IP 四层混淆       |
| TCP/UDP    | TCP 可靠、有序、低效；UDP 快速、无序、丢包   | TCP=可靠，UDP=快速         | TCP 三次握手，UDP 无连接      |
| IP 地址      | 32 位标识符，用于网络层定位设备           | A 类 1-126，B 类 128-191 | 广播地址和网络地址的区别          |
| 网络位（Net Bits） | IP 地址中用于标识网络的位数，从左起数 | 用 /n 表示（如 /24 = 24 位网络） | 网络位 + 主机位 = 32 |
| 主机位（Host Bits） | IP 地址中用于标识主机的位数，从右起数 | 主机位 = 32 - 网络位 | 主机位全 0=网络地址，全 1=广播地址 |
| 可用主机数 | 一个子网中能分配给实际主机的 IP 数量 | **2^主机位 - 2**（减去网络+广播） | 一定要减 2，不能少 |
| DNS        | 域名解析服务，将域名转换为 IP            | Domain Name System    | DNS 工作在应用层（第 7 层）     |
| HTTP/HTTPS | HTTP 无加密，HTTPS 有 SSL/TLS 加密 | HTTPS = HTTP + SSL    | 端口号：HTTP 80，HTTPS 443 |
| 端口         | TCP/UDP 的逻辑通道，0-65535       | 常见端口：80、443、3306      | 端口号是 16 位无符号整数        |
| 子网掩码（Netmask） | 32 位数字，区分网络位和主机位 | /24 = 255.255.255.0 | 掩码中 1 对应网络位，0 对应主机位 |
| 网络地址 | 整个子网的标识，主机位全 0 | IP AND 掩码 = 网络地址 | 网络地址不能分配给主机 |
| 广播地址 | 子网内广播用地址，主机位全 1 | 网络地址 + (2^主机位 - 1) | 广播地址不能分配给主机 |

## 第二层：常考公式 / 对比表

### 主机位计算 - 最核心公式

| 项目 | 公式 | 说明 | 例子 |
|------|------|------|------|
| **主机位** | 32 - 网络位 | IP 地址共 32 位 | /24 → 32-24=8 位 |
| **可用主机数** | 2^主机位 - 2 | 减 2：网络地址、广播地址 | 2^8 - 2 = 254 |
| **网络地址** | IP AND 掩码 | 主机位全 0 | 192.168.1.100 AND 255.255.255.0 = 192.168.1.0 |
| **广播地址** | 网络地址 + (2^主机位 - 1) | 主机位全 1 | 192.168.1.0 + 255 = 192.168.1.255 |

### 掩码与主机位对照表（必背！）

| 掩码表示 | 十进制表示 | 网络位 | 主机位 | 可用主机数 | 常见用途 |
|--------|----------|--------|--------|-----------|---------|
| /24 | 255.255.255.0 | 24 | **8** | 254 | 小型网络（最常用） |
| /25 | 255.255.255.128 | 25 | 7 | 126 | 子网划分 |
| /26 | 255.255.255.192 | 26 | 6 | 62 | 子网划分 |
| /27 | 255.255.255.224 | 27 | 5 | 30 | 子网划分 |
| /28 | 255.255.255.240 | 28 | 4 | 14 | 小型子网 |
| /29 | 255.255.255.248 | 29 | 3 | 6 | 点对点链路 |
| /30 | 255.255.255.252 | 30 | 2 | 2 | 路由器互连（VPN） |

### TCP vs UDP 对比

| 特性   | TCP           | UDP       |
| ---- | ------------- | --------- |
| 连接   | 需要三次握手        | 无连接       |
| 可靠性  | 有序、无丢包        | 无序、可能丢包   |
| 速度   | 慢（需要确认）       | 快         |
| 流量控制 | 有             | 无         |
| 应用   | FTP、HTTP、SMTP | DNS、视频、游戏 |

### OSI 七层 vs TCP/IP 四层

| OSI 七层 | TCP/IP 四层 | 典型协议 |
|--------|-----------|--------|
| 应用 + 表示 + 会话 | 应用层 | HTTP、HTTPS、DNS、FTP、SMTP |
| 传输 | 传输层 | TCP、UDP |
| 网络 | 网络层 | IP、ICMP、ARP |
| 数据链路 + 物理 | 链路层 | Ethernet、WiFi |

### IP 地址分类

| 类别 | 范围 | 默认子网掩码 | 用途 |
|------|------|----------|------|
| A 类 | 1.0.0.0 - 126.255.255.255 | 255.0.0.0 | 大型网络 |
| B 类 | 128.0.0.0 - 191.255.255.255 | 255.255.0.0 | 中型网络 |
| C 类 | 192.0.0.0 - 223.255.255.255 | 255.255.255.0 | 小型网络 |
| D 类 | 224.0.0.0 - 239.255.255.255 | — | 多播 |
| E 类 | 240.0.0.0 - 255.255.255.255 | — | 保留 |

## 第三层：高频题型示例

### 题型 0️⃣：计算主机位和可用主机数（超高频！）

**题目示例**：IP = 192.168.1.100，掩码 = 255.255.255.224，可用主机数是多少？

**答案框架**：
```
第 1 步：确定掩码类型
掩码：255.255.255.224
最后一位：224 = 11100000（二进制）
1 的个数：3 个
前三个 255 贡献：24 位
总网络位：24 + 3 = 27 位（/27）

第 2 步：计算主机位
主机位 = 32 - 27 = 5 位

第 3 步：计算可用主机数
可用主机数 = 2^5 - 2 = 32 - 2 = 30 个 ✅

答案：30 个可用主机
```

**易错点**：
- ❌ 错误：2^5 = 32 个（忘记减 2）
- ✅ 正确：2^5 - 2 = 30 个（要减网络地址和广播地址）

**相似题**：2019年真题、2021年真题、2023年真题都考过

---

### 题型 1️⃣：掩码位数推算（高频）

**题目示例**：掩码 /26，主机位是多少？可容纳多少个主机？

**答案框架**：
```
掩码：/26
主机位 = 32 - 26 = 6 位
可容纳主机数 = 2^6 - 2 = 62 个

答案：6 位主机，62 个可用主机
```

**速记**：
- /24 → 8 位主机 → 254 个
- /25 → 7 位主机 → 126 个
- /26 → 6 位主机 → 62 个
- /27 → 5 位主机 → 30 个
- /28 → 4 位主机 → 14 个

### 题型 2️⃣：TCP 三次握手

**题目示例**：TCP 三次握手的目的是什么？

**核心考点**：
- 第一次握手：客户端 → 服务器，发送 SYN
- 第二次握手：服务器 → 客户端，发送 SYN + ACK
- 第三次握手：客户端 → 服务器，发送 ACK

**目的**：
- 建立连接
- 同步序列号
- 验证两端都可通信

**相似题**：2024 年真题第 X 题，2023 年真题第 Y 题

---

### 题型 3️⃣：IP 地址和子网掩码

**题目示例**：给定 IP 地址 192.168.1.100，子网掩码 255.255.255.0，求网络地址和广播地址。

**核心考点**：
- 网络地址 = IP & 子网掩码 = 192.168.1.0
- 广播地址 = 网络地址 + (2^(32-前缀长度) - 1) = 192.168.1.255
- 可用主机数 = 2^8 - 2 = 254

**易混点**：不要把广播地址当成网络地址

**相似题**：常见题

---

### 题型 4️⃣：DNS 和 HTTP

**题目示例**：访问网站 www.example.com 的过程中，哪个协议负责域名解析？

**答案**：DNS（工作在应用层，使用 UDP 53 端口）

**相关知识**：
- HTTP（80）：网页传输
- HTTPS（443）：加密网页传输
- DNS（53）：域名解析
- FTP（21）：文件传输

**易混点**：DNS 工作在应用层，但使用 UDP，不是 TCP

---

### 题型 5️⃣：网络安全基础

**题目示例**：VPN 的作用是什么？防火墙呢？

**答案**：
- VPN：虚拟专用网络，在公网上建立加密隧道
- 防火墙：检查和过滤网络流量，防止非授权访问

**易混点**：VPN 是加密通道，防火墙是过滤器

### 题型 6️⃣：MAC 地址与 ARP 协议

**题目示例**：MAC 地址和 IP 地址的区别是什么？ARP 协议的作用？

**核心考点**：
- MAC 地址：物理地址，48 位，工作在数据链路层，标识局域网内的设备
- IP 地址：逻辑地址，32 位（IPv4），工作在网络层，标识整个网络中的设备
- ARP：地址解析协议，将 IP 地址解析成 MAC 地址

**答案框架**：
```
MAC 地址 = 本地网络内通信地址（数据链路层）
IP 地址 = 跨网络通信地址（网络层）
ARP = 从 IP 地址查询 MAC 地址的桥梁
```

**易混点**：同一局域网内，MAC 地址决定直接通信；跨网络要靠 IP 地址和路由器

---

### 题型 7️⃣：网络设备的 OSI 层级

**题目示例**：集线器、交换机、路由器、网关分别工作在哪一层？

**答案框架**：
```
集线器 = 第 1 层（物理层）= 转发电信号
交换机 = 第 2 层（数据链路层）= 根据 MAC 地址转发
路由器 = 第 3 层（网络层）= 根据 IP 地址转发
网关 = 第 7 层（应用层）= 协议转换、数据翻译
```

**记忆技巧**：集线→交换→路由，层级逐级上升

**易混点**：不要把路由器和交换机混淆（路由器跨网络，交换机只在局域网内）

---

### 题型 8️⃣：DHCP 协议与自动分配

**题目示例**：DHCP 的作用是什么？如何实现动态 IP 分配？

**核心考点**：
- DHCP = 动态主机配置协议
- 功能：自动分配 IP、网关、DNS、子网掩码等配置
- 四步过程：Discover → Offer → Request → Acknowledge

**答案框架**：
```
DHCP 客户端 → 广播 DHCP Discover（寻找 DHCP 服务器）
DHCP 服务器 → 回复 Offer（提供可用 IP）
DHCP 客户端 → 发送 Request（确认要这个 IP）
DHCP 服务器 → 回复 Acknowledge（确认分配完成）
```

**易混点**：DHCP 和 DNS 不同，DHCP 分配 IP，DNS 解析域名

---

### 题型 9️⃣：IPv6 基本概念

**题目示例**：IPv6 相比 IPv4 有什么优势？IPv6 地址如何表示？

**核心考点**：
- IPv4：32 位，44 亿地址，快要耗尽
- IPv6：128 位，地址近乎无限
- IPv6 格式：十六进制冒号分隔（如 2001:0db8::1）

**答案框架**：
```
IPv4 地址空间 = 2^32 = 43 亿（已接近耗尽）
IPv6 地址空间 = 2^128 = 数百万亿亿个地址（足用几百年）
IPv6 还支持：自动配置、更好的安全性、更高效的路由
```

**易混点**：IPv6 不是简单的"扩展版本"，是全新的设计思想

---

### 题型 🔟：路由与路由协议

**题目示例**：RIP、OSPF、BGP 的区别？

**核心考点**：
- **RIP**（路由信息协议）：距离矢量，最大跳数 15，收敛慢，适合小网络
- **OSPF**（开放最短路径优先）：链路状态，收敛快，适合大网络
- **BGP**（边界网关协议）：用于 AS 间通信，互联网骨干路由协议

**对比表**：

| 协议 | 类型 | 最大跳数 | 收敛速度 | 应用范围 |
|------|------|--------|--------|--------|
| RIP | 距离矢量 | 15 | 慢 | 小网络 |
| OSPF | 链路状态 | 无限制 | 快 | 大型企业网络 |
| BGP | 路径矢量 | 无限制 | 中等 | 互联网、ISP |

**易混点**：RIP 最简单但最效率低，OSPF 成本高但效率高，BGP 用于自治系统间
