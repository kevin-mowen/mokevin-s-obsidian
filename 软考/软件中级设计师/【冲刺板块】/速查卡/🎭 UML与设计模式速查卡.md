## 第一层：UML图型速查

| 图型 | 用途 | 表示元素 | 易混点 |
|------|------|--------|--------|
| 用例图 | 描述系统与用户的交互 | 用例(椭圆)、参与者(人形)、关系线 | 用例不等于功能 |
| 类图 | 描述类的结构和关系 | 类(矩形)、属性、方法、关系(泛化/实现/关联) | 关系方向很重要 |
| 时序图 | 描述对象间的交互顺序 | 对象、生命线、消息 | 强调时间顺序 |
| 活动图 | 描述流程和控制流 | 活动(圆角矩形)、判断(菱形)、同步(横杠) | 类似流程图 |
| 状态图 | 描述对象的状态变迁 | 状态(圆角矩形)、转移(箭头) | 状态vs流程 |
| 组件图 | 描述系统的物理结构 | 组件、接口、依赖关系 | 物理视角 |
| 部署图 | 描述系统的部署情况 | 节点、制品、连接 | 运行时视角 |

## 第二层：UML关系速查

```
关系强度从弱到强：
    依赖关系 < 关联关系 < 聚合关系 < 组合关系 < 继承关系 < 实现关系
    (Dependency)  (Association) (Aggregation) (Composition) (Generalization) (Realization)
```

| 关系 | 符号 | 含义 | 例子 |
|------|------|------|------|
| 泛化 | △← | 继承，is-a | 学生 is-a 人 |
| 实现 | △⫶← | 实现接口 | Dog implements Animal |
| 关联 | →← | 静态关系，has-a | 班级 has-a 学生 |
| 聚合 | ◇← | 弱拥有关系 | 团队 has 成员 |
| 组合 | ◆← | 强拥有关系 | 人 has 心脏 |
| 依赖 | ⋯→ | 临时关系，use-a | 方法中使用某个类 |

## 第三层：23种设计模式速查

### 创建型模式（5个）
- **单例模式**：保证类只有一个实例
- **工厂模式**：通过工厂方法创建对象，隐藏创建逻辑
- **抽象工厂**：创建相关对象族
- **建造者模式**：分步骤构建复杂对象
- **原型模式**：通过克隆创建对象

### 结构型模式（7个）
- **适配器**：接口不兼容时进行转换
- **装饰者**：动态添加功能
- **外观**：提供子系统的统一接口
- **享元**：共享对象以节省内存
- **代理**：代理对象控制对真实对象的访问
- **桥接**：将抽象与实现分离
- **组合**：树形结构中统一处理叶节点和内部节点

### 行为型模式（11个）
- **观察者**：定义对象间一对多的依赖关系
- **策略**：定义一组算法，使用时互相替换
- **命令**：将请求封装为对象
- **状态**：根据状态改变行为
- **模板方法**：定义算法框架，具体实现延迟到子类
- **迭代器**：顺序访问集合中的元素
- **中介者**：多个对象通过中介通信
- **备忘录**：保存对象状态，支持回滚
- **责任链**：将请求沿链传递
- **访问者**：对集合中的元素执行操作
- **解释器**：为语言定义语法和解释器

## 常考设计模式详解

### 单例模式
```
应用场景：数据库连接、日志系统、配置管理
实现：
  class Singleton {
    private static instance = null
    private Singleton() {}
    public static getInstance() {
      if instance == null:
        instance = new Singleton()
      return instance
    }
  }
特点：线程安全、全局唯一访问点
```

### 观察者模式
```
应用场景：事件系统、发布-订阅
角色：Subject(主题)、Observer(观察者)
工作流：
  1. Observer向Subject注册
  2. Subject状态改变
  3. Subject通知所有Observer
特点：松耦合、支持事件驱动
```

### 策略模式
```
应用场景：算法选择、支付方式选择
角色：Strategy(策略)、Context(上下文)
工作流：
  1. 定义一组算法
  2. 运行时选择算法
  3. Context使用选中的Strategy
特点：避免大量if-else、易于扩展
```

### 适配器模式
```
应用场景：接口转换、兼容旧系统
例子：Type-C转USB-A适配器
作用：使接口不兼容的类能一起工作
```

### 装饰者模式
```
应用场景：功能扩展、Java IO流
例子：
  FileInputStream fis = new FileInputStream("file.txt")
  BufferedInputStream bis = new BufferedInputStream(fis)
  特点：动态添加功能，无需修改原类
```

### 工厂模式
```
应用场景：对象创建、数据库连接
简单工厂：一个工厂方法
工厂方法：每个产品一个工厂类
抽象工厂：创建相关产品族
好处：隐藏创建逻辑、易于维护
```
