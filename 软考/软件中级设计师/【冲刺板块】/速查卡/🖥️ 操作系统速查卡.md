## 第一层：核心概念速查表

| 概念 | 定义 | 记忆技巧 | 易混点 |
|------|------|--------|-------|
| 进程 | 程序的执行实例，有独立的内存空间 | Process = 资源分配的最小单位 | 不等于程序 |
| 线程 | 进程内的执行单位，共享内存空间 | Thread = CPU 调度的最小单位 | 线程更轻量 |
| PCB | 进程控制块，记录进程的所有信息 | Process Control Block | 包含 PID、状态、寄存器等 |
| 进程状态 | 就绪、运行、阻塞、创建、终止 | 五态模型 | 不是三态 |
| 死锁 | 多个进程互相等待资源而永远阻塞 | Deadlock | 四个必要条件 |
| 信号量 | 用于同步和互斥的整数变量 | Semaphore：P、V 操作 | P = 申请，V = 释放 |
| 虚拟内存 | 用辅助存储扩展主存的逻辑内存 | Virtual Memory | 包括分页、分段 |

## 第二层：常考公式 / 对比表

### 进程状态转移图

```
创建 → 就绪 → 运行 → 阻塞 → 就绪
              ↓
           终止
```

### 进程调度算法对比

| 算法 | 公平性 | 响应时间 | 吞吐量 | 用途 |
|------|------|--------|------|------|
| FCFS | 不公平 | 差 | 一般 | 单任务系统 |
| RR | 公平 | 中等 | 中等 | 分时系统 |
| 优先级 | 低 | 快 | 高 | 实时系统 |
| 多级反馈 | 公平 | 快 | 高 | 通用系统 |

### 死锁的四个必要条件

1. **互斥**：资源不能共享
2. **占有且等待**：占有资源同时等待其他资源
3. **不可剥夺**：资源不能被强制收回
4. **循环等待**：进程之间形成环形等待

**破坏任一条件即可避免死锁**

### 内存管理方式对比

| 方式 | 优点 | 缺点 | 应用 |
|------|-----|------|------|
| 分页 | 消除外碎片 | 有内碎片 | 现代系统 |
| 分段 | 保护方便 | 有外碎片 | 特定系统 |
| 虚拟内存 | 扩展内存容量 | 性能下降 | 通用系统 |

## 第三层：高频题型示例

### 题型 1：进程调度

**题目示例**：在分时系统中，为什么采用时间片轮转调度而不是先来先服务？

**答案**：因为 FCFS 不能给所有进程公平的响应时间，而 RR 可以。

**相关知识**：时间片过长 = 退化为 FCFS；过短 = 上下文切换开销大

---

### 题型 2：死锁

**题目示例**：检测死锁的银行家算法思路是什么？

**答案**：模拟资源分配，看是否存在安全序列

**易混点**：死锁预防 vs 死锁避免 vs 死锁检测

---

### 题型 3：同步与互斥

**题目示例**：用信号量实现临界区互斥的 P、V 操作。

**答案**：
```
P(mutex)       // 申请信号量，值减 1
// 临界区
V(mutex)       // 释放信号量，值加 1
```

---

### 题型 4：虚拟内存

**题目示例**：页面置换算法中，LRU 和 FIFO 的区别是什么？

**答案**：
- FIFO：置换最早进入的页面
- LRU：置换最久未使用的页面（性能更好）
