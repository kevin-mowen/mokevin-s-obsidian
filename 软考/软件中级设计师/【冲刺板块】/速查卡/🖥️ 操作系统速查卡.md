## 第一层：核心概念速查表

| 概念      | 定义                         | 记忆技巧                  | 易混点                |
| ------- | -------------------------- | --------------------- | ------------------ |
| 进程      | 程序的执行实例，有独立的内存空间           | Process = 资源分配的最小单位   | 进程是动态的，程序是静态的      |
| 线程      | 进程内的执行单位，共享内存空间            | Thread = CPU 调度的最小单位  | 线程比进程轻，更快切换        |
| 进程状态    | 就绪、运行、阻塞、创建、终止             | 就绪 → 运行 → 阻塞 → 就绪 循环  | 就绪≠运行（就绪还没获得 CPU）  |
| 上下文切换   | CPU 从一个进程切换到另一个的过程         | 保存当前进程状态，加载新进程状态      | 切换成本高（缓存污染、TLB 失效） |
| CPU 调度  | 操作系统决定哪个进程使用 CPU           | 目标：公平、高效、响应快          | 不同调度算法优化不同目标       |
| 死锁      | 多个进程互相等待资源而永远阻塞            | 四个必要条件都满足才会发生         | 死锁 vs 饥饿的区别        |
| 信号量     | 用于同步和互斥的整数变量               | P(down)/V(up) 操作      | 二值信号量 = 互斥锁        |
| 虚拟内存    | 用磁盘扩展内存容量的技术               | 分页或分段 + 页面置换算法        | 虚拟地址可大于物理内存        |
| 页面置换    | 物理内存满时，将页面换出到磁盘            | LRU、FIFO、OPT 等算法      | 缺页中断 = 页面不在内存中     |
| 磁盘调度    | 决定磁盘访问顺序以最小化寻道时间           | FCFS、SSTF、SCAN、C-SCAN | 电梯调度（SCAN）最常考      |
| 文件系统    | 管理存储设备上的数据和目录              | 索引节点（inode）+ 块（block） | FAT vs inode 的区别   |
| 中断与异常   | 中断：外部事件；异常：CPU 内部问题        | 中断处理是异步的，异常是同步的       | 中断服务程序必须快速完成       |
| 缓存与 TLB | L1/L2/L3 缓存加速数据；TLB 加速地址翻译 | 工作在硬件层，上下文切换时失效       | TLB 是硬件结构，不是内存页表   |

## 第二层：常考公式 / 对比表

### 进程调度算法对比（必背！）

| 算法 | 规则 | 优点 | 缺点 | 适用 |
|------|------|------|------|------|
| FCFS | 先来先服务 | 简单公平 | 平均等待时间长 | 批处理系统 |
| SJF | 最短任务优先 | 平均等待时间最短 | 长任务可能饥饿 | 已知任务时长 |
| RR | 时间片轮转 | 交互响应好，防饥饿 | 切换成本高 | 分时系统（常用） |
| 优先级 | 按优先级调度 | 可对应不同需求 | 低优先级进程饥饿 | 实时系统 |

**核心差异表**：

| 指标 | FCFS | SJF | RR | 优先级 |
|------|------|-----|-----|--------|
| 平均等待时间 | 长 | **最短** | 中 | 可调整 |
| 响应时间 | 差 | 差 | **快** | 快 |
| 公平性 | 公平 | 不公平 | **公平** | 不公平 |

### 页面置换算法对比

| 算法 | 规则 | 缺页数 | 实现难度 |
|------|------|--------|--------|
| FIFO | 最先进入的先换出 | 较多 | 简单 |
| LRU | 最久未使用的换出 | **最少** | 中等 |
| OPT | 最久不用的换出 | 最少（理想） | 难（需预知） |

**记忆技巧**：LRU 最实用（接近最优），FIFO 最简单，OPT 只在理论中

### 死锁的四个必要条件与破坏

```
必要条件全部满足 → 死锁发生
缺少任何一个条件 → 不会死锁
```

| 条件        | 定义             | 破坏方法         |
| --------- | -------------- | ------------ |
| **互斥**    | 资源同时只能一个进程使用   | 难以破坏（资源本身特性） |
| **持有和等待** | 进程持有资源同时等待其他资源 | 一次分配所有需要的资源  |
| **不可抢占**  | 资源不能被强制夺取      | 允许进程抢占资源     |
| **循环等待**  | 进程形成资源等待的环形链   | 按序列申请资源      |

### 内存管理方式对比

| 方式 | 优点 | 缺点 | 应用 |
|------|-----|------|------|
| 分页 | 消除外碎片 | 有内碎片（平均 0.5 页大小） | 现代系统 |
| 分段 | 逻辑清晰，保护方便 | 有外碎片 | 特定系统 |
| 虚拟内存 | 扩展内存容量，隔离进程 | 性能下降、增加复杂度 | 通用系统 |

### 内存组织结构（从高地址到低地址）

```
高地址
  ↓
栈（Stack）      - 局部变量、返回地址、函数参数（向下增长）
  ↓↓↓
未使用空间
  ↑↑↑
堆（Heap）       - 动态分配内存（向上增长）
  ↑
未初始化数据段（BSS）  - 全局/静态变量（未初始化）
  ↑
初始化数据段      - 全局/静态变量（已初始化）
  ↑
代码段（Text）    - 程序指令（只读）
低地址
```

## 第三层：高频题型示例

### 题型 0️⃣：进程状态转换

**题目示例**：进程从阻塞态变回运行态需要几步？

**答案框架**：
```
阻塞态 → 就绪态 → 运行态（需要两步）
        ↑       ↑
    事件发生   调度器选中

错误认识：阻塞态直接 → 运行态（不对！必须经过就绪态）
```

**记忆技巧**：
- 只有**就绪态**能被调度器选中进入运行态
- 阻塞态的进程再优秀也要先等待事件完成

### 题型 1️⃣：CPU 调度算法计算

**题目示例**：已知任务时长分别为 8, 4, 2, 6 ms，用 RR（时间片 4ms）计算平均等待时间。

**答案框架**：
```
时间轴图示：
0-4:   P1(剩余4)    等待时间 = 完成时间 - 任务长度
4-8:   P2(剩余0)
8-10:  P3(完成)     平均等待时间 = (总等待时间) / (任务数)

核心步骤：
1. 画出完整的时间轴
2. 计算每个进程的等待时间
3. 求平均值
```

**易错点**：
- ❌ 混淆等待时间和周转时间
- ✅ 等待时间 = 进程完成时间 - 任务长度

### 题型 2️⃣：死锁判断与破坏

**题目示例**：系统中有 3 个进程和 2 种资源，每个进程都持有 1 个资源等待另外 1 个，是否形成死锁？

**答案框架**：
```
检查四个必要条件：
✓ 互斥：资源不能共享
✓ 持有和等待：已持有资源，又等待新资源
✓ 不可抢占：不能强制夺取
✓ 循环等待：P1→P2→P3→P1 形成环

结论：全部满足 → 死锁发生！

破坏方法选择（选一个）：
1. 允许资源抢占（不现实）
2. 进程一次性申请所有资源（简单）
3. 按序申请资源（避免循环）
```

**易混点**：死锁 vs 饥饿
- 死锁：相互等待，都无法执行
- 饥饿：某个进程一直得不到资源，但其他进程可以执行

### 题型 3️⃣：页面置换算法

**题目示例**：内存只能装 3 页，访问序列 1,2,3,4,2,1,5,2,1,3，用 LRU 计算缺页数。

**答案框架**：
```
初始化：内存空 []

按序访问，记录何时缺页：
- 访问 1: [1] 缺页（计数 1）
- 访问 2: [1,2] 缺页（计数 2）
- 访问 3: [1,2,3] 缺页（计数 3，满）
- 访问 4: [4,2,3] 缺页（计数 4，换出最久未用的 1）
- 访问 2: [4,2,3] 无缺页（2 在内存）
- 访问 1: [4,1,3] 缺页（计数 5，换出最久未用的 2）
- 访问 5: [5,1,3] 缺页（计数 6，换出最久未用的 4）
- 访问 2: [5,1,2] 缺页（计数 7，换出最久未用的 3）
- 访问 1: [5,1,2] 无缺页（1 在内存）
- 访问 3: [5,3,2] 缺页（计数 8，换出最久未用的 1）

总缺页数 = 8
```

**记忆技巧**：LRU = "最后用过最久的淘汰"，每次访问更新"最后使用时间"

### 题型 4️⃣：信号量与互斥

**题目示例**：两个进程使用信号量同步，S 初值为 1。P(S) 和 V(S) 的作用是什么？

**答案框架**：
```
信号量 S 初值 = 1（二值信号量 = 互斥锁）

P(S)（申请，值减 1）：
  if S > 0:
    S--      （获取资源）
  else:
    wait     （等待其他进程释放）

V(S)（释放，值加 1）：
  S++        （释放资源）
  wake up    （唤醒一个等待的进程）

效果：同一时刻只有一个进程进入临界区（互斥）
```

**易混点**：P/V 的顺序不能颠倒，P 必须在前，V 必须在后

### 题型 5️⃣：虚拟内存与页表

**题目示例**：32 位系统，页大小 4KB，一张页表最多能映射多大内存？

**答案框架**：
```
已知：
- 虚拟地址空间 = 32 位 = 2^32 字节 = 4GB
- 页大小 = 4KB = 2^12 字节

计算：
- 页的个数 = 虚拟空间 / 页大小 = 2^32 / 2^12 = 2^20 = 100 万页
- 每个页表项大小 ≈ 4-8 字节
- 页表大小 = 100 万 × 4 字节 = 4MB

答案：一张完整页表大小为 4MB，可映射整个 4GB 虚拟空间
```

**易混点**：
- 页大小决定虚拟地址中"页内偏移"的位数
- 页表项数量 = 2^(虚拟地址位数 - 页内偏移位数)

### 题型 6️⃣：磁盘调度

**题目示例**：磁盘磁头初始在 100，访问顺序 55, 160, 10, 185, 70，用 SSTF 计算总寻道时间。

**答案框架**：
```
SSTF = 最短寻道时间优先（贪心算法）

步骤：
1. 当前位置：100
2. 距离最近：70（|100-70|=30）→ 寻道时间 30
3. 当前位置：70，距离最近：55（|70-55|=15）→ 寻道时间 15
4. 当前位置：55，距离最近：10（|55-10|=45）→ 寻道时间 45
5. 当前位置：10，距离最近：160（|10-160|=150）→ 寻道时间 150
6. 当前位置：160，距离最近：185（|160-185|=25）→ 寻道时间 25

总寻道时间 = 30 + 15 + 45 + 150 + 25 = 265

对比 SCAN（电梯调度）：
- SSTF：贪心，某些请求可能被"饿死"
- SCAN：来回扫描，更公平，通常更优
```

### 题型 7️⃣：缓存与 TLB

**题目示例**：CPU 缓存和页表缓存（TLB）的作用是什么？上下文切换为什么成本高？

**答案框架**：
```
L1/L2/L3 缓存：
  - 加速数据访问（速度：L1 > L2 > L3 > 内存）
  - 减少内存访问延迟（10 个 CPU 周期 → 1 个周期）
  - 工作在硬件层，与进程无关

TLB（Translation Lookaside Buffer）：
  - 缓存虚拟地址 → 物理地址的映射
  - 加速地址翻译（不用查页表）
  - 与进程相关（ASID 标记）
  - 上下文切换时需要刷新（FLUSH）
  
上下文切换成本 = 
  1. 缓存污染（L1/L2/L3 被新进程覆盖）
  2. TLB 失效（需要重新加载映射）
  3. 保存/恢复寄存器和页表基址等
  4. 操作系统开销（1000+ CPU 周期）
```

### 题型 8️⃣：文件系统与 inode

**题目示例**：inode 节点包含哪些信息？为什么要用 inode 而不是 FAT？

**答案框架**：
```
inode 包含：
✓ 文件大小（字节数）
✓ 文件权限（rwx：所有者、组、其他）
✓ 所有者/组信息（UID、GID）
✓ 修改/访问时间（mtime、atime）
✓ 链接计数（硬链接个数）
✓ 指向数据块的指针（直接指针 + 间接指针）
  例：12 个直接指针 + 1 个一级间接 + 1 个二级间接 + 1 个三级间接
✗ 文件名（存在目录项中，不在 inode）

inode vs FAT：
```

| 特性 | inode | FAT |
|------|-------|-----|
| 访问方式 | 随机访问（快） | 链式访问（慢，需遍历 FAT） |
| 大文件支持 | 支持（间接指针） | 支持但低效 |
| 空间浪费 | 低（每文件一个 inode） | 低（FAT 表集中） |
| 实现复杂度 | 中等 | 简单 |

**易混点**：inode 本身不存储文件名，文件名存在目录项中

### 题型 9️⃣：中断与异常处理

**题目示例**：中断和异常的区别是什么？中断优先级如何管理？

**答案框架**：
```
中断（Interrupt）：
  - 来源：外部设备（网卡、磁盘、键盘）
  - 时机：异步发生（任意时刻）
  - 可屏蔽：可以关闭中断（CLI/STI 指令）
  - 示例：时钟中断（定时器）、I/O 中断

异常（Exception）：
  - 来源：CPU 内部（指令执行错误）
  - 时机：同步发生（执行某条指令时）
  - 不可屏蔽：必须处理
  - 示例：除零错、页缺故、权限错误

中断优先级：
  - 高优先级中断 ✓ 可打断低优先级中断（嵌套）
  - 同优先级按顺序处理
  - 中断服务程序要快（1-100 微秒）
  - 复杂处理→下半部/软中断（延迟处理）
```
