## 第二层：关键技术

### 范式与规范化

| 范式 | 条件 | 目标 | 风险 |
|------|------|------|------|
| **1NF** | 原子属性（无重复列） | 消除数据冗余 | 查询需要Join |
| **2NF** | 1NF + 无部分函数依赖 | 消除部分依赖 | 仍可能冗余 |
| **3NF** | 2NF + 无传递函数依赖 | 消除所有冗余 | 易导致过度规范化 |
| **BCNF** | 更严格的3NF | 最高规范化 | 实际很少使用 |

**范式选择**：软考通常要求达到3NF（第三范式）

### 事务特性（ACID）

| 特性 | 含义 | 例子 |
|------|------|------|
| **原子性(A)** | 全部执行或全不执行 | 转账：扣款+入账必须都成功 |
| **一致性(C)** | 数据从一致状态到另一致状态 | 转账前后总金额不变 |
| **隔离性(I)** | 并发事务互不影响 | 两人同时取款不会冲突 |
| **持久性(D)** | 提交后永久保存 | Commit后系统故障也不丢失 |

### SQL执行顺序

```
FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT

重点：WHERE在GROUP BY前（分组前筛选）
      HAVING在GROUP BY后（分组后筛选）
```

### 索引优化

| 索引类型 | 优点 | 缺点 | 适用 |
|--------|------|------|------|
| **B树索引** | 范围查询、排序快 | 插入/删除成本高 | 大多数情况 |
| **哈希索引** | 等值查询超快 | 不支持范围、排序 | 等值查询 |
| **全文索引** | 文本检索快 | 占空间、维护复杂 | 全文搜索 |
| **复合索引** | 多列查询优化 | 优化顺序关键 | 多条件查询 |

## 第三层：高频题型

### SQL优化原则

```
✓ 避免SELECT * → SELECT 需要的列
✓ 使用WHERE过滤 → 减少返回行数
✓ 使用JOIN替代子查询 → 性能更优
✓ 避免全表扫描 → 为过滤条件建立索引
✓ 避免OR条件 → 使用IN或UNION
✓ 避免函数调用 → 放在WHERE左边会破坏索引
```

### 锁与并发控制

| 锁类型 | 级别 | 用途 | 冲突 |
|--------|------|------|------|
| **共享锁(S)** | 行/页/表 | 读操作 | 与X冲突 |
| **排他锁(X)** | 行/页/表 | 写操作 | 与S和X冲突 |
| **意向锁(I)** | 表 | 表示下层有锁 | 防止表级锁冲突 |

### 常见数据库题型

**题型1：确定主键和外键**
```
关键：主键唯一标识实体，外键建立表间关系
```

**题型2：确定范式**
```
检查是否存在：
- 重复列（不是1NF）
- 部分依赖（不是2NF）
- 传递依赖（不是3NF）
```

**题型3：SQL语句编写**
```
注意顺序：FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY
```

**题型4：性能优化**
```
思路：建立适当的索引、优化查询语句、避免全表扫描
```
