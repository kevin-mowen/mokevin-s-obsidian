# 📐 性能与系统配置速查卡

---

## 性能评价指标

| 指标 | 定义 | 单位 | 目标 |
|------|------|------|------|
| 响应时间 | 请求到回应的时间 | ms / 秒 | **越短越好** |
| 吞吐量 | 单位时间内完成的任务数 | 任务/秒 | **越大越好** |
| 处理能力 | 系统每秒能处理的数据量 | Mbps / Gbps | **越大越好** |
| 利用率 | 资源被使用的百分比 | % | 50-70% 最优 |
| 可靠性 | 系统正常运行的时间比例 | % | **越高越好** |
| 可用性 | 系统能被访问的时间比例 | % | 99%+ |

---

## 性能瓶颈分析

### 常见瓶颈

| 资源 | 表现 | 优化方法 |
|------|------|--------|
| CPU | 处理时间长 | 多进程、并行处理 |
| 内存 | 缺页频繁 | 增加内存、虚拟内存优化 |
| 磁盘 | I/O 慢 | SSD、缓存、索引 |
| 网络 | 带宽不足 | 压缩、流量控制、CDN |

### Amdahl 定律

```
加速比 S = 1 / (f + (1-f)/N)

f = 串行部分比例
N = 并行处理器数
```

---

## 系统可靠性

### 串并联系统

**串联系统**：`R = R1 × R2 × ... × Rn`（所有部件正常才能正常）

**并联系统**：`R = 1 - (1-R1) × (1-R2) × ... × (1-Rn)`（任意一个部件正常即可）

## 成本效益分析

| 指标 | 公式 | 含义 |
|------|------|------|
| ROI | （收益 - 成本）/ 成本 × 100% | 投资回报率 |
| TCO | 购置 + 运维 + 升级 | 总体拥有成本 |
