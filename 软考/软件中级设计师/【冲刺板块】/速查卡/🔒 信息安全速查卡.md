# 🔒 信息安全速查卡

---

## 加密算法对比（重点！）

| 算法 | 类型 | 密钥 | 速度 | 用途 |
|------|------|------|------|------|
| DES | 对称 | 56 位 | 快 | 已淘汰 |
| AES | 对称 | 128/192/256 位 | 快 | **推荐** |
| RSA | 非对称 | 1024+ 位 | 慢 | 密钥交换 |
| ECC | 非对称 | 160+ 位 | 中等 | 新兴标准 |

---

## 安全协议

### 混合加密（HTTPS）

1. **密钥交换**：RSA（非对称）
2. **数据加密**：AES（对称）
3. **完整性检验**：摘要算法（MD5、SHA）

### 数字签名流程

**签名端**：
```
信息 → [哈希] → 摘要 → [私钥加密] → 签名
```

**验证端**：
```
签名 → [公钥解密] → 摘要'
信息 → [哈希] → 摘要
比较：摘要 == 摘要'？→ 验证成功
```

## 核心概念

| 概念 | 含义 | 作用 |
|------|------|------|
| 认证 | 验证身份 | 确保对方是真实的 |
| 授权 | 分配权限 | 确定能做什么 |
| 数字证书 | CA 签发的身份证明 | 公钥的凭证 |
| PKI | 公钥基础设施 | CA、证书、密钥管理 |
| VPN | 虚拟专用网络 | 加密隧道通信 |
| 防火墙 | 网络边界防护 | 过滤非授权流量 |

## 安全机制

### 防护措施

- **机密性**：加密
- **完整性**：数字签名、消息认证码
- **认证**：身份验证
- **不可否认性**：数字签名
- **访问控制**：权限管理
