## 第一层：安全威胁与对策

| 威胁类型 | 描述 | 防护方法 |
|--------|------|--------|
| **SQL注入** | 在输入中插入SQL语句，获取非法数据 | 使用参数化查询、输入验证、转义特殊字符 |
| **XSS攻击** | 在网页中注入脚本，窃取用户信息 | HTML转义、CSP策略、输入/输出验证 |
| **CSRF攻击** | 冒充用户执行非意愿操作 | Token验证、同源检查、SameSite Cookie |
| **恶意软件** | 病毒、木马、勒索软件等 | 杀毒软件、安全扫描、隔离 |
| **DDoS攻击** | 大量请求导致服务瘫痪 | 速率限制、负载均衡、CDN、防火墙 |
| **暴力破解** | 尝试所有密码组合 | 强密码策略、登录次数限制、验证码、多因素认证 |
| **中间人攻击** | 拦截通信，窃听或篡改数据 | HTTPS/TLS加密、证书验证 |
| **权限提升** | 获取高于授权的权限 | 最小权限原则、权限审计、隔离 |

## 第二层：加密与认证

### 对称加密 vs 非对称加密

| 特性 | 对称加密 | 非对称加密 |
|------|--------|---------|
| 密钥 | 发送方和接收方使用相同密钥 | 公钥加密，私钥解密 |
| 速度 | **快** | **慢** |
| 安全性 | 密钥必须保密 | 公钥可公开 |
| 应用 | 数据加密、通信加密 | 身份验证、密钥交换、电子签名 |
| 例子 | AES、DES | RSA、ECC |

### 哈希函数与数字签名

| 操作 | 用途 | 特点 |
|------|------|------|
| **哈希** | 生成数据指纹，检测篡改 | 单向不可逆，输入相同输出必同 |
| **HMAC** | 带密钥的哈希，认证消息 | 发送方和接收方都验证 |
| **数字签名** | 证明身份和消息完整 | 发送方用私钥签名，接收方用公钥验证 |
| **数字证书** | 证明公钥的真实性 | CA签名的公钥容器，包含身份信息 |

### TLS/SSL握手过程

```
客户端 ←→ 服务器

1. ClientHello：客户端发送支持的算法、随机数
2. ServerHello：服务器选择算法、发送随机数和证书
3. 密钥交换：协商共享密钥
4. Finished：两端验证加密通信
5. 应用数据加密传输
```

## 第三层：高频考点

### 易混概念对比

| 概念 | 含义 | 用途 |
|------|------|------|
| **加密** | 使用密钥将明文转换为密文 | 保护数据机密性 |
| **签名** | 用私钥对数据签名，用公钥验证 | 认证和不可否认性 |
| **认证** | 验证身份和数据完整性 | 确保通信对象真实 |
| **授权** | 确定用户能做什么 | 控制访问权限 |

### 常见安全漏洞修复

**SQL注入修复**：
```
❌ 危险：sql = "SELECT * FROM user WHERE id=" + userId
✓ 安全：PreparedStatement pst = conn.prepareStatement("SELECT * FROM user WHERE id=?")
        pst.setInt(1, userId)
```

**XSS修复**：
```
❌ 危险：<%= userInput %>
✓ 安全：<%= htmlEscape(userInput) %>
```

**CSRF修复**：
```
✓ 验证Token：request中的Token与session中的Token一致
✓ 检查Referer：确保请求来自同一域
✓ 使用SameSite Cookie：防止跨站请求发送Cookie
```
