# 🧩 软考软件设计师 —— 12种常考设计模式学习笔记

软考软件设计师考试中常考 12 种设计模式，涵盖**创建型、结构型、行为型**三大类。

---

## 📋 总览表

| 分类 | 模式 | 主要作用 | 高频考点 |
|:--|:--|:--|:--|
| **创建型（5种）** | 工厂方法（Factory Method） | 让子类决定实例化哪一个类 | ✅ 高频 |
| | 抽象工厂（Abstract Factory） | 创建一组相关对象，而无需指定具体类 | ✅ 高频 |
| | 单例模式（Singleton） | 保证系统中只有一个实例 | ✅ 高频 |
| | 建造者模式（Builder） | 分步骤构建复杂对象 | ✅ 中频 |
| | 原型模式（Prototype） | 通过克隆已有实例创建新对象 | ⚙️ 较少 |

| | 装饰器模式（Decorator） | 动态扩展对象功能 | ✅ 高频 |

## 🏗 一、创建型模式（5种）

### 1️⃣ 工厂方法模式（Factory Method）
**定义：** 定义一个用于创建对象的接口，由子类决定实例化哪一个类。

```java
interface LoggerFactory {
    Logger createLogger();
}
class FileLoggerFactory implements LoggerFactory {
    public Logger createLogger() { return new FileLogger(); }
}
```
> 💡 关键考点：由子类决定实例化；区别于“简单工厂”。

---

### 2️⃣ 抽象工厂模式（Abstract Factory）
**定义：** 提供一个创建一组相关或依赖对象的接口，而不指定具体类。

```java
interface GUIFactory {
    Button createButton();
    TextField createTextField();
}
class MacFactory implements GUIFactory {
    public Button createButton() { return new MacButton(); }
    public TextField createTextField() { return new MacTextField(); }
}
```
> 💡 关键考点：同系列产品族的创建。

---

### 3️⃣ 单例模式（Singleton）
**定义：** 一个类仅有一个实例，并提供全局访问点。

```java
class ConfigManager {
    private static ConfigManager instance = new ConfigManager();
    private ConfigManager() {}
    public static ConfigManager getInstance() { return instance; }
}
```
> 💡 考点：构造函数私有、静态方法、唯一实例。

---

### 4️⃣ 建造者模式（Builder）
**定义：** 将一个复杂对象的构建与表示分离，使构建过程可复用。

```java
class ComputerBuilder {
    void buildCPU() {}
    void buildDisk() {}
    Computer getResult() { return new Computer(); }
}
```
> 💡 考点：分步构建，通常与指挥者（Director）搭配使用。

---

### 5️⃣ 原型模式（Prototype）
**定义：** 通过克隆已有实例创建新对象。

```java
class Prototype implements Cloneable {
    public Prototype clone() { return (Prototype) super.clone(); }
}
```
> 💡 考点：使用 clone()，适合对象创建代价高的情况。

---

## 🧱 二、结构型模式（4种）

### 6️⃣ 适配器模式（Adapter）
**定义：** 将一个类的接口转换为客户端期望的接口。

```java
class VoltageAdapter implements Target {
    private Adaptee adaptee;
    public void convert() { adaptee.specificRequest(); }
}
```
> 💡 考点：接口不兼容但想复用。

---

### 7️⃣ 装饰器模式（Decorator）
**定义：** 动态地给对象添加新功能，而不改变其结构。

```java
class Coffee {
    public double cost() { return 5; }
}
class MilkDecorator extends Coffee {
    private Coffee coffee;
    public MilkDecorator(Coffee c){ this.coffee=c; }
    public double cost(){ return coffee.cost() + 2; }
}
```
> 💡 考点：继承不如组合，功能扩展的动态性。

---

### 8️⃣ 外观模式（Facade）
**定义：** 提供一个统一接口来访问多个子系统。

```java
class ComputerFacade {
    CPU cpu; Disk disk; Memory memory;
    void start() {
        cpu.boot(); memory.load(); disk.read();
    }
}
```
> 💡 考点：对子系统的简化接口，常与MVC结构结合。

---

### 9️⃣ 代理模式（Proxy）
**定义：** 为对象提供代理以控制访问。

```java
interface Image { void display(); }
class ProxyImage implements Image {
    private RealImage real;
    public void display() {
        if(real==null) real = new RealImage();
        real.display();
    }
}
```
> 💡 考点：静态代理、虚拟代理、远程代理。

---

## ⚙️ 三、行为型模式（3种）

### 🔟 观察者模式（Observer）
**定义：** 一个对象状态变化时，自动通知依赖它的对象。

```java
interface Observer { void update(); }
class Subject {
    List<Observer> list = new ArrayList<>();
    void notifyAllObs() { for(Observer o:list) o.update(); }
}
```
> 💡 考点：一对多关系、事件通知。

---

### 11️⃣ 策略模式（Strategy）
**定义：** 定义一系列算法，封装并使它们可以互换。

```java
interface Strategy { void pay(); }
class WeChatPay implements Strategy { public void pay(){} }
class Alipay implements Strategy { public void pay(){} }

class Context {
    private Strategy s;
    Context(Strategy s){this.s=s;}
    void execute(){ s.pay(); }
}
```
> 💡 考点：封装算法、替代多重 if-else。

---

### 12️⃣ 模板方法模式（Template Method）
**定义：** 定义算法骨架，允许子类实现部分步骤。

```java
abstract class Game {
    void play() {
        start();
        playing();
        end();
    }
    abstract void start();
    abstract void playing();
    abstract void end();
}
```
> 💡 考点：算法步骤固定、子类实现细节。

## 🧠 备考建议

1. **重点记忆**：工厂方法、单例、代理、观察者、策略、模板方法。
2. **对比记忆**：  
   - 工厂方法 vs 抽象工厂  
   - 适配器 vs 装饰器 vs 代理  
   - 策略 vs 模板方法  
3. **理解关键词**：题目中常出现“统一接口”“动态扩展”“唯一实例”“封装算法”等字样，对应不同模式。
4. **结合场景理解**：
   - “保证唯一对象” → 单例  
   - “增加功能不修改类” → 装饰器  
   - “子类决定实例化” → 工厂方法  
   - “接口不兼容需转换” → 适配器  
   - “多种算法可切换” → 策略  
   - “定义算法框架” → 模板方法
