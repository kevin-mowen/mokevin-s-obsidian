# 🔒 信息安全

> 状态：🟡 骨架（待补充）。错题积累见 [[错题/信息安全与加密错题]]。

## 一、信息安全五要素

| 要素 | 含义 |
| ---- | ---- |
| 机密性 | 信息不被未授权者获取 |
| 完整性 | 信息不被未授权者修改 |
| 可用性 | 授权用户能正常访问 |
| 可控性 | 可控制信息流向与行为 |
| 可审查性 | 出事可追溯 |

## 二、密码学

### 1. 对称加密
- DES（56 位）、3DES、AES（128/192/256 位）
- IDEA、RC4、RC5

### 2. 非对称加密
- RSA、ECC、ElGamal、DSA、背包算法

### 3. 哈希 / 摘要
- MD5（128 位）、SHA-1（160 位）、SHA-256/512

### 4. 数字签名 / 数字证书
- 私钥签名、公钥验证
- CA、X.509 证书

## 三、网络安全协议

| 协议 | 层 | 用途 |
| ---- | --- | ---- |
| SSL / TLS | 传输层 | HTTPS 基础 |
| IPSec | 网络层 | VPN |
| SSH | 应用层 | 安全远程登录 |
| HTTPS | 应用层 | HTTP + TLS |
| PGP | 应用层 | 邮件加密 |

## 四、攻击与防御

### 常见攻击
- 拒绝服务（DoS / DDoS）
- 中间人（MITM）
- SQL 注入、XSS、CSRF
- 缓冲区溢出
- 病毒 / 蠕虫 / 木马 / 后门

### 防御机制
- 防火墙（包过滤、状态检测、应用层）
- IDS / IPS（入侵检测 / 防御）
- 蜜罐
- 访问控制（DAC / MAC / RBAC）

## 五、风险管理

- 风险 = 资产价值 × 威胁 × 脆弱性
- 应对策略：避免、转移、降低、接受
