# 🛠️ 软件工程

> 状态：🟡 骨架（待补充）。错题积累见 [[错题/软件工程错题]]。

## 一、软件生命周期模型

| 模型 | 特点 |
| ---- | ---- |
| **瀑布模型** | 文档驱动、阶段顺序，需求稳定时适用 |
| **快速原型** | 通过原型获取需求 |
| **增量模型** | 分块交付，每块功能完整 |
| **螺旋模型** | 风险驱动，结合瀑布与原型 |
| **V 模型** | 强调测试，每个开发阶段对应一个测试阶段 |
| **RUP / 统一过程** | 用例驱动、架构为中心、迭代增量 |
| **敏捷 / Scrum / XP** | 拥抱变化、短迭代、持续交付 |

## 二、需求工程

- 需求获取、分析、定义、验证
- 需求分类：业务需求、用户需求、系统需求；功能 vs 非功能
- **数据流图（DFD）**：外部实体、加工、数据流、数据存储
- **数据字典（DD）**：4 类条目（数据流、数据项、数据存储、基本加工）

## 三、软件设计

### 1. 模块设计原则
- 高内聚、低耦合
- 内聚分级、耦合分级 → 详见 [[错题/软件工程错题#内聚（cohesion） · 由低到高]]

### 2. 结构化设计
- 数据流图 → 结构图（变换型 / 事务型）

### 3. 面向对象设计
- 设计原则：SOLID、CRP、CCP
- 详见 [[设计模式]]

## 四、软件测试

### 1. 测试层次
- 单元 → 集成 → 系统 → 验收

### 2. 测试方法
- 黑盒：等价类、边界值、因果图、决策表
- 白盒：语句覆盖、判定覆盖、条件覆盖、判定-条件覆盖、路径覆盖

### 3. 调试 vs 测试
- 测试：找 bug
- 调试：定位并修复 bug

## 五、项目管理

### 1. 进度管理
- Gantt 图、PERT 网络图
- 关键路径法（CPM）

### 2. 成本估算
- COCOMO II（应用组装、早期设计、体系结构）
- 三种估算方法：对象点、功能点、代码行

### 3. 风险管理
- 识别 → 评估 → 监控

### 4. 配置管理
- 版本控制、变更控制、配置审计

## 六、软件质量

- 质量模型：ISO 9126、McCall
- 6 大特性：功能性、可靠性、易用性、效率、可维护性、可移植性

## 七、软件维护

| 类型 | 占比 | 说明 |
| ---- | ---- | ---- |
| 改正性维护 | ~21% | 修 bug |
| 适应性维护 | ~25% | 适应环境变化 |
| 完善性维护 | ~50% | 增强功能（最多） |
| 预防性维护 | ~4% | 提前优化 |
