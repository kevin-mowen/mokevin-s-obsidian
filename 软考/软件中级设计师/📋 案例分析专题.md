# 📋 案例分析专题（下午题）

> 状态：🟡 骨架（待补充）。
> 软考软件设计师下午题共 6 道大题，每题 15 分，总分 75 分；通常 5 道必做 + 1 道二选一（C++ / Java）。

## 一、下午题题型

| 题号 | 题型 | 考点 | 备注 |

| 试题一 | 数据流图（DFD） | 软件工程 | 必做 |
| 试题二 | 数据库设计 | E-R 图、SQL、关系模式 | 必做 |
| 试题三 | UML 建模 | 类图 / 用例图 / 状态图 | 必做 |

## 二、试题一 · 数据流图

### 常考题型
- 补全顶层 DFD / 0 层 DFD 中的实体、加工、数据流
- 数据流命名
- 找平衡错误（父图与子图数据流不一致）

### 核心要点
- 4 大组件：外部实体、加工、数据流、数据存储
- 平衡原则：父图与子图的输入输出数据流必须一致

## 三、试题二 · 数据库设计

### 常考题型
- E-R 图转关系模式
- 补全主键、外键
- SQL 查询语句

### 核心要点
- 实体 → 关系；属性 → 列；联系 → 表（多对多）或外键（一对多）
- 范式：1NF、2NF、3NF、BCNF
- 详见 [[数据库]]

## 四、试题三 · UML 建模

### 常考题型
- 补全类图中的关系（聚合、组合、继承、依赖）
- 补全用例图中的 include / extend
- 补全状态图、活动图

详见 [[🎭 面向对象与UML]]

## 五、试题四 · 算法设计

### 常考题型
- 给伪代码，补全空白
- 时间复杂度分析
- 常见算法：动态规划、回溯、分治、贪心

详见 [[数据结构]]

## 六、试题五/六 · 设计模式

### 常考题型
- 给出场景，识别使用的设计模式
- 补全 C++/Java 代码（类定义、关键方法）
- 类图绘制

详见 [[设计模式]]

## 七、答题技巧

1. **审题先看问题**：带着问题读材料，节省时间
2. **图必有标注**：箭头、命名、约束都要写
3. **代码补全留意命名**：用题目已有变量名
4. **时间分配**：每题约 25 分钟，先做擅长的
