## 一、时间复杂度

- 算法复杂度以算法中基本操作重复执行的次数（建成频度）
 1、大O表示法
 ### **✅ 算法时间复杂度概念整理**
 ### **📌 时间复杂度定义**

算法的时间复杂度常以**基本操作重复执行的次数（称为频度）**来度量。
无需精确计算执行次数，只要估算出对应的“数量级”即可，例如：
👉 O(1)、O(log₂n)、O(n)、O(n²) 等。

### **📈 常见时间复杂度从小到大排序（性能从快到慢）**

| **时间复杂度** | **排序关系（从低到高）** | **名称** |
| --------- | -------------- | ------ |
| O(1)      | 最快             | 常数阶    |
| O(log₂n)  | ↑              | 对数阶    |
| O(n)      | ↑              | 线性阶    |
| O(nlog₂n) | ↑              | 线性对数阶  |
| O(n²)     | ↑              | 平方阶    |
| O(n³)     | ↑              | 立方阶    |
| O(2ⁿ)     | ↑              | 指数阶    |
| O(n!)     | ↑              | 阶乘阶    |
| O(nⁿ)     | 最慢             | n 次方阶  |
✅ 综合表示：
O(1) < O(log₂n) < O(n) < O(nlog₂n) < O(n²) < O(n³) < O(2ⁿ) < O(n!) < O(nⁿ)

### **🧮 时间复杂度化简规则（大O取法规则）**
| **规则** | **说明**                    | **示例**                     |
| ------ | ------------------------- | -------------------------- |
| ✅ 加法规则 | 多项相加时，保留**最高阶项**，并将系数化为 1 | O(n² + n) = O(n²)          |
| ✅ 乘法规则 | 多项相乘时，**全部保留阶次**，系数化为 1   | O(n * log n) = O(nlogn)    |
| ✅ 混合规则 | 小括号按乘法规则计算后，再按加法规则处理      | O((2n + 1)logₙ) = O(nlogn) |
|        |                           |                            |

### **✅ 总结（记忆技巧）**

📍 常用复杂度从快到慢口诀（记忆版）：
👉 1（常数） < log（对数） < n（线性） < nlogn（线对数） < n² < n³（多项式） < 2ⁿ（指数） < n!（阶乘） < nⁿ（最慢）
📍 越快的算法越接近 O(1)，越慢的算法趋近 O(n!) 或 O(nⁿ)。

## 三、递归式主方法

### **✅ 主方法（Master Theorem）简介**

主方法用于快速求解以下形式的递归式时间复杂度：

$T(n) = aT\left(\frac{n}{b}\right) + f(n)$

其中：

- $a \ge 1$、b > 1 为常数；
    
- f(n) 是渐进正函数；
    
- T(n) 定义在非负整数上的递归函数。
    
---
### **✅ 主定理（定理 8.1）判定规则**

设 $a \ge 1$、b > 1，则根据 $f(n) 与 n^{\log_b a}$ 的关系，时间复杂度分以下三种情况

> 情况1 和情况2公式记一下

| **情况**             | **条件**                                                                                                                  | **结论**                                       |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------- | -------------------------------------------- |
| **情况 1（子问题占主导）**   | 若存在某常数 $\varepsilon > 0$，使 $f(n) = O(n^{\log_b a - \varepsilon})$                                                       | 则 $T(n) = \Theta(n^{\log_b a})$              |
| **情况 2（平衡）**       | 若 $f(n) = \Theta(n^{\log_b a} \log^k n)$                                                                                | 则 $T(n) = \Theta(n^{\log_b a} \log^{k+1} n)$ |
| **情况 3（外部工作量占主导）** | 若存在某常数 $\varepsilon > 0$，使 $f(n) = \Omega(n^{\log_b a + \varepsilon})$，且满足正则性条件：存在 c < 1，以及足够大的 n 使 $af(n/b) \le cf(n)$ | $则$ $T(n) = \Theta(f(n))$                    |

### **✅ 示例应用**

例题：

$T(n) = 9T(n/3) + n$

| **参数**                            | **值** |
| --------------------------------- | ----- |
| $a = 9$                             |       |
| $b = 3$                             |       |
| $f(n) = n$                          |       |
| $n^{\log_b a} = n^{\log_3 9} = n^2$ |       |
对比：$f(n) = n = O(n^{2 - 1}) = O(n^{2-\varepsilon})$，满足情况 1。

✅ 因此：

$T(n) = \Theta(n^2)$

## 五、顺序表

### **✅ 插入操作时间复杂度**
假设顺序表当前长度为 n，要在第 i 个位置插入一个元素（1 ≤ i ≤ n+1）：

### **📍 最好情况：**

插入位置在末尾（即 i = n+1，无需移动元素）
👉 时间复杂度：O(1)

### **📍 最坏情况：**

插入位置在表头（即 i = 1，需要移动全部 n 个元素）
👉 时间复杂度：O(n)

### **📍 平均情况：**

插入位置随机分布在 1 到 n+1 之间

平均需要移动的元素数量为：$\frac{n}{2}$

👉 时间复杂度：O(n)

---
### **✅ 删除操作时间复杂度**

假设顺序表长度为 n，删除第 i 个元素（1 ≤ i ≤ n）：

### **📍 最好情况：**

删除最后一个元素（i = n，无需后移元素）

👉 时间复杂度：O(1)

### **📍 最坏情况：**
删除第一个元素（i = 1，需要将后续 n-1 个元素前移）

👉 时间复杂度：O(n)

### **📍 平均情况：**

删除位置随机分布在 1 到 n

平均移动次数为：\frac{n-1}{2}

👉 时间复杂度：O(n)

---

### **✅ 总结表格（适合背诵/记忆）**
| **操作** | **最好情况** | **最坏情况** | **平均情况** |     |
| ------ | -------- | -------- | -------- | --- |
| 插入     | O(1)     | O(n)     | O(n)     |     |
| 删除     | O(1)     | O(n)     | O(n)     |     |

## 九、树

### 1、树与二叉树

- 1对多
- 双亲、孩子、兄弟：节点的子树的根称为该节点的孩子；相应地，该节点称为其子结点的双亲。具有相同双亲的结点互为兄弟。
- 结点的度：结点的子树的个数，记为该结点的度。
- 叶子结点：叶子结点也称为终端结点，指度为0的结点
- 内部结点：度不为0的结点
- 结点的层次：根为第一层
- 树的高度：一颗树的最大层数记为树的高度（或深度）

### 2、树的性质

- 树中的结点总数等于树中所有结点的度数之和加1
- 度为*m* 的树中第 *i* 层上至多有 *m<sup>i-1</sup>* 个结点（ *i* $\geq$ 1) 
- 高度为 *h* 的 *m* 次树（度为m）至多有 $\frac{m^h -1}{m-1}$ 个结点
- 具有 *n* 个结点、度为 *m* 的树的最小高度为$\log_{m}(n(m-1)+1)$

### 3、二叉树

- 二叉树是 *n(n$\geq$ 0)* 个结点的有限集合，它或者空树（*n=0*），或者是由一个根结点及两颗不相交的且分别称为左、右子树的二叉树所组成。
- 二叉树中结点的子树要区分左子树和右子树，即使在结点只有一棵子树的情况下，也要明确指出该子树是左子树还是右子树
- 二叉树结点最大度为2
- 卡特兰树：$\frac{C_{2n}^n}{n+1}$ 解 n个结点有多少种二叉树
- 
#### 二叉树性质
- 二叉树第 *i* 层 *(i $\geq$ 1)* 上最多2<sup>i-1</sup> 个结点
- 高度为h 的二叉树最多有$2^h-1$ 个结点
- 对于任何一棵二叉树，度为0的结点数等于度为2的结点树+1
- 具有n个结点的完全二叉树的高度为 $\log_{2}n$+1 （取下）或 $\log_{2}(n+1)$ 取上

####  满二叉树

- 若深度为k的二叉树有 $2^k -1$ 个结点，则称其为满二叉树
#### 完全二叉树

- 深度为k，有n个结点的二叉树，当且仅当每一个结点都与深度为k的满二叉树中编号从1至n的结点一一对应时，称之为安全二叉树


### 4、二叉树的存储结构

#### 顺序存储结构

- 完全二叉树，采用顺序存储既简单又节省空间
- 但是一般二叉树的不宜采用顺序存储结构
- 最坏的情况下：一个深度为k且只有k个结点的二叉树（单支树：除了也结点，其他结点的度都有1）需要$2^k -1$ 个存储单元

#### 链式存储

- 只有单一孩子域的和唯一父结点域的：n个结点，有n+1个空域
- 有两个孩子的域和唯一父结点域的：n个结点，有n+2个空域

### 5、二叉树的遍历

#### 先序遍历

- 根 -  左 - 右

#### 中序遍历

- 左 - 根 - 右

#### 后续遍历

- 左 - 右 - 根

#### 层次遍历

- 从上到下，从左到右

#### 通过遍历构造二叉树

- 每一种单一的序列，不唯一
- 中序遍历最重要，可与其他遍历配合，确认树
- 层次序列和后续遍历也可以确定树
### 6、平衡二叉树
- 二叉树中的任意一个结点的左右子树高度之差的绝对值不超过1


### 7、二叉排序树

- 根结点的关键字
- 大于左子树所有结点的关键字
- 小于右子树所有结点的关键字
- 左右子树也是一个二叉排序树

- 中序遍历得到的序列是有序序列

### 完全图

- 若一个无向图具有 *n* 个顶点，而每一个顶点与其他 *n-1* 个顶点之间都有边，则称之为无向完全图。
- 无向完全图：含有 *n* 个顶点的无向完全图共有 $\frac{n(n-1)}{2}$ 条边
- 有向完全图：有向边为n（n-1）

- 度：顶点 *v* 的度指关联于该顶点的边的数目

- 回路或环：第一个顶点和最后一个顶点相同的路径称为回路或环
- 若一条路径上除了v<sub>p</sub> 和 v<sub>q</sub> 可以相同外，其余顶点均不相同，则称其为简单路径

## 十四、查找算法（Searching）


### ✅ 1. 顺序查找（线性查找）
- 从表头开始，逐个元素进行比较。
- 适用于：无序或小规模数据集。
- 时间复杂度：平均和最坏为 O(n)。

### ✅ 2. 折半查找（二分查找）
- 要求查找表必须是**有序表**（升序或降序）。
- 每次将查找区间折半，定位目标在哪半边。
- 时间复杂度：O(log n)

折半查找过程简图：

mid = (low + high) / 2

若 key < a[mid]，则 high = mid - 1

若 key > a[mid]，则 low = mid + 1

若 key == a[mid]，返回 mid

### ✅ 3. 哈希表（Hash Table）定义
- 哈希表是一种通过**哈希函数（Hash Function）**将关键字映射到表中一个位置（地址）进行存储的数据结构。
- 若两个关键字映射到同一位置则产生**冲突（Collision）**。
- 常见解决冲突方法：
  - 开放定址法（线性探测、平方探测、再哈希）
  - 链地址法（拉链法）

## 📦 十五、堆（Heap）结构

### ✅ 1. 小顶堆（Min Heap）
- 性质：根节点最小，任一节点值 ≤ 其子节点值。
- 适用于：求最小值、堆排序（升序时用大顶堆，降序时用小顶堆）。

### ✅ 2. 大顶堆（Max Heap）
- 性质：根节点最大，任一节点值 ≥ 其子节点值。
- 适用于：堆排序（升序时常用大顶堆）。

### ✅ 堆特点总结
|类型|根节点|用途|
|---|---|---|
|小顶堆|最小|求最小或降序排序|
|大顶堆|最大|求最大或升序排序|

## 📊 十六、常考排序算法汇总

### ✅ 1. 直接插入排序（Insertion Sort）
- 思想：将元素逐个插入到已有序序列中。
- 时间复杂度：O(n²)
- 适用于：基本有序的小规模数据。

### ✅ 2. 希尔排序（Shell Sort）
- 基于插入排序的**分组优化版本**。
- 先分组排序，再逐步减小增量 gap。
- 时间复杂度：约为 O(n¹·³~n¹·⁵)

### ✅ 3. 简单选择排序（Selection Sort）
- 每次从未排序区中选出最小（或最大）元素放到前面。
- 时间复杂度：O(n²)
- 交换次数少。

### ✅ 4. 堆排序（Heap Sort）
- 通过构造大顶堆或小顶堆实现。
- 时间复杂度：O(n log n)
- 不稳定排序。

### ✅ 5. 冒泡排序（Bubble Sort）
- 通过相邻元素比较交换将最大/小值“冒泡”到末尾。
- 时间复杂度：O(n²)
- 稳定排序。

### ✅ 6. 快速排序（Quick Sort）
- 采用分治法选取基准，将数据分为左右两部分。
- 时间复杂度：平均 O(n log n)，最坏 O(n²)
- 不稳定排序。

划分过程示意：

i 从左向右找大于 pivot 的元素

j 从右向左找小于 pivot 的元素

交换 i 和 j

左右递归

### ✅ 7. 归并排序（Merge Sort）
- 采用分治思想，先递归分割再两两归并。
- 时间复杂度：O(n log n)
- 稳定排序。
