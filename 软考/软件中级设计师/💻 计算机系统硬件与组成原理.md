# 💻 计算机系统硬件与组成原理笔记（最终美化版）

## 一、计算机系统硬件组成

计算机的基本硬件系统由 **5 大部件** 组成：

| 部件   | 英文名称            | 功能        |
| ---- | --------------- | --------- |
| 运算器  | Arithmetic Unit | 执行算术与逻辑运算 |
| 控制器  | Control Unit    | 控制指令执行流程  |
| 存储器  | Memory          | 存储数据与指令   |
| 输入设备 | Input Device    | 输入外部信息    |
| 输出设备 | Output Device   | 输出处理结果    |

> 🧠 运算器与控制器合称为 **中央处理单元（CPU）**，是整个硬件系统的核心。

---

## 二、中央处理单元（CPU）

### 1️⃣ CPU 的功能组成

| 组件                       | 功能        |
| ------------------------ | --------- |
| **控制器（Controller）**      | 程序控制、操作控制 |
| **运算器（Arithmetic Unit）** | 时间控制、数据处理 |

---

### 2️⃣ 运算器（Arithmetic Unit）

| 组件              | 功能          |
| --------------- | ----------- |
| **算术逻辑单元（ALU）** | 执行算术运算与逻辑运算 |
| **累加寄存器（AC）**   | 存放运算数据及中间结果 |

---

### 3️⃣ 控制器（Controller）

| 组件            | 功能            |
| ------------- | ------------- |
| **指令寄存器（IR）** | 存放当前执行的指令     |
| **程序计数器（PC）** | 存放下一条指令的地址    |
| **地址寄存器（AR）** | 暂存访问的地址信息     |
| **指令译码器（ID）** | 负责指令译码与控制信号生成 |

---

## 三、数的编码表示

| 编码类型   | 特点                       |
| ------ | ------------------------ |
| **原码** | 符号位 0 表示正，1 表示负          |
| **反码** | 正数反码=原码；负数反码=按位取反（符号位不变） |
| **补码** | 正数补码=原码；负数补码=反码+1；0 唯一   |
| **移码** | 在补码基础上阶码加偏置常数            |

> 💡 **口诀：** 原反补移记心中，负数求补反加一。

---

## 四、浮点数表示

若浮点数的：

- 阶码（含 1 位阶符）用 **R 位移码表示**；
- 尾数（含 1 位数符）用 **M 位补码表示**；

其可表示范围为：

$$[-(1 - 2^{-(M-1)}) × 2^{2^{R-1}-1}, (1 - 2^{-(M-1)}) × 2^{2^{R-1}-1}]$$

> 💡 **口诀：** 尾数决定精度，阶码决定范围。

---

## 五、寻址方式（Addressing Modes）

| 寻址方式                           | 特点              | 示例                |
| ------------------------------ | --------------- | ----------------- |
| **立即寻址（Immediate）**            | 操作数在指令中直接给出     | `MOV AX, 5`       |
| **直接寻址（Direct）**               | 指令中包含操作数地址      | `MOV AX, [1000H]` |
| **寄存器寻址（Register）**            | 操作数在寄存器中        | `MOV AX, BX`      |
| **寄存器间接寻址（Register Indirect）** | 寄存器中存放操作数地址     | `MOV AX, [BX]`    |
| **间接寻址（Indirect）**             | 指令中存放操作数地址的地址   | 双重访问，速度慢          |
| **相对寻址（Relative）**             | 地址 = 当前地址 + 偏移量 | 用于跳转、循环           |

---

## 六、检验码（Error Detection & Correction）

| 类型        | 特点                            | 说明   |
| --------- | ----------------------------- | ---- |
| **奇偶校验码** | 增加一位校验位使 1 的个数为奇/偶；仅能检错不能纠错   | 码距=2 |
| **海明码**   | 具有纠错功能；满足 $$2^k - 1 ≥ n + k$$ | 码距≥3 |
| **CRC 码** | 可检错但不可纠错；采用模 2 加减运算           | 码距=2 |

---

## 七、RISC 与 CISC

| 项目    | RISC（精简指令集计算机） | CISC（复杂指令集计算机） |
| ----- | -------------- | -------------- |
| 指令种类  | 少，精简           | 多，丰富           |
| 指令复杂度 | 简单             | 复杂             |
| 指令长度  | 固定             | 可变             |
| 寻址方式  | 少              | 多样             |
| 实现方式  | 硬布线逻辑          | 微程序控制          |
| 寄存器数量 | 多              | 少              |
| 流水线技术 | 易于实现           | 难以实现           |

---

## 八、指令流水线（Instruction Pipeline）

| 概念          | 公式                                |
| ----------- | --------------------------------- |
| **流水线执行时间** | $$T = 第1条指令总时间 + (n-1) × 最长阶段时间$$ |
| **加速比**     | $$S = \frac{不采用流水线时间}{采用流水线时间}$$  |
| **吞吐率**     | $$TP = \frac{1}{最长阶段时间}$$         |
**例：**

- I：输入（0.1）
- C：计算（0.2）
- O：输出（0.3）

## 九、存储器系统（Memory System）

| 分类方式 | 类型                         | 说明          |
| ---- | -------------------------- | ----------- |
| 按位置  | 内存 / 外存                    | 主存（RAM）/ 硬盘 |
| 按材料  | 磁 / 半导体 / 光                | 不同介质存储      |
| 按功能  | RAM / ROM / EEPROM / Flash | 可读写或只读      |
| 按寻址  | 随机 / 顺序 / 直接               | 不同访问方式      |

### 局部性原理（Locality Principle）

| 类型    | 含义            | 示例      |
| ----- | ------------- | ------- |
| 时间局部性 | 访问过的数据会很快再次访问 | 循环、递归   |
| 空间局部性 | 访问某地址后访问相邻地址  | 数组、顺序结构 |

💡 应用：Cache、虚拟内存、预取机制。

### 高速缓存（Cache）

| 分类       | 内容             |
| -------- | -------------- |
| **位置**   | 位于 CPU 与主存之间   |
| **替换算法** | 随机、FIFO、LRU、最优 |
| **映像方法** | 直接映像、全相联、组相联   |

## 十一、输入输出（I/O）控制方式

| 类型   | 特点                    | CPU 干预情况 |
| ---- | --------------------- | -------- |
| 程序查询 | CPU 轮询设备状态，利用率低       | 全程参与     |
| 中断驱动 | I/O 设备主动中断 CPU        | 仅中断时参与   |
| DMA  | 外设直接访问内存，CPU 与 I/O 并行 | 仅开始和结束时  |

## 十三、加密与认证技术（Encryption & Authentication）

| 类型    | 原理         | 代表算法 / 特点    |
| ----- | ---------- | ------------ |
| 对称加密  | 同一密钥加解密    | AES、DES、3DES |
| 非对称加密 | 公钥加密、私钥解密  | RSA、ECC、DSA  |
| 混合加密  | 对称 + 非对称结合 | HTTPS        |
| 摘要算法  | 哈希摘要，不可逆   | MD5、SHA 系列   |
| 数字签名  | 私钥签名、公钥验证  | 防伪、防否认       |
| 数字证书  | CA 机构签发    | SSL 证书       |

---

## 十四、可靠性公式（System Reliability）

| 系统类型 | 公式                                         | 特点          |
| ---- | ------------------------------------------ | ----------- |
| 串联系统 | $$R = R_1 × R_2 × ... × R_n$$              | 所有部件正常系统才正常 |
| 并联系统 | $$R = 1 - (1 - R_1)(1 - R_2)...(1 - R_n)$$ | 任意一个部件正常即可  |
| 混合系统 | 串并结合后化简求值                                  | 综合计算        |

💡 串联可靠性下降快；并联提高系统稳定性。
