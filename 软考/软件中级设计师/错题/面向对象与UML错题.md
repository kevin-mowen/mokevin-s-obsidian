# 🎭 面向对象与 UML 错题

## 多态（Polymorphism）的四种类型

| 类型 | 说明 |
| ---- | ---- |
| **参数多态** | 应用最广泛、最纯粹的多态（如泛型） |
| **包含多态** | 同样操作可用于一个类型及其子类型；常需运行时类型检查；最常见即**子类型化** |
| **过载多态**（重载） | 同一名字（操作符、函数名）在不同上下文中有不同类型 |
| **强制多态** | 编译程序通过语义操作把操作对象类型**强行变换**以符合要求 |

> 💡 **记忆**：参数 / 包含 = **通用多态**；过载 / 强制 = **特定多态**。

## UML 图

| 图 | 描述 |
| ---- | ---- |
| **类图** | 一组对象、接口、协作和它们之间的关系。**抽象类（abstract）不能有直接实例对象** |
| **对象图** | 某一时刻一组对象及其关系（类图的**实例快照**） |
| **用例图** | 一组用例、参与者及它们之间的关系（**需求视角**） |
| **序列图（顺序图）** | 场景的图形化表示，**以时间顺序**组织对象间交互 |

> 💡 软考易混淆：**类图 vs 对象图**——前者是模板，后者是某时刻的实例；**序列图 vs 协作图**——序列图强调时间顺序，协作图强调对象关系。
