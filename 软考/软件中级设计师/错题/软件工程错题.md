## 软件设计：内聚与耦合

### 内聚（cohesion） · 由低到高

| 类型 | 含义 |
| ---- | ---- |
| **逻辑内聚** | 模块内执行若干个**逻辑上相似**的功能，通过参数确定执行哪一个 |
| **过程内聚** | 一个模块完成多个任务，**必须按指定过程执行** |
| **通信内聚** | 模块内所有处理元素**在同一数据结构上**操作，或使用相同输入/产生相同输出 |
| **功能内聚** | 模块内所有元素**共同完成一个功能**，缺一不可（**最高内聚**） |

### 耦合（coupling） · 由低到高

| 类型 | 含义 |
| ---- | ---- |
| **数据耦合** | 通过**参数表**传递简单信息（最低，最优） |
| **公共耦合** | 多个模块访问**同一公共数据环境** |
| **外部耦合** | 一组模块访问同一**全局简单变量**（不通过参数表） |
| **内容耦合** | 模块直接访问另一模块内部数据；非正常入口跳转；代码重叠；多入口（最高，最坏） |

> 💡 **设计原则**：高内聚、低耦合。

## 设计原则（OO）

| 原则 | 关键句 |
| ---- | ----- |
| **共同重用原则（CRP）** | 包中的类应被共同重用 |
| **开放-封闭原则（OCP）** | 对扩展开放，对修改封闭 |
| **接口隔离原则（ISP）** | 多个专门接口优于单一总接口 |
| **共同封闭原则（CCP）** | 包中的类对同一种性质的变化共同封闭 |

## 模块设计原则补充

- 模块大小适中
- 减少调用深度
- **多扇入，少扇出**
- 单入口、单出口
- 模块作用域应在模块之内
- 功能可被预测

## 软件测试层级

| 测试 | 关注点 |
| ---- | ----- |
| **单元测试** | 程序代码问题；针对详细设计与实现 |
| **集成测试** | 模块之间的接口是否符合设计 |
| **系统测试** | 系统是否满足需求规格说明（功能 + 非功能） |

## 面向对象开发阶段

| 阶段 | 主要活动 |
| ---- | -------- |
| **OOA 分析** | 认定对象、组织对象、对象间相互作用、给予对象的操作 |
| **OOD 设计** | 识别类与对象、定义属性、定义服务、识别关系、识别包 |
| **OOP 编程** | 程序设计范型、选择 OOPL |
| **OOT 测试** | 算法层、类层、模板层、系统层 |
