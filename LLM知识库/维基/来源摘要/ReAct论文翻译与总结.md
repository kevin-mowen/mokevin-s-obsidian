## 论文总结

### 核心思想

ReAct 是一种让大语言模型(LLM)以**交替方式**同时生成**推理轨迹(reasoning traces)**和**任务特定动作(actions)**的方法，实现推理与行动的协同：

- **推理轨迹**帮助模型归纳、追踪和更新行动计划，并处理异常情况
- **动作**允许模型与外部环境（如知识库）交互，获取额外信息辅助推理

### 主要贡献

1. **提出 ReAct 范式**：一种基于提示的新方法，让语言模型协同推理和行动来解决通用任务
2. **广泛实验验证**：在多跳问答(HotpotQA)、事实验证(FEVER)、文本游戏(ALFWorld)和网页导航(WebShop)四个基准上展示优势
3. **系统性分析**：阐明行动对推理任务、推理对交互任务的重要性
4. **可解释性提升**：生成人类可解读的决策轨迹，提高模型的可信度和可控性

### 关键发现对比

| 对比维度 | ReAct | Chain-of-Thought (CoT) | Act-only |
|---------|-------|------------------------|----------|
| 幻觉问题 | 6% 假阳性 | 14% 假阳性 | - |
| 事实基础 | 基于外部知识库，更可靠 | 依赖内部知识，易幻觉 | 缺乏推理指导 |
| 推理灵活性 | 结构约束降低灵活性 | 更灵活 | 无推理能力 |
| ALFWorld | 71% 成功率 | - | 45% 成功率 |
| WebShop | 40% 成功率 | - | 30.1% 成功率 |

### 一句话总结

> ReAct 通过让语言模型交替生成"思考"和"动作"，将内部推理与外部环境交互相结合，在保持可解释性的同时显著提升了任务完成能力。

## 1. 摘要

虽然大语言模型(LLM)在语言理解和交互式决策任务中表现出色，但其推理能力（如思维链提示）和行动能力（如动作计划生成）主要被作为独立主题研究。

本文探索使用LLM以**交替方式**同时生成推理轨迹和任务特定动作，使两者产生更大的协同效应：

- **推理轨迹**帮助模型归纳、追踪和更新行动计划，并处理异常
- **动作**允许模型与外部源（如知识库或环境）交互以获取额外信息

我们将这种方法命名为 **ReAct**，应用于多种语言和决策任务，展示了其相对于最先进基线的有效性，同时提高了人类可解释性和可信度。

**具体成果**：

- 在**问答(HotpotQA)**和**事实验证(FEVER)**任务上，ReAct通过与简单的Wikipedia API交互，克服了思维链推理中普遍存在的幻觉和错误传播问题，生成比无推理轨迹基线更具人类可解释性的任务解决轨迹

- 在两个**交互式决策基准(ALFWorld和WebShop)**上，ReAct仅用1-2个上下文示例就分别以34%和10%的绝对成功率优势超越了模仿学习和强化学习方法

## 2. 引言

### 2.1 人类智能的特点

人类智能的一个独特特征是能够**无缝结合面向任务的行动和语言推理**（或内心独白），这在人类认知中被认为对实现自我调节或策略制定以及维持工作记忆起着重要作用。

**厨房做菜的例子**：

在任何两个具体动作之间，我们可能用语言推理来：

1. **追踪进度**："既然所有东西都切好了，我应该把水烧开"
2. **处理异常或调整计划**："我没有盐，那就用酱油和胡椒代替"
3. **意识到需要外部信息**："怎么准备面团？让我上网搜一下"

我们也可能通过行动（打开食谱阅读、打开冰箱、检查食材）来支持推理和回答问题（"我现在能做什么菜？"）。

这种"行动"和"推理"之间的紧密协同使人类能够快速学习新任务，并在以前未见的情况下或面对信息不确定性时进行稳健的决策或推理。

### 2.2 现有方法的局限

**思维链(Chain-of-Thought)推理的问题**：

- 是一个**静态黑箱**——模型使用内部表示生成想法
- **不基于外部世界**，限制了其反应性推理或更新知识的能力
- 可能导致**事实幻觉**和**推理过程中的错误传播**

**行动预测方法的问题**：

- 专注于通过语言先验预测动作
- 通常将多模态观察转换为文本，使用语言模型生成领域特定动作或计划
- **不使用语言模型对高级目标进行抽象推理或维持工作记忆**

### 2.3 ReAct 的提出

在本工作中，我们提出 **ReAct**，一种结合推理和行动的通用范式，用于使用语言模型解决多样的语言推理和决策任务。

ReAct 提示 LLM 以**交替方式**生成与任务相关的语言推理轨迹和动作，这使模型能够：

- 执行**动态推理**来创建、维持和调整高级行动计划（推理以行动）
- 与外部环境（如Wikipedia）交互，将**额外信息纳入推理**（行动以推理）

## 3. ReAct: 协同推理与行动

### 3.1 问题设置

考虑智能体与环境交互解决任务的一般设置：

- 在时间步 $t$，智能体从环境接收观察 $o_t \in O$
- 根据策略 $\pi(a_t|c_t)$ 采取动作 $a_t \in A$
- 其中 $c_t = (o_1, a_1, \cdots, o_{t-1}, a_{t-1}, o_t)$ 是智能体的上下文

当映射 $c_t \mapsto a_t$ 高度隐式且需要大量计算时，学习策略是具有挑战性的。

### 3.2 核心思想

**ReAct 的核心思想很简单**：我们扩展智能体的动作空间为：

$$\hat{A} = A \cup L$$

其中 $L$ 是语言空间。

语言空间中的动作 $\hat{a}_t \in L$（我们称之为**"思考"**或**"推理轨迹"**）：

- **不影响外部环境**，因此不会产生观察反馈
- 旨在通过对当前上下文 $c_t$ 进行推理来**组合有用信息**
- **更新上下文** $c_{t+1} = (c_t, \hat{a}_t)$ 以支持未来的推理或行动

### 3.3 有用思考的类型

1. **分解任务目标并创建行动计划**
   - 例："我需要搜索x，找到y，然后找到z"

2. **注入与任务解决相关的常识知识**
   - 例："胡椒瓶更可能出现在柜子、台面上..."

3. **从观察中提取重要部分**
   - 例："x始于1844"，"该段落没有说明x"

4. **追踪进度和转换行动计划**
   - 例："现在我找到了胡椒瓶。接下来，我需要把它放在抽屉里"

5. **处理异常和调整行动计划**
   - 例："Front Row未找到。我需要搜索'Front Row (software)'"

### 3.4 ReAct 的独特优势

**A) 直观且易于设计**

设计 ReAct 提示非常简单，人工标注者只需在其采取的动作之上用语言写下想法。本文未使用任何临时格式选择、思考设计或示例选择。

**B) 通用且灵活**

由于灵活的思考空间和思考-动作出现格式，ReAct 适用于具有不同动作空间和推理需求的多样任务，包括但不限于问答、事实验证、文本游戏和网页导航。

**C) 性能优异且鲁棒**

ReAct 仅从1-6个上下文示例学习就能强泛化到新任务实例，在不同领域中持续优于仅推理或仅行动的基线。

**D) 人类对齐且可控**

ReAct 提供了可解释的顺序决策和推理过程，人类可以轻松检查推理和事实正确性。此外，人类还可以通过**编辑思考**来即时控制或纠正智能体行为。

## 4. 知识密集型推理任务

### 4.1 实验设置

#### 数据集

| 数据集 | 任务类型 | 描述 |
|-------|---------|------|
| **HotPotQA** | 多跳问答 | 需要对两个或更多Wikipedia段落进行推理 |
| **FEVER** | 事实验证 | 每个声明标注为"支持(SUPPORTS)"、"反驳(REFUTES)"或"信息不足(NOT ENOUGH INFO)" |

本工作采用**仅问题设置**：模型只接收问题/声明作为输入，无法访问支持段落，必须依赖内部知识或通过与外部环境交互检索知识来支持推理。

#### 动作空间

设计了简单的 Wikipedia Web API，包含三种动作：

| 动作               | 功能                               |
| ---------------- | -------------------------------- |
| `search[entity]` | 返回对应实体维基页面的前5个句子，如不存在则建议前5个相似实体  |
| `lookup[string]` | 返回页面中包含该字符串的下一个句子（模拟浏览器Ctrl+F功能） |
| `finish[answer]` | 完成当前任务并给出答案                      |

### 4.2 方法

#### ReAct 提示设计

对于 HotpotQA 和 FEVER，分别从训练集随机选择6个和3个案例，手动编写 ReAct 格式的轨迹作为少样本示例。

每个轨迹由多个**思考-动作-观察**步骤组成（即密集思考），自由形式的思考用于各种目的：

- **分解问题**："我需要搜索x，找到y，然后找到z"
- **提取信息**："x始于1844"，"该段落没有说明x"
- **常识/算术推理**："x不是y，所以z必须是..."，"1844 < 1989"
- **指导搜索重新表述**："也许我可以搜索/查找x"
- **综合最终答案**："...所以答案是x"

#### 基线方法

| 方法           | 描述                       |
| ------------ | ------------------------ |
| **Standard** | 标准提示，移除所有思考、动作、观察        |
| **CoT**      | 思维链提示，移除动作和观察，仅推理基线      |
| **CoT-SC**   | 自一致性基线，采样21条CoT轨迹并采用多数答案 |
| **Act**      | 仅行动提示，移除ReAct轨迹中的思考      |

#### 结合内部和外部知识

观察到 ReAct 的问题解决过程更加基于事实和有据可依，而 CoT 在表述推理结构方面更准确但容易受到事实或思考幻觉的影响。因此提出结合策略：

**A) ReAct → CoT-SC**
- 当 ReAct 在给定步数内无法返回答案时，回退到 CoT-SC
- HotpotQA 设置7步，FEVER 设置5步

**B) CoT-SC → ReAct**
- 当 CoT-SC 样本中的多数答案出现次数少于 n/2 时（即内部知识可能无法自信地支持任务），回退到 ReAct

### 4.3 结果与观察

#### 主要结果

| 提示方法 | HotpotQA (EM) | FEVER (Acc) |
|---------|---------------|-------------|
| Standard | 28.7 | 57.1 |
| CoT | 29.4 | 56.3 |
| CoT-SC | 33.4 | 60.4 |
| Act | 25.7 | 58.9 |
| **ReAct** | **27.4** | **60.9** |
| CoT-SC → ReAct | 34.2 | **64.6** |
| **ReAct → CoT-SC** | **35.1** | 62.0 |
| 监督SOTA | 67.5 | 89.5 |

#### 成功与失败模式分析

| 类型 | 定义 | ReAct | CoT |
|-----|------|-------|-----|
| **成功：真阳性** | 正确的推理轨迹和事实 | 94% | 86% |
| **成功：假阳性** | 幻觉的推理轨迹或事实 | 6% | 14% |
| **失败：推理错误** | 错误的推理轨迹（包括无法从重复步骤中恢复） | 47% | 16% |
| **失败：搜索结果错误** | 搜索返回空或不包含有用信息 | 23% | - |
| **失败：幻觉** | 幻觉的推理轨迹或事实 | 0% | 56% |
| **失败：标签歧义** | 正确预测但与标签不精确匹配 | 29% | 28% |

#### 关键发现

**A) 幻觉是 CoT 的严重问题**

- CoT 的假阳性率(14%)远高于 ReAct(6%)
- 幻觉构成 CoT 的主要失败模式(56%)
- 相比之下，ReAct 的问题解决轨迹更加基于事实、有据可依、值得信赖，得益于对外部知识库的访问

**B) 结构约束的权衡**

- 交替推理、动作和观察步骤提高了 ReAct 的可靠性和可信度
- 但这种结构约束也**降低了其表述推理步骤的灵活性**，导致比 CoT 更高的推理错误率
- ReAct 的一个常见错误模式是重复生成之前的思考和动作

**C) 成功检索信息至关重要**

- 对于 ReAct，成功检索信息性知识是关键
- 非信息性搜索占错误案例的23%，使模型推理偏离轨道，难以恢复和重新表述思考

#### 微调结果

使用3,000条正确答案的轨迹对较小语言模型(PaLM-8/62B)进行微调：

- 微调后，**ReAct 成为四种方法中最好的**
- PaLM-8B 微调 ReAct 优于所有 PaLM-62B 提示方法
- PaLM-62B 微调 ReAct 优于所有 540B 提示方法
- 微调 Standard 或 CoT 显著差于微调 ReAct 或 Act（前者本质上是教模型记忆可能幻觉的知识事实，后者教模型如何推理和行动以从Wikipedia访问信息）

## 5. 决策任务

### 5.1 ALFWorld

#### 任务描述

ALFWorld 是一个合成的基于文本的游戏，与具身 ALFRED 基准对齐。包含6种任务类型：

| 任务类型 | 描述示例 |
|---------|---------|
| Pick | 拿起物品 |
| Clean | 清洗物品 |
| Heat | 加热物品 |
| Cool | 冷却物品 |
| Look | 在灯光下检查物品 |
| Pick 2 | 拿起两个物品 |

智能体需要通过文本动作（如 `go to coffeetable 1`、`take paper 2`、`use desklamp 1`）导航和交互来实现高级目标（如"在台灯下检查纸张"）。

**挑战**：
- 一个任务实例可能有超过50个位置
- 专家策略需要50多步才能解决
- 需要规划和追踪子目标，以及系统性探索
- 需要确定常见家居物品的可能位置（如台灯可能在桌子、架子或梳妆台上）

#### 提示设计

为每种任务类型随机标注3个训练轨迹，每个轨迹包含稀疏思考：

1. 分解目标
2. 追踪子目标完成
3. 确定下一个子目标
4. 通过常识推理物品可能在哪里以及如何处理

#### 实验结果

| 方法 | Pick | Clean | Heat | Cool | Look | Pick 2 | 总体 |
|-----|------|-------|------|------|------|--------|------|
| Act (best of 6) | 88 | 42 | 74 | 67 | 72 | 41 | 45 |
| ReAct (avg) | 65 | 39 | 83 | 76 | 55 | 24 | 57 |
| **ReAct (best of 6)** | **92** | **58** | **96** | **86** | **78** | **41** | **71** |
| ReAct-IM (avg) | 55 | 59 | 60 | 55 | 23 | 24 | 48 |
| ReAct-IM (best of 6) | 62 | 68 | 87 | 57 | 39 | 33 | 53 |
| BUTLER_g (best of 8) | 33 | 26 | 70 | 76 | 17 | 12 | 22 |
| BUTLER (best of 8) | 46 | 39 | 74 | 100 | 22 | 24 | 37 |

**关键发现**：

- ReAct 最佳试验达到 **71%** 平均成功率
- 显著优于 Act 最佳(45%)和 BUTLER 最佳(37%)
- 即使 ReAct 最差试验(48%)也超过了两种方法的最佳试验
- ReAct 相对 Act 的优势在六次控制试验中一致，相对性能提升从33%到90%不等，平均62%

### 5.2 WebShop

#### 任务描述

WebShop 是一个真实的在线购物网站环境：

- 包含 **118万真实产品** 和 **1.2万人类指令**
- 包含高度多样的结构化和非结构化文本（产品标题、描述、选项等）
- 智能体需要根据用户指令（如"我正在寻找一个有抽屉的床头柜。它应该有镍饰面，价格低于140美元"）通过网页交互购买产品

**评估指标**：
- **平均分数**：所选产品覆盖的期望属性百分比
- **成功率**：所选产品满足所有要求的回合百分比

#### 实验结果

| 方法 | 分数 | 成功率 |
|-----|------|--------|
| Act | 62.3 | 30.1 |
| **ReAct** | **66.6** | **40.0** |
| IL（模仿学习） | 59.9 | 29.1 |
| IL+RL | 62.4 | 28.7 |
| 人类专家 | 82.1 | 59.6 |

**关键发现**：

- 仅用一个示例的 Act 提示就能与 IL 和 IL+RL 方法持平
- 通过额外的稀疏推理，ReAct 实现了显著更好的性能，比之前最佳成功率绝对提升 **10%**
- ReAct 更可能通过推理来识别与指令相关的产品和选项
  - 例："对于'客厅节省空间的奥斯曼长凳'，该商品有选项'39x18x18英寸'和'蓝色'，看起来值得购买。"

### 5.3 内部推理 vs. 外部反馈的价值

#### 与 Inner Monologue (IM) 的对比

| 方面 | ReAct | Inner Monologue |
|-----|-------|-----------------|
| 思考内容 | 灵活且稀疏，允许多样推理类型 | 仅限于环境状态观察和目标完成情况 |
| 高级目标分解 | 支持 | 不支持 |
| 常识推理 | 支持 | 不支持 |

#### 消融实验结果

ReAct 大幅优于 IM 风格提示（ReAct-IM）：**71 vs. 53** 整体成功率，在六个任务中的五个具有一致优势。

**ReAct-IM 的问题**：
- 经常在识别子目标何时完成或下一个子目标应该是什么方面出错（缺乏高级目标分解）
- 难以确定物品可能在环境中的位置（缺乏常识推理）

## 6. 相关工作

### 6.1 语言模型用于推理

| 工作 | 描述 |
|-----|------|
| **Chain-of-Thought (CoT)** | 揭示了LLM自行制定"思考程序"的能力 |
| **Least-to-most prompting** | 通过分解复杂任务解决问题 |
| **Zero-shot-CoT** | 无需示例的思维链推理 |
| **Self-consistency** | 采样多条推理路径并选择多数答案 |
| **Selection-Inference** | 将推理过程分为"选择"和"推断"两步 |
| **STaR** | 通过模型自身生成的正确理由进行引导训练 |
| **Faithful reasoning** | 将多步推理分解为三步，每步由专用LM执行 |
| **Scratchpad** | 在中间计算步骤上微调LM |

**ReAct 的区别**：不仅执行孤立的固定推理，还将模型动作及其相应观察整合到连贯的输入流中，使模型能够更准确地推理并处理推理之外的任务（如交互式决策）。

### 6.2 语言模型用于决策

| 工作 | 描述 | 与ReAct的区别 |
|-----|------|--------------|
| **WebGPT** | 使用LM与浏览器交互回答问题 | 不显式建模思考和推理过程，依赖昂贵的人类反馈进行强化学习 |
| **BlenderBot, Sparrow** | 对话机器人 | 不显式考虑推理过程，依赖昂贵的数据集和人类反馈 |
| **SayCan** | 机器人动作规划 | 直接预测可能动作，由视觉环境基础的可供性模型重新排序 |
| **Inner Monologue** | 具身推理 | "内心独白"仅限于环境状态观察 |

**ReAct 的优势**：以更廉价的方式学习策略，因为决策过程只需要推理过程的语言描述。

---

## 7. 结论

### 7.1 主要成果

我们提出了 **ReAct**——一种让大语言模型协同推理和行动的简单而有效的方法。

通过在多跳问答、事实核查和交互式决策任务上的多样实验，我们展示了 ReAct 能够产生具有**可解释决策轨迹**的**优越性能**。

### 7.2 局限性

1. **上下文长度限制**：复杂任务需要更多演示来学习，可能超出上下文学习的输入长度限制
2. **提示设置的限制**：推理和行动行为的支持有限

### 7.3 未来方向

1. **使用更多高质量人类标注进行微调**可能是释放 ReAct 潜力的更好方式
2. **多任务训练**扩展 ReAct
3. 与**强化学习**等互补范式结合，可能产生更强大的智能体，进一步释放大语言模型在更多应用中的潜力

### 7.4 伦理声明

ReAct 提示大语言模型生成比以前方法更具人类可解释性、可诊断性和可控性的任务解决轨迹。

然而，将大语言模型与动作空间连接以与外部环境（如网络、物理环境）交互存在潜在危险，例如查找不当或隐私信息，或在环境中采取有害动作。

实验通过将交互限制在特定网站（Wikipedia或WebShop）来最小化此类风险，这些网站不含隐私信息，动作空间设计中没有危险动作。
