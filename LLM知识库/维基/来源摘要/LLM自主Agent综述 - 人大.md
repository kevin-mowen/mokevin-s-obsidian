# LLM 自主 Agent 综述 (人大)

## 1. 论文概述

本文是中国人民大学高瓴人工智能学院 Lei Wang、Chen Ma 等人发表于 *Frontiers of Computer Science*（2024）的综合性综述论文，系统梳理了基于大语言模型（LLM）的自主 Agent 研究全貌。论文从 Franklin 和 Graesser（1997）对自主 Agent 的经典定义出发——"一个位于环境中并作为环境一部分的系统，能够感知环境并据此行动，在未来追求自身的议程"——论述了 LLM 如何通过海量 Web 知识赋予 Agent 接近人类水平的智能，使其从传统的简单启发式策略和受限环境中解放出来。

论文围绕三大核心维度展开：（1）**Agent 构建**，包括架构设计与能力增强两个子问题；（2）**Agent 应用**，覆盖社会科学、自然科学和工程三大领域；（3）**Agent 评估**，涵盖主观与客观两类策略。论文系统收录了从 2021 年 12 月至 2023 年 8 月间发表的数十项代表性工作，绘制了该领域的研究全景图，并总结出一个统一的 [[LLM知识库/维基/概念/AI Agent|AI Agent]] 架构框架。

与此前侧重于某一方面的综述不同，本文首次将构建、应用和评估三个维度纳入统一框架，具有承上启下的里程碑意义。论文还发布了持续更新的 GitHub 仓库（收录 100 余篇相关论文），形成了活跃的社区资源。该综述被广泛引用，是 LLM Agent 领域最重要的入门文献之一。

## 2. Agent 构建

### 2.1 架构设计：统一框架

论文提出了一个包含四大模块的统一 Agent 架构，将此前大量分散的工作纳入一个一致的分析框架：

**Profile 模块（角色画像）** 负责定义 Agent 的身份信息与行为倾向。Profile 通常包含人口统计信息（年龄、性别、职业）、心理特征（人格、偏好）和社会关系。生成方式有三种：（1）手工编写（Handcrafting），直接在 prompt 中指定角色描述；（2）LLM 自动生成（LLM-Generation），如通过种子信息让 ChatGPT 批量生成 Agent 档案，可大幅节省人工成本；（3）数据集对齐（Dataset Alignment），从真实世界数据集中提取人类属性来构建 Agent 角色，使之更接近现实分布。Profile 模块是其他所有模块的基础，其配置直接影响 Agent 的行为模式。

**Memory 模块（[[LLM知识库/维基/概念/记忆机制|记忆机制]]）** 使 Agent 能够积累经验、从过去行为中学习并自我改进。论文将记忆类比为人类认知科学中的短时记忆与长时记忆。短时记忆存储当前上下文窗口内的信息（包括环境观察、Agent 的思维过程等），受 LLM 上下文长度限制；长时记忆则通过外部向量存储实现持久化，可以快速检索历史经验。记忆的表示格式包括自然语言、嵌入向量、数据库和结构化列表等。记忆操作涵盖三大核心过程：**记忆读取**（基于相关性、时效性和重要性三维加权检索）、**记忆写入**（将新经验编码存储，含去重机制）和 **记忆反思**（从具体记忆中抽象出高层洞察和规律）。Generative Agents 等工作率先引入了完整的记忆读写反思流水线。

**Planning 模块（[[LLM知识库/维基/概念/任务规划|任务规划]]）** 使 Agent 能够将复杂任务分解为可执行的子步骤。论文将规划策略分为两大类：

- **无反馈规划**：Agent 在行动前一次性生成完整计划，不进行中间调整。包括单路径推理（如 [[LLM知识库/维基/概念/思维链|思维链]] CoT、Zero-shot CoT）和多路径推理（如 CoT-SC 自一致性、Tree of Thoughts 思维树、基于 MCTS 的推理等）。多路径方法生成多条候选方案后择优选取，鲁棒性更强。
- **有反馈规划**：Agent 在执行过程中根据反馈动态调整计划。反馈来源包括环境反馈（如任务完成信号、游戏状态变化）、人类反馈（用户对齐引导、偏好信号）和模型反馈（如 [[LLM知识库/维基/概念/反思与优化|反思与优化]] 机制——Reflexion 等让 Agent 自我审视推理过程并迭代改进）。Inner Monologue 结合多种反馈源的方法展现了更强的适应性。

**Action 模块** 是将 Agent 内部决策转化为外部行为的执行层。论文从四个维度刻画行动：（1）行动目标（任务完成 vs 环境探索）；（2）行动生成（基于记忆回放、遵循计划或交互产出）；（3）行动空间——包括内部知识工具（LLM 自身推理）和外部 [[LLM知识库/维基/概念/工具使用|工具使用]]（API 调用、代码执行、数据库查询、外部模型调用等）；（4）行动影响（对环境状态、Agent 自身状态和其他 Agent 的影响）。

### 2.2 能力增强策略

Agent 的能力获取主要依赖两条路径：

**微调（Fine-tuning）与数据驱动方法**：利用人工标注数据集或 LLM 生成的数据集进行模型微调。例如在人工标注方面，ToolBench 收集了 16,000+ 真实世界 API 调用轨迹训练工具使用能力；在 LLM 生成方面，MIND2WEB 收集大量真实世界数据集增强 Web 操作能力。微调适合特定领域的深度优化，但受限于标注成本和数据覆盖面。

**Prompt 工程与机制工程**：无需修改模型参数，通过精心设计的提示策略增强能力。具体方法包括：（1）试错法（Trial-and-error），Agent 在执行后根据反馈迭代修正，如 Voyager 和 DEPS；（2）众包式自我进化（Crowdsourcing），通过多 Agent 交互汇聚集体智慧达成共识；（3）经验积累（Experience accumulation），将历史成功经验存入长时记忆供未来复用，如 GITM 将成功的行动序列存储为可复用的技能库；（4）自我驱动演化（Self-driven evolution），Agent 自主设定目标、探索环境并逐步提升能力，如 Voyager 通过自动课程机制不断发现新任务。

## 3. 应用场景

### 3.1 社会科学

**心理学**：LLM Agent 被用于模拟心理学实验，通过赋予不同 Profile 进行人格测试、行为实验等。研究发现大模型能够产生与人类被试一致的实验结果，但也可能输出"超准确失真"（hyperaccuracy distortion）预测，且可能对心理健康产生正面或负面影响。

**政治科学与经济学**：Agent 被用于模拟意识形态检测、投票行为预测和消费者行为模拟。例如让 LLM 分析政治演讲中的修辞策略和说服元素，或模拟不同经济场景下的市场行为。

**社会模拟**：这是最引人注目的应用方向之一。Generative Agents 在虚拟小镇中部署 25 个 Agent 模拟日常社交行为；AgentSims 构建了可扩展的沙盒平台用于研究涌现社会行为；S3 构建了社交网络模拟器研究信息传播。社会模拟中，Agent 被赋予特定的人格特质、偏好和社会背景，通过 [[LLM知识库/维基/概念/多Agent协作|多Agent协作]] 和交互产生涌现性的群体行为，为社会科学研究提供了全新的"硅基实验对象"。

**法学与社会科学研究辅助**：ChatLaw 是专为中国法律设计的模型，支持关键词搜索和对话式法律咨询；多个 Agent 系统被设计用于辅助文献检索、学术写作和研究流程管理。

### 3.2 自然科学

**文献管理与数据处理**：LLM Agent 能够自动检索、整理和综合大量文献，降低科研信息管理的工作负荷。Agent 可执行数据库查询、信息提取、实验数据管理等任务。

**实验辅助**：ChemCrow 集成了 18 种专家工具，可以协助化学研究人员完成有机合成规划、安全风险评估和化合物属性预测；在数学领域，多个系统利用 LLM 进行计算推理和定理验证；在生物学领域，Agent 被用于蛋白质结构预测、基因组分析等。

**自然科学教育**：Math Agents 辅助数学问题的探索和解决过程；CodeHelp 和 PENTESTGPT 分别在编程教学和网络安全教育中发挥作用，通过对话式交互提供个性化学习支持。

### 3.3 工程

**土木工程**：LLM Agent 被应用于建筑设计优化、道路规划和 3D 仿真环境中的建筑任务协调。

**计算机科学与软件工程**：这是应用最密集的领域。ChatDev 提出了一个端到端的软件开发框架，多个 Agent 分别扮演产品经理、架构师、程序员、测试员等角色，通过自然语言协作完成从需求分析到代码实现的全流程。MetaGPT 引入了标准操作流程（SOP），使多 Agent 协作更加规范高效。此外还有用于代码补全（GitHub Copilot）、代码审查、漏洞检测（PENTESTGPT）、软件测试等各种专门 Agent。

**工业自动化**：Agent 被整合到工业自动化流水线中，用于柔性制造、流程优化和产线调度，展示了 LLM 在实体工业中的应用潜力。

**机器人与具身 AI**：这一方向将 LLM Agent 与物理执行能力结合。SayCan 将 LLM 的语言理解能力与机器人的物理可行性评估相结合；Voyager 在 Minecraft 中持续探索和学习新技能；SayPlan 在机器人导航中结合 3D 场景图实现大规模任务规划。论文指出具身 Agent 需要解决动作空间的精细控制、跨模态融合和长期规划三大核心挑战。

**航空航天**：初步有工作将 LLM Agent 引入航空航天设计和仿真场景。

## 4. 评估策略

### 4.1 主观评估

**人工标注（Human Annotation）**：直接由人类评估者对 Agent 输出进行评分或排序。常用方法包括对关键问题的定性反馈（如评估合理性、有用性、参与度）和跨模型比较评分。研究发现，针对特定角色设计的模型在人工评估中表现更佳。

**图灵测试（Turing Test）**：让人类评估者判断 Agent 输出是否来自人类。例如在自由格式 Partisan 文本实验中测试 Agent 是否能通过人类审查；在情感模拟实验中，评估者需区分 LLM 生成的情感表达和人类表达。

### 4.2 客观评估

论文系统梳理了客观评估的三大维度：

**评估指标**：（1）任务成功率——衡量 Agent 完成特定任务的能力；（2）人类相似度——衡量 Agent 行为与人类行为的匹配程度，包括轨迹准确性、对话自然度、决策一致性等；（3）效率指标——评估完成任务所需的步骤数、时间成本和计算资源消耗。

**评估协议**：包括环境仿真评估（在 ALFWorld、Minecraft 等虚拟环境中测试）、社会评估（测试多 Agent 的协作和沟通能力，以及对社会规范的遵从度）、多任务评估（跨领域综合能力测试）和软件测试方法（如对代码生成 Agent 的覆盖率和错误率评估）。

**评估基准**：论文收录了大量评测基准，包括 AgentBench（跨 8 种环境的综合测试）、ToolBench（工具使用能力）、WebArena（Web 交互任务）、SocKET（社会知识评估）、EmotionBench（情感能力评估）等。这些基准从不同维度量化了 Agent 的能力边界。论文指出，现有评估体系仍存在碎片化问题，亟需更统一、更全面的评估框架。

## 5. 关键挑战与未来方向

论文识别出以下核心挑战：

**角色扮演能力（Role-playing）**：LLM 难以准确扮演训练数据中较少出现的角色（如冷门职业或新兴角色），且缺乏自我意识来完整建模认知心理特征。未来方向包括利用真实人类数据微调和设计专门的 Agent 提示/架构来增强角色模拟的真实性。

**广义人类对齐（Generalized Human Alignment）**：传统对齐使 LLM 仅表现出"正确"的行为，但在社会模拟场景中，Agent 需要真实地模拟包括负面特质在内的多样人类行为。这对现有的对齐策略提出了根本性挑战——模拟器需要忠实反映现实而非仅展示理想化行为。

**提示鲁棒性（Prompt Robustness）**：当前 Agent 对提示设计高度敏感，微小的 prompt 变化可能导致行为剧变。Memory、Planning 等模块的引入使 prompt 更加复杂，进一步加剧了鲁棒性问题。不同 LLM 对同一 prompt 框架的响应差异也很大，缺乏跨模型通用的鲁棒设计原则。

**[[LLM知识库/维基/概念/幻觉问题|幻觉问题]]（Hallucination）**：LLM 产生不忠实或虚假内容的倾向在 Agent 场景中尤为危险，因为幻觉可能直接导致错误的行动决策。在代码生成中，幻觉可能导致安全漏洞；在科学研究中，可能导致错误的实验方案。论文指出将人类验证环节嵌入 Agent 的行动-感知循环是缓解幻觉的重要方向。

**知识边界（Knowledge Boundary）**：LLM Agent 的知识局限于训练数据的截止日期和覆盖范围，在面对实时信息、专业领域知识或涌现性新知识时存在根本性限制。社会模拟中，Agent 可能基于过时知识做出与现实不符的决策。

**效率问题（Efficiency）**：由于 Agent 需要频繁调用 LLM 进行推理、记忆检索和多轮交互，计算和经济成本可能极高。多 Agent 系统中成本问题更加严峻，需要在能力和效率之间寻求平衡。
