# 腾讯乐享《AI 知识工程解决方案-金融行业》

> 原标题：**金融企业智能化跃迁必由之路 — AI 知识工程的建设实践和思考**
> 演讲人：蓝艺伟@腾讯 · 发布时间：2025.03
> 定位：面向金融行业的企业知识库产品解决方案白皮书

## 核心命题

> **对于组织来说，知识沉淀的最大价值，是让每一位员工站在组织的智慧高地上，实现能力的加速进化。**
>
> **知识体系的成功，不仅在于"存"，更在于"活"**（知识鲜活、用户活跃）。

与学术界讨论 [[LLM知识库/维基/概念/AI Agent|AI Agent]] 的技术能力不同，这篇材料从**企业落地**视角给出了一套完整的知识工程方法论：知识库不只是"存文档"，而是**"知识库 + 智能体 + 大模型"铁三角**中的基础设施。

## 01 · 金融企业为什么需要建知识库

### 五大核心痛点

1. **用不好**：员工知识获取效率低，找不到、找不全
2. **质量差**：知识质量参差不齐，版本混乱
3. **答不对**：员工对监管合规、产品条款理解不一致
4. **太封闭**：知识孤岛，部门墙严重
5. **不合规**：监管政策频繁变更，人工追踪成本高、易遗漏

### 金融行业的特殊性

- 监管政策密集、合规要求高（央行、银保监会等频繁发文）
- 产品条款复杂（理财、保险、基金等，每个产品数十至数百份材料）
- 客户经理需即时应答客户，对"有用 + 好用"要求极高
- 跨部门协同需求强（合规 / 市场 / 销售 / 投研）

## 02 · 金融企业需要什么样的知识库

**五大特征画像**（闭环结构）：

```
触手可及 → 智能运营 → 安全合规 → 精准严谨 → 开放协同 → (回到触手可及)
```

### 2.1 一个"触手可及"的知识库（连 · 萃）

**"连"：作为企业知识连接器**

| 知识来源类型 | 接入方式 |
|--------------|----------|
| 组织架构 | 同步企微/钉钉/飞书通讯录、打通员工工作台 |
| 会议成果 | 知识库 + 腾讯会议（会议纪要、培训录屏、客户拜访录屏自动入库） |
| 文档流 | 知识库 + 腾讯文档 + 企业微信微盘 |
| 外部数据 | HTML 导入、API 同步、数据库接入、文件夹批量上传 |
| 消费入口 | 企微聊天/公众号/微信聊天文件直接回流至知识库 |

**"萃"：知识萃取 — 隐性知识显性化**

- **FAQ 萃取**：从金融产品说明书/协议自动提炼投资者 FAQ，用"大白话""类比"（如把"复利"讲成"滚雪球"）降低理解门槛
- **话术萃取**：从销冠的成功对话中系统性提炼标准化沟通流程、应答模板；按场景/客户类型/效果标签分类，把**个人经验**转化为**团队方法论**
- **个人知识库**：支持个人隐私隔离 + AI 加持，让每个员工都有自己的"第二大脑"

**应用场景举例**
- 合规团队：自动追踪央行/银保监官方渠道 + 内部跨团队知识库，实时整理政策发布、解读文章、专家观点
- 政策文件智能解读：AI 自动提取核心条款、对比历史版本、关联相关法规、生成图文解读并推送至基层

### 2.2 一个"智能运营"的知识库（检 · 变 · 看）

**"检"：知识库智能体检 — AI 检查知识质量问题**

- 合规政策知识一致性检查（不同部门对同一政策的解读差异检测）
- 投研知识库质量巡检（数据是否最新、结论是否有依据、敏感信息泄露风险）
- 自动生成整改清单

**"变"：知识整改与更新**

- **AI 重新分类知识库**：输出行动项 + 优先级 + 预计工时（如"重命名文件夹编号：10 分钟 · 高优先级"），给出综合评分
- **以单个文档为基准审核知识库**：央行/银保监发布新政策时，AI 自动识别政策变更内容，批量更新所有涉及该政策的文档（合规手册、操作流程、风控指引）
- **知识点变化批量更新**：某基金/理财产品费率调整，AI 自动扫描知识库中所有包含该产品费率的文档（产品说明书、营销材料、合同模板），批量替换并生成变更记录

**核心痛点**：政策频繁变更，人工逐一更新文档耗时 1-2 周；一项政策可能影响数十份文档，易遗漏；更新滞后 → 合规风险高。

**"看"：知识运营报告**

- AI 自动统计知识库使用情况、价值评估、用来写知识运营周报
- "媒体 AI 服务的视频字幕能力与 ASR 的视频字幕能力哪个好？性价比更高？"等智能分析

### 2.3 一个"能干活"的知识库（Agent 模式升级）

从"**信息检索器**"升级为"**自主问题解决者**"。

**Agent 核心层架构**：

```
用户请求/任务
   ↓
┌─ Agent 核心层 ──────────────────────┐
│  上下文管理器（记忆与历史）         │
│  决策引擎（意图理解）               │
│  任务规划器（步骤分解）             │
└──────────┬──────────────────────────┘
           ↓
      能力调用（分发至 Tool / Skill）
         ↓              ↓
   工具层（通用）    技能层（专业）
   - 搜索            - PPT 生成
   - 文件读写        - 会议洞察
   - 代码执行        - 数据分析
   - 媒体生成        - 报告生成
         ↓              ↓
         最终结果/输出（执行反馈回流）
```

**三大能力升级**：

1. **主动拆解复杂任务**：引入 [[LLM知识库/维基/概念/思维链|Chain-of-Thought]] 机制，复杂查询自动拆为多个子任务，拒绝"答非所问"
2. **深度释放模型潜能**：从"片段拼凑"到"**多跳推理**（Multi-hop Reasoning）"，在海量文档中跨文档建立知识关联
3. **上下文深度理解**：连续对话中保持记忆力，理解"言外之意"，必要时反问澄清

**文件自审 & 库审文**
- 文件自审：基于单个文档一句话生成评审方案（如"微信公众号推文要求：标题不能用虚假夸大词、正文不能出现未经充分验证的专业描述等"），AI 高亮修改区
- 库审文：将新编写的规范指引提交到"管理规范知识库"，AI 检查错别字、语法、业务流程合规性、制度一致性
- 库审图：上传产品宣传海报，AI 按"金融广告法规"+"消费者权益保护"审核合规性

**典型金融场景**
- 场景 1：理财产品营销 PPT 定制（按客户类型/营销目标，从产品知识库提取匹配信息生成个性化推介 PPT）
- 场景 2：政策解读 PPT 一键生成（面向领导层/基层/公众的不同版本）

### 2.4 一个"开放协同"的知识库（MCP + Agent Skill）

**开放的"知识虾"（OpenClaw / 小龙虾 企业知识协作体）**

- **专家经验 / 隐性知识 / 业务流程 → 封装成 Skill → 全员随时调用**
- 典型 Skill：
  - 面客材料 Skill（Top Sales 的成交技巧 → 新人快速上手）
  - 故障排查 Skill（资深工程师的排错思路 → 运维效率提升）
  - 数据报表 Skill（财务专家的分析模板 → 决策有据可依）

**四大价值**：
1. **知识资产化**：隐性经验变成可沉淀、可传承的数字资产，核心员工离职不再等于知识流失
2. **效率跃升**：新人不再需要"传帮带"周期，从"入职即焦虑"变成"入职即产出"
3. **能力平权**：基层员工也能输出符合企业标准的专业成果，从"明星驱动"转向"系统驱动"
4. **持续进化**：Skill 可迭代、可优化，始终保持最新、最准、最好用

**铁三角架构**：

```
应用层：知识助手 / 智慧办公 / 营销支持 / 风险管理 / 投资决策 / 运营优化
   ↓
平台层：[知识库平台] ←→ [智能体开发平台]
         - 知识生产/管理/处理/运营/开放
         - 工作流引擎 / Agent 引擎 / 应用管理
   ↓
模型层：专业模型（OCR/Embedding/MLLM/阅读理解） + 大语言模型（语义/推理/深度思考）
   ↓
算力层：GPU 芯片 / 高性能计算集群 / 存储 / 网络
```

**MCP 接入方案**
- 提供**临时凭证**与 **OAuth**两种接入方式，支持 Cursor、VS Code、CodeBuddy 等客户端
- 推荐默认配置：**preset-meta**，仅返回 4 个元工具（`list_tool_categories`、`search_tools`、`get_tool_schema`、`call_tool`），AI 助手再通过这些工具动态发现调用其他工具（节省 Token + 获得完整能力）
- Agent Skill：持续迭代操作日志、权限范围管理、异常封禁等安全机制

### 2.5 一个"安全合规"的知识库（四级权限）

- **权限继承**：企业 → 团队 → 知识库 → 文档 四级权限体系
- **明暗水印**：水印内容（姓名/账号/部门）+ 水印密度（稀疏/标准/密集）
- **公私分离**：限制企业员工身份登录、企业账号独立体系管理、权限与组织架构关联
- **合规操作日志**：所有知识操作留痕
- **本地化部署 & 全栈信创适配**：海光/鲲鹏/中科可控 CPU · 昇腾 910B GPU · 腾讯 TDSQL · 麒麟/红旗操作系统

## 03 · 金融企业怎么用知识库（三大案例）

### 案例 1：重庆农商行 · "小渝智问" 营销知识库

- **功能**：企业微信客户通系统中嵌入智能问答机器人，赋能直营客户经理掌握产品知识、合规要求、沟通技巧
- **场景**：客户经理一边和客户聊天，AI 一边在侧边栏推荐回复话术（如"渝快贷的征信要求：1. 近两年连续逾期不超过 3 次、2. 累计逾期不超过 6 次..."）
- **技术栈**：知识库大模型 + DeepSeek
- **拓展**：渝见各类员工知识库参考建设

### 案例 2：上海证券交易所 / 证监会 · 知识中台

**建设背景**：
- 监管端：政策文件、案例、数据报告分散，协同效率低
- 券商端：合规与业务要求获取成本高，动态更新快，中小机构能力薄弱
- → 交易所牵头构建**行业级**知识共建共享中心

**建设目标**：
1. 构建新型基础设施，打破知识壁垒（连接监管端、机构端的统一知识平台）
2. 打造智能服务体系（法规 AI 助手、财富 AI 助手、基金 AI 助手、债券 AI 助手、机构知识服务）
3. 推进知识升级，树立生态标杆

**知识平台架构**：
```
知识源（行业监管库/交易所数据中台/债券案例库/外部优质数据集）
   ↓ 知识导入
知识初始化加工（OCR 解析 → MLLM 图片理解 → 知识向量化）
   ↓ 知识转换
知识空间构建（主题库空间/专属空间/机构空间/共享空间）
   ↓
AI 知识服务平台（债券 AI 助手 / 财富 AI 助手 / 法规 AI 助手 / 基金 AI 助手）
   ↓
浏览器、Iframe 嵌入等方式接入
```
**安全增强**：知识源校验 / 角色管控 / 模型有效性管理 / 数据质量管理 / 权限管控 / 知识准确性审核

### 案例 3：某资管公司 · 中金"小金助手"主智能体

通过企微 - 小金助手**主智能体（A2A）**实现与企业微信原生智能机器人的无缝集成，打通**子智能体**：
- 会议场景 / 知识库问答场景（企业版 ima）/ 文档场景（利旧）
- 待办场景 / 政策场景（利旧）/ 翻译场景（利旧）

**分层架构**：
```
应用层：企微-小金助手（A2A 主智能体） → 6 大子智能体
开发层：腾讯云智能体开发平台 ADP（RAG + Workflow + Multi-Agent + MCP + Prompt）
模型层：腾讯云自研 RAG 模型（意图识别/文档解析/文档切片/Embedding/Rerank）
知识层：腾讯乐享（员工/团队/部门/企业 四级知识库 + 权限管理）
```
效果：收敛员工散乱的问答入口，AI 不仅能问答**更会执行**，大幅提升中金公司内部组织协同效率。
