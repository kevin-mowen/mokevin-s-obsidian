# LLM Agent 规划能力综述

## 1. 论文概述

本文是首篇系统性分析基于大语言模型（LLM）的智能体规划能力的综述论文，由中国科学技术大学与华为诺亚方舟实验室联合发表（2024年2月）。论文全面梳理了利用 LLM 作为自主智能体规划模块的最新研究进展，提出了一套涵盖五大方向的分类体系，并对每个方向进行了深入的技术分析。

规划（Planning）被视为智能体最核心的能力之一，涉及对环境的感知、复杂推理和决策制定。传统方法主要依赖符号方法（如 PDDL）或强化学习，但这些方法存在灵活性不足、数据需求大等局限。LLM 的出现带来了范式转变——它们在推理、工具使用、规划和指令遵循方面展现出显著能力，为构建更强大的智能体规划系统开辟了新路径。

论文不仅梳理了技术方法，还在四个代表性基准上对多种方法进行了实验评估，讨论了各方向的优势与不足，并指出了幻觉、可行性验证、高效规划等未来研究方向，为该领域提供了全景式的学术参考。

## 2. 规划能力分类体系

论文将现有 LLM Agent 规划方法划分为五大方向，形成系统化的分类框架：

| 方向 | 核心思想 | LLM 的角色 |
|------|--------|----------|
| **任务分解** | 分而治之，将复杂任务拆分为子任务 | 分解任务 + 子任务规划 |
| **多计划选择** | 生成多个候选计划，选择最优方案 | 计划生成 + 计划评估 |
| **外部模块辅助** | 借助外部规划器提升效率和可行性 | 任务形式化 |
| **反思与优化** | 从经验和失败中反思并优化计划 | 计划生成 + 反思优化 |
| **记忆增强规划** | 利用外部记忆模块辅助规划 | 规划生成 + 记忆检索 |

这五个方向并非互斥，实际系统中常常同时采用多种技术。

## 3. 任务分解（Task Decomposition）

任务分解采用"分而治之"的策略，将复杂任务分解为若干子任务，然后对每个子任务分别进行规划。论文将其分为两种范式：

### 3.1 Decomposition-First（先分解后执行）

这类方法首先将任务分解为所有子目标，然后依次对每个子目标制定计划。代表性工作包括：

- **Chain-of-Thought (CoT)**：通过"让我们一步步思考"等提示词，引导 LLM 进行逐步推理，是 zero-shot 推理的基础方法。后续衍生出 Zero-shot CoT、Plan-and-Solve 等变体。
- **HuggingGPT**：利用 ChatGPT 作为控制器，将来自 HuggingFace Hub 的多模态模型组织为智能体，能够处理图像分类、目标检测、语音合成等任务。LLM 负责任务分解、模型选择和结果整合。
- **ProgPrompt**：将自然语言任务描述转化为编程语言中的代码结构，通过函数和断言表达任务和动作。
- **PAL (Program-Aided Language Models)**：利用 LLM 生成代码，再通过 Python 解释器执行，提升数学推理和符号推理能力。

### 3.2 Interleaved Decomposition（交错分解）

交错分解方法将分解和执行过程交替进行——每次只分解出一到两个子任务，执行后根据反馈动态调整后续分解。代表性方法包括：

- **Chain-of-Thought (CoT) 系列**：连续版本的思维链推理。
- **ReAct**：将推理（Reasoning）和行动（Acting）交错进行，在每一步既产生思考又执行动作。
- **Visual ChatGPT**：引入 Chain-of-Thought 的视觉版本，增强多模态推理。

**两种范式的比较**：Decomposition-First 方法由于预先建立了子任务间的映射关系，降低了任务遗忘和幻觉风险；但交错分解方法能够根据环境反馈动态调整，不过也可能因错误累积导致幻觉。

## 4. 多计划选择（Multi-plan Selection）

多计划选择旨在引导 LLM 进行更深入的"思考"，生成多个候选计划后选择最优方案，包含两个阶段：

### 4.1 多计划生成

通过生成模型的不确定性产生多个候选计划。主流方法包括：

- **Self-consistency（自一致性）**：采用一种简单而强大的策略——对同一问题多次采样生成不同推理路径，然后通过多数投票选取最一致的答案。支持 temperature 采样、top-k 采样等策略。
- **Tree-of-Thought (ToT)**：提出将推理过程组织为树状结构，每个节点代表一个"思考"，通过 BFS 或 DFS 探索不同推理路径。
- **Graph-of-Thought (GoT)**：进一步将思维结构扩展为图结构，能够处理更复杂的推理依赖关系。

### 4.2 最优计划选择

选择最优计划的策略包括：

- **基于搜索的选择**：Tree search（如 Monte Carlo Tree Search, MCTS）、A* 搜索等经典算法。
- **LLM 自评估**：利用 LLM 本身作为启发式评估函数，对候选计划进行评分和排序。
- **外部验证器**：使用独立验证模块检查计划的可行性。

ToT-SC 等方法将 Self-consistency 与 Tree-of-Thought 结合，进一步提升了计划质量。多计划选择的可扩展性是其显著优势，但也带来了更高的计算开销。

## 5. 外部模块辅助规划（External Planner-Aided Planning）

当 LLM 自身的规划能力不足时，可借助外部规划器来提升效率和可行性。

### 5.1 符号规划器（Symbolic Planners）

符号规划器在自动化规划领域有数十年的研究积累，具有理论完备性、稳定性和可解释性等优势：

- **PDDL（Planning Domain Definition Language）**：经典的规划领域定义语言，LLM 将自然语言任务转化为 PDDL 格式，再由专业规划器求解。代表性工作包括 LLM+P、LLM+PDDL 等。
- **CLINGO**：基于 Answer Set Programming (ASP) 的求解器，用于将任务转化为逻辑程序。
- **LLM 的角色转变**：在这种范式下，LLM 主要负责将自然语言问题形式化为符号表示，而非直接进行规划。

### 5.2 神经规划器（Neural Planners）

神经规划器是通过监督学习或模仿学习在特定领域数据上训练的深度模型：

- **Decision Transformer**：将规划过程建模为马尔可夫决策过程，通过强化学习训练。
- **SayCan**：结合 LLM 的语言理解能力和机器人的可行性评估，通过 affordance function 筛选可执行的动作。
- **SwiftSage**：将规划分为快思考（fast thinking）和慢思考（slow thinking），快思考模块处理简单任务，慢思考模块处理复杂问题，两者协同工作。

## 6. 反思与优化（Reflection & Refinement）

反思和优化是提升 LLM Agent 容错能力和纠错能力的关键机制，使智能体能够从失败中学习并改进计划：

- **Self-refine**：采用迭代式的生成-反馈-优化循环，LLM 生成初始计划后自我评估并给出反馈，然后基于反馈进行优化，无需额外训练数据或强化学习。
- **Reflexion**：将反思能力与强化学习结合，智能体在执行失败后进行语言化的自我反思，将反思存储在记忆中，指导后续决策。它既包含短期记忆（当前轨迹）也包含长期记忆（反思总结）。
- **CRITIC**：让 LLM 通过与外部工具（如搜索引擎、代码执行器）交互来验证和修正自己的输出，增强自我纠错能力。
- **Self-Refine 与环境反馈的区别**：Self-refine 依赖 LLM 自身的内部反馈，而 Reflexion 和 CRITIC 则利用环境交互获取外部反馈。基于强化学习的方法通过环境奖励信号更新策略，而自我反思则通过语言化的文本反馈实现改进。

论文指出，当前自我反思方法的收敛性缺乏理论保证，持续的自我优化可能无法始终带来性能提升。

## 7. 记忆增强规划（Memory-augmented Planning）

记忆模块为智能体提供额外的知识来源，增强规划能力：

### 7.1 RAG-based Memory（检索增强记忆）

- **基本思路**：将过去的经验、常识知识、领域知识等存储在外部记忆中，在规划时通过检索获取相关信息作为辅助。
- **代表性工作**：
  - **Generative Agents**：模拟人类日常行为，将经验以文本形式存储，通过相关性和重要性评分检索记忆。
  - **MemoryBank**：使用文本编码模型将记忆存储在向量数据库中，通过 FAISS 等工具进行高效检索。
  - **MemGPT**：将操作系统的内存层级概念引入 LLM 上下文管理，通过类似虚拟内存的机制扩展有效上下文窗口。
  - **REMEMBER**：将历史信息组织为 Q-table 结构，存储环境状态、动作、奖励等信息，在规划时检索正面和负面经验。

### 7.2 Embodied Memory（具身记忆）

具身记忆侧重于将 LLM Agent 在物理或虚拟环境中的交互经验嵌入到模型参数中：

- **方法**：通过微调（fine-tuning）将环境交互经验内化到模型权重中，使用 LoRA 等参数高效微调技术降低计算成本。
- **代表性工作**：AgentTuning、ScienceWorld 任务中的微调方法、TDT（Text Decision Transformer）等。
- **优势与局限**：具身记忆能提升模型在特定领域的规划能力，但训练数据稀缺是主要瓶颈，且过度依赖有限训练数据可能导致泛化能力不足。

## 8. 评估方法与基准

论文在四个代表性基准上对多种方法进行了实验评估：

| 基准 | 类型 | 特点 |
|------|------|------|
| **ALFWorld** | 交互式文本游戏 | 基于文本的家庭环境任务，含 6 种任务类型 |
| **ScienceWorld** | 交互式文本游戏 | 科学实验模拟，含 30 个任务，难度更高 |
| **HotpotQA** | 信息检索 | 多跳问答，需要跨文档推理 |
| **FEVER** | 信息检索 | 事实验证任务，判断陈述的真假 |

**关键评估发现**：
- Reflexion 在成功率方面表现突出，说明反思机制对复杂任务至关重要。
- 尽管 CoT 的单步提示简洁有效，但在复杂环境中可能导致更多推理错误。
- 外部反馈（如 Reflexion）比纯自我反馈（如 Self-refine）通常更有效。
- ZeroShotCoT 在某些基准上出现性能下降，说明示例对 LLM 理解任务的重要性。

**未来挑战**：
- **幻觉问题**：LLM 在规划中可能产生幻觉，导致不可行的计划。
- **可行性规划**：如何生成高效且可行的计划仍是开放问题。
- **多模态环境反馈**：现有评估多依赖文本反馈，缺乏对真实多模态环境的评估。
- **细粒度评估**：大多数基准仅关注最终结果，缺少对中间步骤的细粒度评估。
