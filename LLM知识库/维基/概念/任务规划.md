# 任务规划（Planning）

## 定义

任务规划是 [[LLM知识库/维基/概念/AI Agent|AI Agent]] 最核心的能力之一。它指 Agent 在给定目标和环境约束下，自动生成一系列可执行步骤的能力。形式化表示为：

$$p = (a_1, a_2, ..., a_t) = plan(E, g; \Theta, P)$$

其中 $E$ 是环境，$g$ 是目标，$\Theta$ 是 LLM 参数，$P$ 是提示词。

## 五大方向分类

根据规划能力综述，LLM Agent 规划方法分为五大类：

### 1. 任务分解（Task Decomposition）
将复杂任务分解为可管理的子任务。

- **先分解法（Decomposition-First）**：先全部分解再逐一执行
  - [[LLM知识库/维基/概念/思维链|CoT]]、HuggingGPT、Plan-and-Solve
- **交错分解法（Interleaved）**：边分解边执行
  - ReAct、Chain-of-Thought with interleaving

### 2. 多计划选择（Multi-plan Selection）
生成多个候选方案，选择最优。

- **自一致性（Self-consistency）**：多次采样取多数
- **思维树（Tree-of-Thought）**：树状搜索空间探索
- **思维图（Graph-of-Thought）**：图结构推理

### 3. 外部模块辅助规划（External Planner-Aided）
结合外部规划器提升可靠性。

- **符号规划器**：PDDL、CLINGO（ASP 求解器）
- **神经规划器**：Decision Transformer、SayCan
- LLM 负责形式化，外部规划器负责求解

### 4. 反思与优化（Reflection & Refinement）
从执行反馈中学习改进。

- **Self-refine**：迭代式生成-反馈-优化
- **Reflexion**：失败经验转化为语言反馈
- **CRITIC**：使用外部工具验证并修正

详见 → [[LLM知识库/维基/概念/反思与优化]]

### 5. 记忆增强规划（Memory-augmented Planning）
利用外部记忆辅助规划。

- **RAG 式记忆**：检索相关知识辅助规划
- **具身记忆**：从物理交互中积累经验

详见 → [[LLM知识库/维基/概念/记忆机制]]

## 核心挑战

- **幻觉传播**：分解过程中的错误会逐级放大
- **计算成本**：树/图搜索策略资源消耗大
- **长程依赖**：多步规划中上下文可能丢失
- **评估困难**：缺乏统一的规划能力评估基准
