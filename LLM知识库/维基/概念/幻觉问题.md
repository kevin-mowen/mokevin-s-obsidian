# 幻觉问题（Hallucination）

## 定义

幻觉是指 LLM 生成的内容看起来合理但实际上**不准确、虚构或与事实不符**的现象。在 [[LLM知识库/维基/概念/AI Agent|AI Agent]] 场景下，幻觉问题尤为严重 — Agent 不只是生成文本，还会基于幻觉内容**执行实际操作**，可能导致严重后果。

## 在 Agent 中的影响

- **规划幻觉**：[[LLM知识库/维基/概念/任务规划|任务分解]] 中生成不可行的步骤
- **事实幻觉**：生成不存在的 API、错误的命令
- **传播放大**：[[LLM知识库/维基/概念/多Agent协作|多 Agent]] 系统中，一个 Agent 的幻觉可能通过通信传播给其他 Agent
- **记忆污染**：幻觉内容被存入 [[LLM知识库/维基/概念/记忆机制|长期记忆]]，长期影响 Agent 表现

## 缓解方法

- **检索增强生成（RAG）**：用外部知识库校验和补充 LLM 输出
- **RLHF / DPO 对齐**：训练模型拒绝编造信息
- **[[LLM知识库/维基/概念/反思与优化|反思机制]]**：如 CRITIC，使用工具验证输出
- **具身化验证**：Agent AI 综述指出，在具身环境中 Agent 会收到环境真实反馈，天然减少幻觉
- **多 Agent 交叉验证**：多个 Agent 互相检查输出

## 四篇论文的共识

这是四篇论文唯一**全部讨论**的挑战，共识是：幻觉是 LLM Agent 大规模落地的最大障碍之一。
