# 多 Agent 协作（Multi-Agent Collaboration）

## 定义

多 Agent 协作是指多个 [[LLM知识库/维基/概念/AI Agent|AI Agent]] 通过通信和协调来共同完成复杂任务的范式。相比单 Agent 系统，多 Agent 系统通过专业化分工和集体智能，能处理更复杂的问题。

## 系统架构

一个 LLM 多 Agent 系统包含四个关键层面：

### 1. Agent-环境接口
Agent 与环境的交互方式：
- **沙盒环境**：Minecraft、代码环境、游戏规则
- **物理环境**：机器人、现实世界
- **人类交互**：人在回路中

### 2. Agent 画像（Profiling）
定义每个 Agent 的角色和行为特征：
- **预定义型**：人工指定角色（如"你是一个程序员"）
- **模型生成型**：由 LLM 自动生成角色画像
- **数据驱动型**：从数据中学习角色特征

### 3. Agent 通信

**通信范式**：
| 范式 | 描述 | 适用场景 |
|------|------|----------|
| **合作** | 共享目标，信息交换 | 软件开发、科研协作 |
| **辩论** | 论点对抗，达成共识 | 决策制定、科学辩论 |
| **竞争** | 各自目标可能冲突 | 博弈、市场模拟 |

**通信结构**：
- **分层（Layered）**：层级通信，如 DyLAN
- **去中心化（Decentralized）**：P2P 网络
- **集中式（Centralized）**：中心节点协调
- **共享消息池（Shared Message Pool）**：MetaGPT 的方案，Agent 发布和订阅消息

### 4. 能力获取
Agent 如何在协作中成长：
- **反馈学习**：从环境反馈、人类反馈、Agent 间反馈中学习
- **自我演化**：Agent 自主反思和调整策略

## 应用场景

### 问题求解
- **软件开发**：ChatDev、MetaGPT — 多角色协作开发（产品经理→架构师→程序员→测试）
- **科学实验**：多 Agent 进行科学辩论和实验设计
- **具身协作**：多机器人协调（RoCo、Co-NavGPT）

### 世界模拟
- **社会模拟**：Stanford Generative Agents（"小镇"实验）、SOTOPIA
- **博弈与策略**：Werewolf、Avalon 等游戏中的 Agent 博弈
- **经济模拟**：推荐系统、市场行为模拟

## 关键框架

| 框架 | 特点 |
|------|------|
| **MetaGPT** | SOP 驱动的多角色软件开发 |
| **AutoGen** | 灵活的多 Agent 对话框架 |
| **CAMEL** | 角色扮演驱动的双 Agent 协作 |
| **ChatDev** | 虚拟软件公司，端到端开发 |
| **AgentVerse** | 动态 Agent 组合和协调 |

## 关键挑战

- **通信效率**：Agent 间大量消息交换的开销
- **一致性**：多 Agent 间信息同步和冲突解决
- **可扩展性**：Agent 数量增加时的管理复杂度
- **幻觉放大**：错误信息可能在 Agent 间传播
