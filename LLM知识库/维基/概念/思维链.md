# 思维链（Chain-of-Thought, CoT）

## 定义

思维链是一种通过引导 LLM **逐步展示推理过程**来提升复杂任务表现的提示技术。由 Wei et al. (2022) 提出，核心思想是在输入提示中加入中间推理步骤的示例，从而让模型"展示它的工作过程"。

## 变体与扩展

| 方法 | 思路 | 特点 |
|------|------|------|
| **标准 CoT** | 提供手动编写的推理示例 | 少样本，需人工示例 |
| **Zero-shot CoT** | 仅添加 "Let's think step by step" | 零样本，简单但有效 |
| **Self-consistency** | 多次采样 + 多数投票 | 提升可靠性，增加计算 |
| **思维树 (ToT)** | 树状搜索多个推理路径 | 适合需要探索的问题 |
| **思维图 (GoT)** | 图结构连接推理节点 | 更灵活的推理拓扑 |
| **Plan-and-Solve** | "Let's first devise a plan" | 先规划后执行 |

## 在 Agent 中的角色

CoT 是 [[LLM知识库/维基/概念/AI Agent|AI Agent]] 推理和 [[LLM知识库/维基/概念/任务规划|任务规划]] 的基础方法：

- **任务分解**：CoT 自然地将复杂任务分解为步骤
- **决策制定**：Multi-Agent 系统中 Agent 的决策思维依赖 CoT
- **自我反思**：CoT 的推理轨迹是 [[LLM知识库/维基/概念/反思与优化|反思]] 的基础

## 局限性

- 推理步骤中的错误会被后续步骤放大
- 长推理链可能导致上下文窗口溢出
- 在某些任务上不如直接回答有效
