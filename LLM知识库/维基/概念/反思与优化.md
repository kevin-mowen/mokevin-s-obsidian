# 反思与优化（Reflection & Refinement）

## 定义

反思与优化是 [[LLM知识库/维基/概念/AI Agent|AI Agent]] 从执行反馈中学习、识别错误并迭代改进的能力。这是 Agent 自我提升的核心机制，类似于人类的"复盘"过程。

## 核心范式

### Self-refine（自我优化）
迭代式的生成-反馈-优化循环：
1. LLM 生成初始输出
2. LLM 自己评估输出质量，生成反馈
3. LLM 根据反馈修正输出
4. 重复直到满意

### Reflexion（反思学习）
将失败经验转化为语言形式的反馈：
$$r_i = reflect(E, g, p_i; \Theta, P)$$
$$p_{n+1} = refine(E, g, p_n, r_i; \Theta, P)$$
- 不更新模型权重，而是用自然语言记录教训
- 下次遇到类似问题时检索这些"反思笔记"
- 将反思存入 [[LLM知识库/维基/概念/记忆机制|长期记忆]]

### CRITIC（工具验证式反思）
使用外部工具验证 LLM 输出：
- 代码执行器验证代码正确性
- 搜索引擎验证事实准确性
- 计算器验证数值结果

### SwiftSage（快慢思考）
受心理学双系统理论启发：
- **快思考**：小模型快速处理简单问题
- **慢思考**：大模型仔细推理复杂问题
- 在两者间动态切换

## 在不同场景中的应用

| 场景 | 反思方式 |
|------|----------|
| [[LLM知识库/维基/概念/任务规划\|规划]] | 规划失败后反思原因，修正计划 |
| 代码生成 | 运行测试 → 分析错误 → 修复代码 |
| [[LLM知识库/维基/概念/多Agent协作\|多 Agent]] | Agent 间互相评审和纠错 |
| 问答 | 检索验证 → 修正错误事实 |

## 局限性

- 自我反思可能陷入"echo chamber"，无法发现自身盲点
- 多轮反思增加延迟和计算成本
- 形式化的反思信号（如测试结果）比自然语言反馈更可靠
