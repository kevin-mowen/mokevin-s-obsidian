# AI Agent（智能体）

## 定义

AI Agent 是一类能够**感知环境、做出决策并执行行动**的智能系统。在 LLM 时代，Agent 通常指以大语言模型为核心"大脑"，结合工具使用、记忆管理和规划能力，能够自主完成复杂任务的系统。

不同论文对 Agent 有不同侧重的定义：

- **Agent AI 综述**将其定义为能在物理世界和虚拟世界中感知和行动的多模态交互系统，是通向 AGI 的一条路径。核心架构为 Agent Transformer — 一个统一的端到端模型。
- **个人 LLM Agent 综述**聚焦于与个人数据深度集成、面向个人助理场景的 LLM Agent，提出 L1-L5 的智能等级划分。
- **多 Agent 综述**关注多个 LLM Agent 如何协作、通信和编排，利用集体智能解决复杂问题。
- **规划能力综述**则聚焦 Agent 最核心的规划能力，提出五大方向的分类框架。

## Agent 分类

### 按能力范围
- **通用型 Agent（Generalist）**：跨领域、跨模态的通用智能体
- **具身 Agent（Embodied）**：在物理或虚拟环境中行动，包括动作 Agent 和交互 Agent
- **生成式 Agent（Generative）**：擅长内容生成，包括 AR/VR/混合现实
- **知识与推理 Agent**：知识 Agent、逻辑 Agent、情感推理 Agent、神经符号 Agent

### 按规模
- **单 Agent 系统**：单一 LLM 驱动，适合明确任务
- **多 Agent 系统**：多个 Agent 协作/辩论/竞争，适合复杂问题求解和世界模拟

### 按个人化程度（L1-L5）
- **L1 通知型**：被动信息推送
- **L2 简单对话型**：Q&A 交互
- **L3 条件执行型**：在用户确认下执行任务
- **L4 自主学习型**：根据上下文自主行动
- **L5 完全自主型**：全面管理用户数字生活

## 关键挑战

- [[LLM知识库/维基/概念/幻觉问题]]：Agent 可能生成不可靠信息并据此行动
- 安全与隐私：个人数据保护、对抗攻击、提示注入
- 效率：推理延迟、资源消耗、端侧部署
- 可解释性：Agent 决策过程的透明度
- 评估：缺乏统一的 Agent 能力评估基准
