# 知识库概述

本知识库基于 Andrej Karpathy 提出的 **LLM Wiki 模式**构建。

## 核心理念

与传统 RAG 不同，本知识库不是每次查询时从原始文档重新检索，而是由 LLM **渐进式构建和维护一个持久化的 Wiki**。知识被编译一次，持续更新，不断复合增值。

## 架构

```
原始资料（不可变） → LLM 处理 → 维基（持续演进） ← 模式说明（指导规范）
```

- **原始资料**：信息源头，LLM 只读不写
- **维基**：LLM 生成的结构化知识，持续维护
- **模式说明**：指导 LLM 如何维护 Wiki 的配置文件

## 操作模式

1. **摄入**：添加新来源 → LLM 阅读、提取、整合到 Wiki
2. **查询**：提问 → LLM 搜索 Wiki、综合答案 → 有价值的答案归档
3. **审查**：定期健康检查 → 修复矛盾、补充缺失、更新过时内容
