# LLM 知识库 — 模式说明

> 本文件是 LLM Agent 的核心配置文件，定义了 Wiki 的结构、约定和工作流。
> 灵感来源：[[ai文献/Karpathy大神的LLM Wiki翻译|Karpathy 的 LLM Wiki 模式]]

## 目录结构

```
LLM知识库/
├── 模式说明.md          ← 本文件（Schema 层）
├── 索引.md              ← 内容目录，按类别组织
├── 日志.md              ← 时间线记录，只追加
│
├── 原始资料/            ← 不可变的信息源头
│   ├── 文章/            ← 网络文章、博客
│   ├── 论文/            ← 学术论文
│   ├── 书籍/            ← 书籍笔记、章节
│   ├── 播客笔记/        ← 播客、视频笔记
│   ├── 会议记录/        ← 会议摘要
│   ├── 数据文件/        ← 数据、表格
│   └── 附件/            ← 图片、PDF 等
│
└── 维基/                ← LLM 生成和维护的知识层
    ├── 概述.md          ← Wiki 全局概述
    ├── 实体/            ← 人物、组织、工具、产品
    ├── 概念/            ← 抽象概念、理论、方法论
    ├── 主题/            ← 具体主题领域
    ├── 来源摘要/        ← 每个来源的摘要页面
    ├── 综合分析/        ← 跨来源的综合性分析
    └── 比较/            ← 对比分析页面
```

## 约定

### 文件命名
- 使用中文命名，简洁明了
- 实体页面：`实体/张三.md`、`实体/OpenAI.md`
- 概念页面：`概念/注意力机制.md`
- 来源摘要：`来源摘要/[来源标题].md`

### Frontmatter 规范
每个 Wiki 页面应包含以下 YAML frontmatter：
```yaml
---
tags: [相关标签]
date: YYYY-MM-DD         # 创建日期
updated: YYYY-MM-DD      # 最后更新日期
sources: []              # 关联的原始资料
type: 实体|概念|主题|摘要|综合|比较
---
```

### 链接规范
- 使用 Obsidian Wiki-links：`[[维基/概念/注意力机制]]`
- 交叉引用其他 Wiki 页面时使用相对路径
- 引用原始资料时使用：`[[原始资料/文章/xxx]]`

## 工作流

### 1. 摄入（Ingest）
1. 用户将新来源放入 `原始资料/` 对应子文件夹
2. LLM 阅读来源，与用户讨论关键要点
3. 在 `维基/来源摘要/` 创建摘要页面
4. 更新 `索引.md`
5. 更新或创建相关的实体/概念/主题页面
6. 在 `日志.md` 追加记录

### 2. 查询（Query）
1. LLM 先读 `索引.md` 定位相关页面
2. 深入阅读相关 Wiki 页面
3. 综合出带引用的答案
4. 有价值的答案可归档为新 Wiki 页面

### 3. 审查（Lint）
定期检查：
- [ ] 页面间是否存在矛盾
- [ ] 是否有被新来源取代的过时信息
- [ ] 是否有无入链的孤岛页面
- [ ] 是否有被提及但缺少独立页面的重要概念
- [ ] 交叉引用是否完整
- [ ] 是否有可通过搜索填补的数据空白
